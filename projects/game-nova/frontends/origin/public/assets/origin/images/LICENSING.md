## Лицензионный статус

Все указанные ассеты — оригинальные битмапы из исходной игры oxsar2
(см. `d:\Sources\oxsar2`). Они либо тривиальные графические примитивы
(одноцветные градиенты, точки, стрелки, прозрачные слои), либо
анимированный звёздный фон без узнаваемых сторонних элементов. По
**R4 плана 41** («оригинальность ассетов») приняты в репо без
изменений.

## Что НЕ перенесено и почему

### Шрифты (проприетарные, не используются)

В [`projects/game-legacy-php/public/fonts/`](../../../../game-legacy-php/public/fonts)
лежат:

- `AGHELV_J.TTF` (Helvetica-derived, 36 KB) — **не переносится**.
- `FUTUR_14.TTF` / `FUTUR_19.TTF` (Futura-derived, 45 KB) —
  **не переносится**.

**Причина:** оба — коммерческие шрифты-наследники Helvetica/Futura,
использование без лицензии запрещено. **Грэп по `legacy/css/`
показал, что они нигде не подключены через `@font-face`** — то есть
в самом legacy-PHP они просто лежат, фактически не применяясь.
Origin-фронт использует системный fallback `Georgia, Verdana, Arial,
Helvetica, sans-serif` (см. `theme.css` → `--font-base`), полностью
покрывающий визуал legacy без проприетарных файлов.

### Спрайты юнитов / зданий / артефактов / отчётов

Папки `images/buildings/`, `images/units/`, и подобные (~700 MB
суммарно в legacy-PHP) **на Ф.1 не переносятся**. Они будут
портироваться по мере подключения соответствующих экранов в
Spring 1-3 (план 72 Ф.2-Ф.4) — каждая партия сопровождается
проверкой:

1. Оригинальность каждого спрайта (не лицо известного актёра, не
   лого компании, не кадр из стороннего фильма).
2. По возможности — переход на SVG (вес + масштабируемость).
3. Запись в этот `LICENSING.md` (или его дочерний по подпапке).

### Реклама / баннеры

Файлы `bann.png`, `b.gif` и любые рекламные блоки **не переносятся**
по решению плана 72 («Что НЕ делаем в первой итерации» → «Баннеры и
рекламные тексты из legacy-PHP не переносятся»).

## Процедура добавления новых ассетов

1. Перед `cp` — `file <asset>` чтобы убедиться в типе.
2. Проверить размер: `>200 KB` файлы требуют отдельного обоснования
   (избегаем bloat репо).
3. Поиск по EXIF / визуальная инспекция — нет ли стороннего лого,
   фотографии человека, кадра из фильма/игры.
4. Дописать строку в таблицу выше + git commit.

При сомнении — **не добавлять**, поднять вопрос в плане 72.
