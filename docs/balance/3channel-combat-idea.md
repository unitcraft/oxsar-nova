# 3-канальная боевая система

В исходной legacy `oxsar2-java` задумывалась 3-канальная боевая система,
но никогда не была реализована — код закомментирован в
[`Participant.java:530-538`](../legacy/game-reference.md). В oxsar-nova
MVP она тоже отсутствует — упрощено до scalar Attack/Shield 2026-04-25
(см. [docs/simplifications.md](../simplifications.md) → M4.1).

## Дизайн системы

**3 технологии оружия** (вместо одной `gun_tech`):
- лазер
- ион
- плазма

**3 типа щита**:
- физический
- магнитный
- силовой

**Матрица соответствия `ADV_TECH_MATRIX[3][3]`** — насколько каждый тип
атаки эффективен против каждого типа щита (вроде ножниц-камень-бумага).
В legacy заявлено как `Assault.ADV_TECH_MATRIX`.

**Механика выбора:**
- Игрок распределяет очки между 3 технологиями → его флот специализируется
  на одном типе атаки.
- Балансные ножницы: лазерный флот силён против магнитных щитов, но
  слаб против силовых.

## Зачем это игре

Глубина выбора в PvP. Сейчас один способ победы — больше Звёзд смерти.
Ножницы между типами атак дают:

- **Реальные контр-флоты** — мотивирован выбор технологий, а не «качай
  всё подряд».
- **Смысл в шпионаже** — узнать тип атаки противника, чтобы выставить
  правильный щит.
- **3 ветки исследований вместо 1** — больше long-term-цели.
- **Альянсы** — игроки специализируются по разным типам атаки и
  объединяются.

## Почему не сейчас

- **Сложность для новичка** — нужно понимать матрицу 3×3 эффективности.
- **Ребаланс всех юнитов** — атака распределяется по каналам.
- **Это расширение, а не порт** — выходит за рамки 1:1 oxsar2.

## Как реализовать (когда дойдут руки)

1. Восстановить инфраструктуру: `Attack [3]float64` / `Shield [3]float64`
   в `backend/internal/battle/types.go`. Раньше она была, удалена 2026-04-25.
2. Восстановить функцию `primaryChannel` в `engine.go` или реализовать
   per-shot распределение по каналам.
3. Добавить 3 технологии в `configs/research.yml`: `laser_tech`, `ion_tech`,
   `plasma_tech`.
4. Добавить матрицу `ADV_TECH_MATRIX[3][3]` в `configs/battle.yml`
   (новый файл).
5. Распределить атаку юнитов по каналам: например, истребитель
   `[100, 0, 0]` (чисто лазерный), Звезда смерти `[100k, 50k, 50k]`
   (микс).
6. Распределить щит по типам: например, легкий лазер блокирует физический,
   плазма-пушка — все три равномерно.

## Где это уместно

- **Отдельный режим / сервер** — «классический oxsar» (1 канал) и
  «расширенный» (3 канала). Игроки выбирают на регистрации.
- **Не основная ветка** — иначе порт-паритет с Java JAR ломается
  навсегда.
