# ADR-0011: Display name для вселенной origin — Oxsar Classic

- Status: **Accepted** (2026-04-28)
- Date: 2026-04-28
- Связанные документы:
  - [ADR-0010](0010-universe-domain-naming.md) — поддомены `uni##`.
  - [план 62](../plans/62-origin-on-nova-feasibility.md) — feasibility
    ремастера origin.
  - [план 72](../plans/72-remaster-origin-frontend-pixel-perfect.md) —
    pixel-perfect клон фронта.
  - [план 45](../plans/45-trademark.md) — товарный знак «oxsar».

## Context

В архитектуре имя вселенной живёт в двух слоях:

1. **Технический slug** — `universe_id` в БД, имя в URL, имя папки
   (`projects/game-nova/frontends/origin/`), имя в configs
   (`configs/balance/origin.yaml`). Сейчас — `origin`. **Менять не будем**:
   200+ упоминаний в коде, миграциях, doc'ах. Цена смены = план уровня 78.

2. **Display name** — что игрок видит на портале при выборе вселенной,
   в шапке игрового фронта, в маркетинговых материалах. Здесь свобода.

До этого ADR display name не был зафиксирован: в research-доках
встречалось «origin» как в техническом, так и в маркетинговом смысле,
что путало.

## Варианты, рассмотренные

| Вариант | Плюсы | Минусы | Решение |
|---|---|---|---|
| **Origin 2010** | ностальгия, чёткая привязка к эпохе | legacy oxsar2 запущен в 2008 не 2010 (фактическая ошибка); цифра стареет; для новичков ничего не значит; редкий маркетинговый паттерн (год в названии = одноразовое издание) | ❌ |
| **Origin** (без всего) | короткое | конфликт с EA Origin (store/launcher) — SEO-провал, юр-риск товарного знака | ❌ |
| **Oxsar Origin** | бренд + ветка | «Origin» несёт чужой ассоциативный груз (EA) | ❌ |
| **Oxsar Classic / Оксар Классик** | стандартный gaming-паттерн (WoW Classic, CS Classic, D2 Resurrected); понятно сразу; не стареет; без чужих брендов; «Classic» как generic-описательный термин юридически чист в комбинации с собственным брендом; благозвучно на ru+en | паттерн избитый | ✅ принят |
| **Oxsar Legacy** | прямая отсылка | в нашем коде `legacy` = техническое слово (legacy-PHP); «legacy» в gaming часто = «устаревшее, заброшенное» | ❌ |
| **Oxsar Retro** | модно, тренд | подразумевает стилизацию под старое, а не возврат к нему | ❌ |
| **Oxsar: Origins** (мн. ч.) | gaming-паттерн (Far Cry, AC) | конкуренция с EA-Ubisoft брендами | ❌ |
| **Oxsar Prime** | gaming-паттерн (Mass Effect, Metroid) | сильная ассоциация с Amazon Prime | ❌ |
| **Oxsar Original** | прямой смысл | банально | ❌ |
| **Oxsar Vintage** | красиво | редкий gaming-паттерн, ассоциации с вином | ❌ |

## Decision

**Display name для вселенной с техническим slug `origin` —
«Oxsar Classic» (en) / «Оксар Классик» (ru).**

## Раскладка по контекстам

| Контекст | Что показываем |
|---|---|
| Технический slug в БД, configs, URL, импортах | `origin` |
| Папка в репо | `projects/game-nova/frontends/origin/` |
| Шапка origin-фронта (i18n key `app.universe.name`) | «Oxsar Classic» / «Оксар Классик» |
| Портал, выбор вселенной, баланс widget | «Oxsar Nova», «Oxsar Classic» |
| Маркетинговые материалы | «Oxsar Classic — возвращение в золотую эпоху космических стратегий» |
| Юр-документы | «вселенная Oxsar Classic (slug: origin)» |
| Title тег HTML, og:title | «Oxsar Classic» |
| Тех-документация, ADR, плановые документы | `origin` (как идентификатор) или «Oxsar Classic» (когда речь о бренде) |

## Хранение

В таблице `universes` уже есть display-поля (см. план 36 / migration
0073_init или подобную). Вписать:
- `slug = 'origin'`
- `display_name_en = 'Oxsar Classic'`
- `display_name_ru = 'Оксар Классик'`

Если поля называются иначе или их нет — добавить миграцией в момент
реализации (Ф.2 плана 72 или отдельный mini-rename-план).

Все потребители (frontend-портал, headers origin-фронта, маркет-материалы)
читают display_name из этого источника, никаких хардкодов.

## Consequences

**Положительные:**
- Юридически чистое имя, не конфликтует с чужими товарными знаками.
- Стандартный gaming-паттерн → игроки сразу понимают позиционирование.
- Не стареет.
- Хорошо звучит на русском и английском.
- Бренд продукта читается чище: «Oxsar» — головное имя, «Classic/Nova» —
  ветки.

**Нейтральные:**
- Технический slug `origin` остаётся; это ок (slug — внутренний
  идентификатор, не маркетинговое имя).
- Рассинхрон между slug `origin` и display name `Classic` создаёт
  легкую когнитивную нагрузку для разработчиков. Митigation: ADR
  фиксирует соответствие, путаница уходит.

**Отрицательные:**
- В исторических планах/доках слово `origin` использовалось
  амбивалентно (и как slug, и как имя). Менять задним числом не будем
  (правило плана 55: исторические записи не трогаем).

## Применение

Применяется немедленно для:
- Будущих маркетинговых материалов.
- Шапки origin-фронта (план 72 Ф.2 — учесть при реализации Login/Register
  экранов).
- Любых новых текстов для игроков.

Не применяется для:
- Технических идентификаторов в коде/configs/БД (там навсегда `origin`).
- Закрытых планов и project-creation.txt (исторические записи).

## Memory

Записать в memory `project_origin_vs_nova.md` уточнение:
«Технический slug = `origin`. Display name = Oxsar Classic / Оксар
Классик. Использовать display только в UI/маркетинге/юр-документах,
slug — только в коде/configs/URL.»
