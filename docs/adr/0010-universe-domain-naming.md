# ADR-0010: Именование поддоменов вселенных

- Status: **Proposed** (открытый вопрос, решение не принято)
- Date: 2026-04-28
- Связанные документы:
  [docs/plans/36-portal-multiverse.md](../plans/36-portal-multiverse.md) —
  закрытый план мультивселенной, в котором зафиксирован текущий
  префикс `uni##`.

## Context

Сейчас вселенные публикуются на поддоменах вида
`uni01.oxsar-nova.ru`, `uni02.oxsar-nova.ru` (см. план 36, dev-stack
для uni02 — коммит `7d546cd389`). Префикс `uni` происходит от
«universe» и был выбран по умолчанию без обсуждения.

Вопрос: подходит ли `uni##` как долгоживущее именование, или есть
лучший вариант для брендинга / лора / индустриального стандарта?

## Варианты

### A. Оставить `uni##`

`uni01.oxsar-nova.ru`, `uni02.oxsar-nova.ru`, …

**+** Status quo, ничего не нужно мигрировать. Очевидный counter,
легко скриптовать deploy. Универсально (любая вселенная = universe,
вне зависимости от баланса/сезона).

**−** Утилитарно, не продаёт сеттинг. Игроки будут говорить
«я на uni-01» — это не лор, это техническая метка.

### B. Короткий стандартный префикс `s##` (Travian-стиль)

`s1.oxsar-nova.ru`, `s2.oxsar-nova.ru`, …

**+** Индустриальный паттерн браузерных ММО (Travian, eRepublik).
Короче чем `uni`. Минимально-агрессивная маркировка.

**−** Безликость как у `uni`, плюс возможный конфликт с потенциальным
staging/system-доменом (`s.` уже занято в умах админов под
«system»/«staging»). Не лорный.

### C. Тематический префикс `gx##` (galaxy)

`gx01.oxsar-nova.ru`, `gx02.oxsar-nova.ru`, …

**+** Точно ложится на космо-лор oxsar-nova: вселенная →
галактика → сектор → система → планета. Игроки уже мыслят
галактиками. Короткий, индустриальный паттерн (короткий префикс
+ число).

**−** Семантически натяжка: «галактика» в игре — это сущность
**внутри** вселенной (game-nova хранит galaxy=N), а на поддомене
будет внешняя «галактика номер 1». Возможна терминологическая
путаница.

### D. Брендовый префикс `ox##`

`ox01.oxsar-nova.ru`, `ox02.oxsar-nova.ru`, …

**+** Брендовый, короткий.

**−** Визуально дублирует имя домена: `ox01.oxsar-nova.ru` —
два «ox» подряд, плохо читается.

### E. Path-based вместо поддомена

`oxsar-nova.ru/u/uni01`, `oxsar-nova.ru/u/01`.

**+** Один SSL-сертификат, одна Vite-сборка, нет cross-origin
проблем (CORS, ALLOWED_ORIGINS, SameSite cookies). Проще
эксплуатация.

**−** Хуже брендинг — «портал владеет вселенной», а не
вселенная как самостоятельная сущность. Сложнее раздробить
CDN/кеш по вселенным. Меняет всю архитектуру маршрутизации
(сейчас поддомен = отдельный фронт).

### F. Гибрид: `code` + `display_name`

В реестре `universes` два поля:

- `code` — стабильный технический идентификатор (`uni01`,
  `gx01`, `s1` — выбор зависит от пунктов A–C). Используется
  в DNS, БД, deploy-скриптах.
- `display_name` — тематическое имя для UI, footer,
  switcher, чата (`Андромеда`, `Орион`, `Вега`, `Лебедь`).

**+** Можно сохранить любой технический префикс и при этом
дать игрокам живые имена. Перенос/закрытие вселенной не ломает
идентификаторы. Маркетингово сильно.

**−** Требует кода: реестр имён, fallback-логика
(`display_name ?? code`), миграция БД (`ALTER TABLE universes
ADD COLUMN display_name`). Нужно следить, чтобы имена не
заканчивались (10+ красивых созвездий — ок, 50+ —
натянуто).

## Что важно учесть

- **CORS уже работает на `uni01`**, любая смена префикса
  потребует обновления `ALLOWED_ORIGINS` в portal-backend
  (план 56) и admin-bff.
- **JWT issuer / audience** не привязан к домену, но
  fingerprint anti-fraud (план 59) может использовать
  origin — проверить.
- **SEO / закладки** — если уже есть индексация
  `uni01.oxsar-nova.ru`, миграция требует 301-редиректа.
  На текущей стадии индексации нет, миграция дешёвая.
- **Маркетинг** — если решение принимается **до** публичного
  запуска, цена смены ≈ 0. После запуска — болезненно.
- **Количество вселенных** — если планируется 3-5 (классика
  oxsar2, speed, hardcore, тестовая), хватит и тематических
  имён без числового префикса. Если 20+ — нужен счётчик.

## Decision

**Не принято.** Вопрос отложен до момента, когда:

1. Будет ясна целевая структура вселенных (сколько, с какими
   параметрами — ср. план 58 / план 59 могут уточнить).
2. Будет принято решение о публичном запуске и его сроках
   (если до запуска — менять можно бесплатно, после — дорого).

**Текущее значение по умолчанию:** `uni##` (вариант A) —
действует, пока не принято иное.

## Когда вернуться к решению

- Перед публичным запуском (последний дешёвый момент для
  смены).
- При появлении 3-й вселенной (если параметры расходятся,
  тематическое имя становится полезным).
- При запуске реферальной программы (план 59) — если
  реферальная ссылка содержит `&hint=<code>`, формат `code`
  становится виден игроку.

## Альтернативы, которые точно НЕ выбираем

- **Без префикса вообще** (`oxsar-nova.ru` сам по себе =
  вселенная) — несовместимо с мультивселенной из плана 36.
- **Случайные слова** (`pumpkin.oxsar-nova.ru`) —
  непредсказуемо, не масштабируется.

## История

- 2026-04-28 — ADR создан в статусе Proposed как trigger
  для будущего решения. Текущий префикс `uni##` сохраняется
  до явного пересмотра.
