# ADR-0004: Нерф стоимости Lancer Ship (отклонение от legacy)

- Status: Accepted
- Date: 2026-04-24
- План: [docs/plans/21-gameplay-hardening.md](../plans/21-gameplay-hardening.md) блок A
- Симуляция: [docs/balance/simulation-2026-04-24.md](../balance/simulation-2026-04-24.md)

## Context

В аудите баланса 2026-04-24 зафиксирован эксплойт BA-002: Lancer Ship
(id=102) имеет attack/1k = 220 при медиане 12–17 у базовых юнитов —
в 13–15× эффективнее по атаке на единицу ресурсов.

Legacy oxsar2 компенсирует это сильным rapidfire-графом против Lancer:
LF→Lancer ×20, SF→Lancer ×20, Cruiser→Lancer ×35, DS→Lancer ×100.
В nova rapidfire был урезанной заглушкой — эти записи отсутствовали.

Фаза 1 плана 18 (коммит `7f5ce71`) портировала legacy-rapidfire.
Симуляция после порта показала:

| Сценарий | Exchange (def_loss/atk_loss) |
|---|---:|
| 1000 Lancer vs 1000 Cruiser (равная стоимость) | 5.11 |
| 500 Lancer vs 200 LF + 100 Cru + 50 BS | 2.89 |

**Вывод**: rapidfire-порт помог (exchange упал с ожидаемых 13–15× до
5.11×), но не закрыл эксплойт. Причина — rapidfire ×35 срабатывает
только после того, как Cruiser **убил** Lancer, а Lancer убивает
Cruiser быстрее (attack 5500 vs 400, разница ×13). К моменту, когда
Cruiser начинает получать rapidfire-bonus, он уже понёс критические
потери.

## Decision

Поднять стоимость Lancer (`configs/construction.yml:521`):

| Ресурс | Было | Стало | ×  |
|---|---:|---:|---:|
| Metal | 2 500 | 15 000 | ×6.0 |
| Silicon | 7 500 | 35 000 | ×4.7 |
| Hydrogen | 15 000 | 60 000 | ×4.0 |
| **metal-eq** | **25 000** | **110 000** | **×4.4** |

Структура стоимости после изменения: 14% M / 32% Si / 54% H — водород-heavy,
характерно для эндгейм-юнита.

**attack/1k** после изменения: 5500 / 110 ≈ **50** (уровень Battleship=16.7
до учёта shell, но Lancer компенсирует attack-мощью; в exchange-тестах
даёт 1.16).

## Consequences

### Симуляция с новой стоимостью

Инструмент: `backend/cmd/tools/battle-sim/`.

| Сценарий | Exchange до | Exchange после | Вывод |
|---|---:|---:|---|
| Lancer → Cruiser (1000 vs 1000) | 5.11 | **1.16** | Почти равный размен, Lancer чуть выгоднее (оправдывает элитный статус) |
| Lancer → mix LF+Cru+BS | 2.89 | **0.64** | Lancer-spam против микса теперь невыгоден |
| DS → 300 Lancer | ∞ (def 67%) | ∞ (def 67%) | DS-counter сохранён |

Проверено sweep'ом по 6 вариантам стоимости (×1..×6 от базовой):

| Cost ×N | metal-eq | Exchange |
|---:|---:|---:|
| ×1 (legacy) | 25k | 5.11 |
| ×2 | 50k | 2.55 |
| ×3 | 75k | 1.70 |
| ×4 | 100k | 1.28 |
| ×5 | 125k | 1.02 |
| ×6 | 150k | 0.85 |

Выбран 110k metal-eq (между ×4 и ×5) как оптимум: exchange 1.16 —
Lancer остаётся немного выгоднее обычного флота по ресурс-размену,
что оправдывает его «элитный» статус (specialized anti-Lancer
role через rapidfire и высокую attack), но Lancer-spam как мета
мёртв.

### Плюсы

- Закрытие BA-002 — главного эксплойта баланса, подтверждённого
  симуляцией.
- Сохранение attack/shell/shield/speed юнита — **паритет с Java JAR не
  ломается** (golden-тесты не требуют перегенерации). Меняется только
  конфиг стоимости (construction.yml).
- Lancer остаётся жизнеспособным юнитом: его роль «premium front=8
  rapidfire vs DS» сохранена, но его больше нельзя спамить против
  обычного флота.

### Минусы

- **Отклонение от legacy-баланса** (требование CLAUDE.md: формулы «один-в-один»
  из oxsar2 — это прямое исключение). Обосновано только для
  ресурс-стоимости, не для боевых параметров.
- Существующие игроки в тест-инстансе, если они вложились в Lancer по
  старой цене, получат «упадок в доходности» своих Lancer-флотов.
  Актуально только для dev-seed; продакшн-запуск ещё не было.

### Альтернативы отвергнуты

1. **Снизить attack Lancer 5500 → 3500** — ломает паритет с Java JAR
   (стат юнита изменён, golden-тесты перегенерировать). Плюс не решает
   проблему полностью при новой формуле attack/1k.
2. **Добавить новые rapidfire контры** (Frigate→Lancer, Bomber→Lancer) —
   изобретение, которого нет в legacy. Ещё большее отклонение.
3. **Оставить как есть** — эксплойт остаётся, Lancer-spam доминирует в
   мете. Не вариант для продакшна.

## Проверка

```bash
cd backend
go run ./cmd/tools/battle-sim/ --configs=../configs --scenario=lancer-vs-cruiser --runs=50 --rounds=200
# exchange должен быть ≈ 1.16 после применения
```

## Откат

Изменить в `configs/construction.yml:527-530` стоимость обратно на
2500/7500/15000. Всё.
