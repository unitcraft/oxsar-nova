## 1. Контекст

Legacy [`Market.class.php`](../../projects/game-legacy-php/src/game/page/Market.class.php)
имеет 4 действия:
- `Metal_ex(silicon, hydrogen)` — купить metal за silicon+hydrogen.
- `Silicon_ex(metal, hydrogen)` — купить silicon.
- `Hydrogen_ex(metal, silicon)` — купить hydrogen.
- `Credit_ex(metal, silicon, hydrogen)` — купить ресурсы за **кредиты**
  (multi-resource в одном запросе). При списании отправляется AutoMsg
  `MSG_CREDIT_MARKET` в folder=8.

Origin:
- Backend `market.Exchange` — реализует первые три действия.
- Backend `market.ExchangeCredit` — credit → ресурсы (single-resource,
  не multi). AutoMsg НЕ отправляется.
- Frontend `ResourceMarketScreen` — только M↔S↔H обмен, без вкладки credit.

## 2. Что нужно

### Backend
- `service.ExchangeCredit`: после успешного списания — `automsg.SendDirect`
  в folder=8 (FolderCredit) с шаблоном `creditMarketPurchase` (legacy
  `MSG_CREDIT_MARKET`).
- `Service` нужно расширить полем `automsg + bundle` через `WithAutoMsg`.

### Frontend
- `api/market.ts`: `exchangeCredit(planetId, resource, amount)`.
- `ResourceMarketScreen`: добавить **вторую таблицу** «Купить за кредиты»
  ниже основной (3 строки M/S/H + input количества credit + кнопка).
  Простой UI: per-resource, не multi (т.к. backend single-resource).
  Multi-resource — отдельная задача (расширение backend signature).

### i18n
- `autoMessages.creditMarketPurchase.title/body` (с {{credits}} +
  {{resources}}).
- `market.tabCredit`, `market.creditAmountLabel`,
  `market.youGetForCredits`, `market.creditExchangeBtn`.
