# План 72.1.45 — Финальная зачистка отступлений и недореализаций

**Дата**: 2026-05-01
**Статус**: ✅ Закрыт

**Подплан** [плана 72.1](72.1-post-remaster-stabilization.md) §20.12.
Закрывает 9 пунктов, в которых ранее были отступления / TBD-хвосты,
замаскированные комментариями «отдельный план», «MVP», «не блокирует».
По правилу §20.12 такие формулировки запрещены — каждая часть должна
закрываться полностью.

## Контекст

После завершения 72.1.36-44 ревизия выявила 9 пунктов, замаскированных
ранее как «closed»:

1. **i18n дубликаты**: при удалении `creditAmountLabel`/`bsMoonBattle`
   потерялись разные значения для разных секций. Восстановить под
   правильными именами.
2. **ArtefactInfo history** — legacy `ArtefactInfo.class.php` L.66-93
   показывает таблицу боёв с этим артефактом. В 72.1.38 пометил
   как TBD, но audit написал ✅.
3. **Alliance online-status + candidates view** — в 72.1.43 пометил
   «эстетика, не подпадает под §20.12», но это часть legacy
   функционала.
4. **Constructions UI**: energy/credit/points (legacy 322-330) +
   VIP-progress-визуализация (vip_image, event_percent_timeout).
5. **Shipyard UI**: max_units cap (legacy `getMaxCountOrder*`),
   damaged/shell_percent в табличке inventory.
6. **Simulator destroy_chance**: использовалась упрощённая формула
   `min(100, ds_count × 100 / level)`. Реальная legacy-формула —
   из Java engine, нужна корректная реализация.
7. **Empire УМИ колонка** (researchVirtLab): в 72.1.37 удалил из
   заголовка вместо реализации.
8. **/p2p-exchange broker fee** хардкод 5%. Нужен per-broker fee
   из exchange-таблицы (legacy создаёт запись на каждого юзера в
   exchange).
9. **ExchangeOpts (admin P2P)**: legacy `?go=ExchangeOpts` — admin
   страница для настройки fee/title/comission брокером. blocked
   в audit-таблице 20.0.
