## 1. Контекст

`projects/game-nova/backend/internal/battle/` — порт Java-`Assault.jar`
(oxsar2-java) на Go (ADR-0002). Используется в 5 call sites:
`fleet/attack.go`, `fleet/acs_attack.go`, `fleet/expedition.go` (×2),
`alien/alien.go`, `simulator/handler.go`.

Read-only audit (план 87, ✅ 2026-05-01) прочесал `engine.go` (~1040
строк) + `types.go` + `simstats.go` + 5 call sites + сверка с
reference-Java. Найдено **15 проблем (BA-007..BA-021)**, все
зафиксированы в [docs/balance/audit.md](../balance/audit.md).

Текущий статус (на 2026-05-01):
- **BA-007** ✅ patched (`Input.HasPlanet` — опыт ×2 в обычных атаках
  планет).
- **14 находок** open. Закрытие — задача этого подплана.

## 4. Что нужно сделать

Подплан строится **итеративно** — каждая находка закрывается отдельным
коммитом / серией коммитов. Размер каждой итерации — ≤ 400 строк diff
по CLAUDE.md.

### A. BA-015 — расширение validate (этап 1, в работе сейчас)

**Файлы**:
- `internal/battle/engine.go` — расширить `validate()`.
- `internal/battle/engine_test.go` — table-driven тесты на каждый
  guard.

**Что делаем**:
- Лимиты-константы (`maxQuantity`, `maxAttack`, `maxShield`, `maxShell`,
  `maxFront=30`, `maxTechLevel=99`, `maxRapidfire=1000`, `maxRounds=20`,
  `maxNumSim=100`).
- `validate(in)` проверяет:
  - `in.Rounds ∈ [0, 20]`, `in.NumSim ∈ [0, 100]`;
  - `Rapidfire[i][j] ∈ [1, maxRapidfire]`;
  - для каждой `Side`: `Tech.* ∈ [0, 99]`, для каждого `Unit`:
    `Quantity ∈ [0, maxQuantity]`, `Damaged ∈ [0, Quantity]`,
    `ShellPercent ∈ [0, 100]`, `Front ∈ [0, 30]`,
    `Attack/Shield/Shell ∈ [0, maxX]`, `Cost.Metal/Silicon/Hydrogen ≥ 0`.
- Ошибки оборачиваются через `errors.Is(..., ErrInvalidInput)` — тесты
  чек'ают через `errors.Is`, не string match.

**Тесты**:
- `TestValidate_AcceptsHappyPath` — нормальный input проходит.
- `TestValidate_RejectsMaliciousUnit` — table-driven по 7+ полям
  (Quantity, Damaged, ShellPercent, Front, Attack, Shield, Shell, Cost.*).
- `TestValidate_RejectsMaliciousTech` — лимит 99 для всех 8 tech-полей.
- `TestValidate_RejectsRapidfire_OutOfRange` — < 1 и > 1000.
- `TestValidate_RejectsRoundsAndNumSim_OutOfRange`.
- Все тесты — через `errors.Is(err, ErrInvalidInput)`.

**Definition of Done**:
- [ ] `go test ./internal/battle/...` зелёный (включая новые тесты).
- [ ] `go build ./...` + `go vet ./...` чисто.
- [ ] BA-015 в `audit.md` → **patched** + блок «Фикс».
- [ ] Дневник `project-creation.txt` обновлён.
- [ ] PR ≤ 400 строк (engine.go + engine_test.go).

### B. BA-012 — SimStats.AttackerExp (этап 2)

**Файлы**: `internal/battle/simstats.go`, `types.go`.

Заменить proxy-формулу `atkExp += defLostM + defLostS + defLostH` на
**усреднение реальных `Report.AttackerExp`**, который уже считается
правильно в каждой итерации.

```go
// Было:
atkExp += defLostM + defLostS + defLostH
defExp += atkLostM + atkLostS + atkLostH

// Стало:
atkExp += rep.AttackerExp
defExp += rep.DefenderExp
```

Тип агрегата: `int64` (sum of int) → деление на `fn float64` для avg.
Поле `SimStats.AttackerExp` остаётся `float64` (уже среднее).

**Тесты**:
- `TestMultiRun_AvgExp_MatchesSingleRunExp` — для `n=1` `SimStats.AvgExp`
  совпадает с `Report.AttackerExp` единственного прогона.
- `TestMultiRun_AvgExp_IsAverageNotSum` — для `n=10` среднее ≠ сумма.

**DoD**:
- [ ] `simstats.go` использует `rep.AttackerExp/DefenderExp`.
- [ ] Регрессионные тесты зелёные.
- [ ] `audit.md` BA-012 → patched.

### C. BA-008 — Debris formula 50%/1% (этап 3)

**Файлы**: `internal/fleet/attack.go::calcDebris` (и аналог в
`acs_attack.go` если есть).

**Решение**: переписать `calcDebris` чтобы различать ships vs defense
по `unit_type` (поле `Mode` в Unit / config catalog), применять
`bulkFactor = 0.5` для флота, `0.01` для обороны.

**Сейчас формула 30% × (metal+silicon)** — это **OGame-стандарт**, но
**не legacy**. Нужно решение пользователя: вернуть к legacy 50%/1%
или оставить 30% с записью в `docs/simplifications.md`?

**Open Question** для пользователя.

### D. BA-018 — Семантика Front (этап 4)

использование в `shootAtSides`. Если Java тоже даёт «больше weight =
