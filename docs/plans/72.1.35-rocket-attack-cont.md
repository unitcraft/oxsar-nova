## 2. Что сделано

### Backend

Изменений нет. `internal/rocket/handler.go::Launch` и `Stock`
уже были, маршруты в `cmd/server/main.go` зарегистрированы.

### Frontend

#### `api/rocket.ts` (новый)

```typescript
export interface RocketStock { count: number }
export interface LaunchRequest {
  dst: { galaxy: number; system: number; position: number; is_moon?: boolean };
  count: number;
  target_unit_id?: number; // 0 = "all"
}
export function fetchRocketStock(planetId: string): Promise<RocketStock>;
export function launchRocket(planetId: string, req: LaunchRequest): Promise<LaunchResult>;
```

#### `RocketAttackScreen` переписан

- `useResolvedPlanet` для `srcPlanetId`.
- Цель определяется приоритетом:
  1. URL `/rocket-attack/:id` (params.id) → `fetchPlanet(id)` для
     координат.
  2. Query `?g=&s=&p=&moon=` (передаётся, например, из Galaxy при
     клике на «🚀»).
- `useQuery` `rocket-stock` → `maxRockets`. Submit `disabled` если
  `maxRockets === 0`.
- `useMutation` `launchRocket`:
  - `count` clamp (0 ≤ x ≤ maxRockets).
  - `target_unit_id` из селекта defense-юнитов; `'all'` → 0.
  - Селект фильтрует interceptor (51) и interplanetary (52)
    ракеты (legacy строки 135-138).
  - При успехе invalidate `rocket-stock` + `planets`, `navigate('/')`
    (legacy редирект на `/Main`).
  - Ошибки (400/409/403) — сообщение в красном `false`-классе.

Все legacy-валидации (newbie protection, umode/observer, бэшинг,
isPlanetUnderAttack, protection_time, missile range) backend сам
проверяет и возвращает 400/409 с message → фронт отображает.
