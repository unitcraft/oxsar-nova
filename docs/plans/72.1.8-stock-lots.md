## 1. Контекст

Биржа артефактов в origin (`/stock`) показывает список активных лотов
(`fetchExchangeLots(status='active')`), но **никаких операций**:
- кнопка «Купить» — `disabled`, title «Покупка недоступна».
- нет recall (отозвать свой лот).
- нет create lot (выставить артефакт на продажу).
- статусы лотов не различаются (только `status='active'`).

Backend полностью готов в [`internal/exchange/`](../../projects/game-nova/backend/internal/exchange/):

| Endpoint | Метод | Описание |
|---|---|---|
| `GET /api/exchange/lots` | List | список лотов (status фильтруется) |
| `GET /api/exchange/lots/{id}` | Get | детали одного лота |
| `POST /api/exchange/lots` | CreateLot | выставить артефакт |
| `DELETE /api/exchange/lots/{id}` | Cancel | отозвать свой лот |
| `POST /api/exchange/lots/{id}/buy` | BuyLot | купить лот |
| `GET /api/exchange/stats` | Stats | статистика биржи |

Legacy `Stock.class.php` (757 LoC, 7 actions): `showLots, buyLot,
showLotDetails, recall, recallByGET, premiumLot, ban`. Из них:

- ✅ `showLots` (список) — есть в origin.
- ❌ `buyLot` (купить) — в origin disabled.
- ❌ `recall/recallByGET` (отозвать свой) — в origin отсутствует.
- ❓ `showLotDetails` (детали лота) — у нас детальной страницы нет,
  но `GET /api/exchange/lots/{id}` есть; UI можно добавить modal.
- ⚠ `premiumLot` (выставить премиум) — admin-related, отдельный
  endpoint в legacy, не в наших задачах MVP.
- ⚠ `ban` (admin: забанить лот) — moderator-only, не порт.

## 2. Цель

Подключить **3 базовых операции** Stock UI к существующему backend:
1. **Buy** — кнопка «Купить» открывает форму с количеством (если
   `quantity > 1`), POST `/api/exchange/lots/{id}/buy`.
2. **Cancel** — кнопка «Отозвать» рядом со своими лотами.
3. **CreateLot** — отдельная кнопка «Выставить лот» вверху страницы,
   открывает форму выбора артефакта + количества + цены.

`premiumLot` и `ban` — оставляем за scope MVP, фиксируем в
`docs/simplifications.md` как admin-features.

## 3. Декомпозиция

### A. Подключить Buy + Cancel mutations

- `api/exchange.ts` уже имеет `fetchExchangeLots`. Добавить:
  ```ts
  export function buyLot(lotID: string, amount: number): Promise<...>;
  export function cancelLot(lotID: string): Promise<void>;
  ```
- `StockScreen.tsx`:
  * Кнопка «Купить» включена. По клику — popup с инпутом quantity
    (от 1 до `lot.quantity`) и кнопкой подтверждения.
  * Если `lot.seller_id == myUserID` — вместо «Купить» показывается
    «Отозвать» → `cancelLot()`.
  * После успеха — invalidate query'ы на лоты + ресурсы.

### B. CreateLot UI

- Кнопка «Выставить лот» вверху StockScreen.
- Открывает форму:
  * Селект артефакта (из `/api/artefacts/me` — мои инвентарь).
  * `quantity`.
  * `price_oxsarit`.
- POST `/api/exchange/lots`.

### C. Premium / Ban — записи в simplifications

Зафиксировать `premiumLot` и `ban` как scope-out для MVP origin.
Когда понадобится admin-features — отдельный план.
