## Чек-лист ревизии (применяется к каждому экрану)

1. **Все ли поля показаны** — `{lang}`/`{var}` блоки в `.tpl`.
2. **Все ли действия работают** — `setPostAction` / `setGetAction`.
3. **Все ли валидации применяются** — Permission, лимиты, race-conditions.
4. **Идентичны ли формулы** — стоимость / время / опыт / бонусы.
5. **Все ли побочные эффекты происходят** — logs, messages, users, planets.
6. **Все ли особые ветки кода** — луна/планета, рейд/ACS, alien/PvP.

## Легенда

- ✅ — паритет полный.
- ⚠ — частичное расхождение, не блокирует gameplay (детали — в строке).
- ❌ — критическое расхождение / отсутствующая функциональность.
- ❓ — не проверено детально.

**Приоритет починки**: P1 — это критический gameplay (без retreat игрок

без StarGate — нет alt-механики перемещения).

**Рекомендация**: оформить как отдельный план **72.1.3** с подзадачами
по каждому отсутствующему action.

## #2 `/battlestats` — Battlestats.class.php

**Размер**: 343 vs 270 (близко). 1 action `showBattles` с фильтрами.

**Найденные расхождения**:

Legacy `showBattles` принимает 12 параметров фильтрации:
`date_min, date_max, user_filter, alliance_filter, show_no_destroyed,
show_aliens, new_moon, show_drawn, sort_field, sort_order, page,
moon_battle`.

| Фильтр | UI у нас | Backend у нас |
|---|---|---|
| `date_min/max` | ✅ показано | ❌ не передаётся |
| `user_filter` | ✅ показано | ❌ не передаётся |
| `alliance_filter` | ✅ показано | ❌ не передаётся |
| `show_drawn` | ✅ checkbox | ❌ не передаётся |
| `show_aliens` | ✅ checkbox | ❌ не передаётся |
| `show_no_destroyed` | ❌ нет | ❌ нет |
| `new_moon` | ❌ нет | ❌ нет |
| `moon_battle` | ❌ нет | ❌ нет |
| `sort_field/order` | ❌ нет | ❌ нет |
| `page` (пагинация) | ⚠ заменено на cursor | ⚠ cursor |

Backend `ListMine` (`internal/battlereport/handler.go:55-75`) принимает
**только cursor + limit**. WHERE-условие hardcoded:
`(attacker_user_id = $1 OR defender_user_id = $1) AND is_simulation = false`.

**Вывод**: ❌ Все фильтры в UI **фейковые** — отображаются, но в API не
передаются. При вводе значения в `userFilter` ничего не фильтруется.

**Приоритет**: P2 — gameplay не блокируется, но нарушены ожидания
пользователя из legacy. Чинить через: расширение API endpoint
`/api/users/me/battles` параметрами + актуализация `fetchMyBattles`
на фронте + протяжка state в queryFn.

выборочные spot-check'и) для **выявления класса пропусков**, далее
конкретные починки оформляются как подпланы 72.1.3, 72.1.4 и т.д.
