## 1. Контекст

Legacy [`Stock.class.php`](../../projects/game-legacy-php/src/game/page/Stock.class.php)
имеет:

1. **`premiumLot($id)`** (строки 59-109) — любой игрок помечает
   чужой (или свой) лот featured за credit:
   - cost = max(`EXCH_PREMIUM_MIN_COST=10`, lot.price × `EXCH_PREMIUM_PERCENT/100=0.5%`).
   - featured_date = now() → лот сверху списка на `EXCH_PREMIUM_LOT_EXPIRY_TIME=2ч`.
   - Featured-лот: топливо ×0.5 (`EXCH_PREMIUM_LOT_FUEL_COST_MULT`),
     время полёта ×0.5 (`EXCH_PREMIUM_LOT_FLY_TIME_MULT`).
   - max featured = `EXCH_PREMIUM_LIST_MAX_SIZE=5`.
   - AutoMsg `MSG_CREDIT_EXCHANGE_PREMIUM_LOT` в folder=8.
   - NS::isFirstRun дедуп.

2. **`ban($id)`** (строки 111-168) — **admin-only**:
   - send back лот (`Exchange::sendBackLot`).
   - status='banned'.
   - `EVENT_EXCH_BAN` для брокера на `EXCH_BAN_TIME` секунд.
   - удаляет EVENT_EXCH_EXPIRE для этого лота.
   - проверка прав в UI: `if(isAdmin())` (Stock.class.php:496); в
     legacy action нет проверки → у нас закроем дыру (admin-only
     middleware).

Origin отсутствует: ни Premium, ни Ban в backend и UI.

## 2. Что нужно

### Backend

#### Миграция `0088_exchange_premium_ban.sql`
```sql
ALTER TABLE exchange_lots
    ADD COLUMN IF NOT EXISTS featured_at timestamptz,
    ADD COLUMN IF NOT EXISTS banned_at   timestamptz;

-- Partial-index для list query (featured-сначала, expiry 2 часа).
CREATE INDEX IF NOT EXISTS exchange_lots_featured_idx
    ON exchange_lots (featured_at DESC NULLS LAST)
    WHERE banned_at IS NULL AND featured_at IS NOT NULL;
```

#### `internal/exchange/service.go`
- `PromoteLot(ctx, userID, lotID) → CreditCharge`:
  - читает lot (FOR UPDATE), проверяет status active + не banned.
  - cost = max(10, lot.price_oxsarit × 0.005).
  - проверка credit ≥ cost, списать.
  - проверка `COUNT(*) FROM exchange_lots WHERE featured_at > now() - 2h AND banned_at IS NULL ≤ 5`.
  - UPDATE featured_at = now().
  - AutoMsg `creditExchangePremium` в folder=8 (FolderCredit).
  - возвращает CreditCharge { lot_id, credit_debit }.
- `BanLot(ctx, adminUserID, lotID)`:
  - проверка `users.role = 'admin'` для adminUserID.
  - читает lot FOR UPDATE.
  - status='banned', banned_at=now().
  - возвращает escrow продавцу (как Cancel).
  - удаляет соответствующий KindExchangeExpire event.
  - возвращает SuccessResponse.
- `ListLots`: ORDER BY featured (active 2h) DESC, created_at DESC.
- `Buy/Cancel/CreateLot`: фильтр `banned_at IS NULL`.

#### Endpoints
- `POST /api/exchange/lots/{lotID}/premium` — Promote.
- `POST /api/exchange/lots/{lotID}/ban` — Ban (admin-only middleware).
