## 1. Контекст

Legacy [`Market.class.php::Credit_ex(metal, silicon, hydrogen)`](../../projects/game-legacy-php/src/game/page/Market.class.php#L192)
покупает все три ресурса одним запросом. Specifics legacy:
- `commission = 0` (override `$this->comis = 0` строка 194).
- `storage_metal/silicon/hydrogen = UNLIMIT` (capacity-cap не проверяется).
- AutoMsg `MSG_CREDIT_MARKET` отправляется агрегатно.
- Формула:
  ```
  credit_per_res = ceil(amount × curs_credit / curs_res)
  total = sum(credit_per_res)
  ```

72.1.21 закрыл single-resource exchange. 72.1.28 закрывает multi.

## 2. Что нужно

### Backend

- `Service.ExchangeCreditMulti(userID, planetID, metal, silicon, hydrogen)`
  → `CreditExchangeMultiResult`.
- Транзакция: SELECT planet FOR UPDATE, SELECT user.credit FOR UPDATE.
- Cost = ceil(metal/100 × resCost.metal) + ceil(silicon/50 × resCost.silicon)
  + ceil(hydrogen/25 × resCost.hydrogen).
- AutoMsg `creditMarketPurchaseMulti` в folder=8 (legacy MSG_CREDIT_MARKET).
- Endpoint: `POST /api/planets/{id}/market/credit-multi`.

### Frontend

- `api/market.ts`: `exchangeCreditMulti`, `multiCreditCost` (preview).
- `ResourceMarketScreen`: заменить single-resource credit-form на 3
  input'а (M/Si/H) + общий total + 1 кнопка submit (legacy UI 1:1).

### i18n

- `market.creditTotalLabel`, `market.creditLastExchangeMulti`.
- `autoMessages.creditMarketPurchaseMulti.title/body`.
