## 1. Контекст

Legacy [`Search.class.php`](../../projects/game-legacy-php/src/game/page/Search.class.php)
поддерживает поиск по players/planets/alliances. По чек-листу 6 пунктов
обнаружены 3 расхождения с origin:

1. **player_search поля**: legacy выводит `username, points, last_seen
   (useractivity), planetname, galaxy:system:position, ally_tag, ban`.
   Origin: только `username, points, alliance_tag`. Нет: last_seen,
   home planet name+coords, ban-флаг.

2. **planet_search markers**: legacy для планеты добавляет суффикс
   `(HP)` если planet_id == user.hp, или `(MOON)` если ismoon=1.
   Origin не различает.

3. **ORDER BY**: legacy сортирует
   `length(username) - length(query) + 2 ASC, username ASC` — приоритет
   ближайшим по длине совпадениям (legacy `prepareForSearch` добавляет
   `%query%` поэтому −2 = 2 wildcard'а). Origin сортирует players по
   `points DESC` (другая семантика — топовые сначала).

## 2. Что нужно

### Backend (`internal/search/handler.go`)

#### `playerResult` DTO
```go
type playerResult struct {
    UserID       string  `json:"user_id"`
    Username     string  `json:"username"`
    AllianceTag  *string `json:"alliance_tag,omitempty"`
    Points       float64 `json:"points"`
    LastSeen     *string `json:"last_seen,omitempty"`     // RFC3339
    HomePlanet   *string `json:"home_planet,omitempty"`   // имя главной
    HomeGalaxy   *int    `json:"home_galaxy,omitempty"`
    HomeSystem   *int    `json:"home_system,omitempty"`
    HomePosition *int    `json:"home_position,omitempty"`
    Banned       bool    `json:"banned,omitempty"`
}
```

SQL: добавить JOIN на planets для home (по аналогии с
score.Top — `LEFT JOIN LATERAL ... ORDER BY created_at ASC LIMIT 1`),
+ COALESCE(banned-флаг). У нас бан-таблица — TBD: проверю есть ли
`bans` или флаг в users.

ORDER BY:
```sql
ORDER BY (LENGTH(u.username::text) - LENGTH($2::text)) ASC, u.username ASC
```
($2 — длина исходного query без wildcards).

#### `planetResult` DTO + special markers
```go
type planetResult struct {
    PlanetID  string `json:"planet_id"`
    Name      string `json:"name"`
    Galaxy    int    `json:"galaxy"`
    System    int    `json:"system"`
    Position  int    `json:"position"`
    Owner     string `json:"owner"`
    IsMoon    bool   `json:"is_moon"`
    IsHome    bool   `json:"is_home"` // planet.id == user.home_planet_id
}
```

SQL дополнительно:
```sql
SELECT p.id, p.name, p.galaxy, p.system, p.position,
       COALESCE(u.username::text, '') AS owner,
       p.is_moon,
       (p.id = u.home_planet_id) AS is_home
FROM planets p ...
```
TBD: есть ли `users.home_planet_id`? Уточнить в БД.

ORDER BY: тот же `LENGTH-LENGTH ASC, name ASC`.
