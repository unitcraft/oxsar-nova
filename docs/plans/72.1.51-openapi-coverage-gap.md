# План 72.1.51 — OpenAPI coverage gap (closing 72.1.50 backlog)

> Подплан плана [72.1.2](72.1.2-screen-audit.md) / финализация
> [72.1.50](72.1.50-screen-audit-followups.md) ч.4 (idempotency
> rollout). После rollout'а оставался **документационный** gap:
> ~24 мутирующих endpoint'а nova не были описаны в openapi.yaml,
> хотя backend и frontend работали с ними. Этот план — закрытие.

## 1. Контекст

Verify-handsweep части 4A в плане 72.1.50 показал:

> «Документационный gap (вне scope A)**: 8 route'ов
> (`buildings/queue/{taskId}/vip`, `buildings/{unitId}/demolish`, ...)
> **не описаны в openapi вовсе**. Обёртка middleware на бэке
> применена; добавление этих 10 route'ов в спеку — самостоятельный
> план «openapi coverage gap».»

И части 4B:

> «Документационный gap: 16 endpoint'ов (load/unload, promote-to-acs,
> ACS invite/accept, market credit/lots series, planets/order,
> deletion/cancel, alien/holding/pay). Backend обёртка применена;
> добавление в спеку — самостоятельный план.»

Всего 24 endpoint'а. Этот план описывает их.

## 2. Цель

Закрыть документационный gap: каждый из 24 endpoint'ов получает
полное описание в [`openapi.yaml`](../../projects/game-nova/api/openapi.yaml)
с:
- path/method
- parameters (path + IdempotencyKey header где применимо)
- requestBody (если есть)
- responses (200/201/204 + 400/403/404 + 409 IdempotencyConflict)
- summary + description (включая legacy-pointer и ссылку на план)

## 3. Декомпозиция

### A. Buildings/Research/Shipyard/Repair — queue management (10)

- `POST /planets/{id}/buildings/queue/{taskId}/vip` — VIP-instant старт
- `POST /planets/{id}/buildings/{unitId}/demolish` — DEMOLISH_NOW
- `POST /planets/{id}/buildings/{unitId}/pack` — PackConstruction
- `POST /planets/{id}/research/{unitId}/pack` — PackResearch
- `DELETE /research/{queueId}` — abort с 95%/100% refund
- `POST /research/{queueId}/vip` — VIP-instant
- `DELETE /planets/{id}/shipyard/{queueId}` — abort
- `POST /planets/{id}/shipyard/{queueId}/vip` — VIP-instant
- `DELETE /planets/{id}/repair/queue/{queueId}` — cancel
- `POST /planets/{id}/repair/queue/{queueId}/vip` — VIP-instant

### B. Settings + Planets (2)

- `POST /me/deletion/cancel` — отмена pending удаления (план 72.1.30)
- `PATCH /planets/order` — переупорядочить planets

### C. Fleet + ACS (5)

- `POST /fleet/{id}/load` — загрузить ресурсы в HOLDING-флот
- `POST /fleet/{id}/unload` — выгрузить
- `POST /fleet/{id}/promote-to-acs` — single → ACS-formation
- `POST /acs/{groupId}/invite` — пригласить в formation
- `POST /acs/invitations/{groupId}/accept` — принять инвайт

(Заодно добавлен `GET /acs/invitations` — список pending для текущего юзера, иначе фронт не знает откуда брать список.)

### D. Alien holding (1)

- `POST /alien/holding/{event_id}/pay` — выкуп HOLDING за оксары

### E. Market series (8)

- `POST /planets/{id}/market/credit` — single-resource → credit
- `POST /planets/{id}/market/credit-multi` — multi-resource → credit
- `GET + POST /market/lots` — список + создание лота
- `DELETE + POST /market/lots/{id}` — снять / accept
- `GET /market/fleet-lots` — список fleet-лотов
- `POST /planets/{id}/market/fleet-lots` — создать fleet-лот
- `POST /market/fleet-lots/{lotId}/accept` — accept fleet-лот
- `DELETE /market/fleet-lots/{lotId}` — снять fleet-лот

### F. Минимальные market-схемы (4)

В components.schemas добавлены:
- `MarketLot` — DTO лота resource-биржи
- `MarketLotInput` — body для create
- `MarketFleetLot` — fleet-вариант (allOf + src_planet_id, speed_percent)
- `MarketFleetLotInput` — body для fleet-create

Минимально-достаточные поля для openapi-валидаторов; backend может
возвращать больше (origin реализация эволюционирует — спека покрывает
обязательный контракт).

## 4. Что **не делается**

- Не правим backend (коду уже работает).
- Не правим frontend (фронт уже шлёт правильные запросы).
- Не добавляем тесты (это документация, не код).
- Не покрываем GET-only endpoint'ы вне gap-списка (разведка
  показала что почти все GET уже описаны).
- Не вводим новые middleware/политики безопасности.
