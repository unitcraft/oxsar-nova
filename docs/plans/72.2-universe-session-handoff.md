# План 72.2 (ремастер): Universe session handoff (cross-domain SSO)

**Дата**: 2026-04-29
**Статус**: ✅ Завершён
**Зависимости**: план 72.1 (uni03 Classic, identity-юзеры, multiverse в dev).
**Связанные планы**: 36 Ф.12 (identity JWKS-only), 53 (admin-консоль),
72 (origin-фронт).

## Цель

Реализовать **бесшовный переход** с публичного портала
(`oxsar-nova.ru`, `localhost:5174` в dev) в игровые вселенные
(`uniNN.oxsar-nova.ru`, `localhost:5173/5175/5176`) **без повторного
ввода пароля** и **без копирования токена через URL plaintext**.

После реализации: юзер на портале → клик «Играть» в карточке
вселенной → автоматически залогинен в нужном game-фронте.
LoginScreen на origin/nova-фронтах **полностью удаляются** — игровые
домены принимают только handoff-код. Это снижает фишинг-риск
(никаких форм пароля на game-доменах) и упрощает UX.

## Архитектура потока

```
Portal-frontend                Portal-backend           Identity-service           Origin/Nova-frontend
    │                                  │                         │                              │
    │  click «Играть» (uni03)          │                         │                              │
    │─────────────────────────────────▶│                         │                              │
    │                                  │  POST /auth/universe-token                            │
    │                                  │  (Bearer: portal-access-token)                        │
    │                                  │  body: {universe_id:"uni03"}                          │
    │                                  │────────────────────────▶│                              │
    │                                  │                         │  generate one-time code     │
    │                                  │                         │  store Redis: code→{user,   │
    │                                  │                         │      universe} TTL 30s,     │
    │                                  │                         │      single-use             │
    │                                  │ ◀───────────────────────│  {code, expires_in}          │
    │ ◀────────────────────────────────│  {redirect_url}                                        │
    │                                                                                            │
    │  window.location.assign(redirect_url + '?code=XXX')                                        │
    │────────────────────────────────────────────────────────────────────────────────────────▶│
    │                                                                                            │
    │                                                            POST /auth/token/exchange       │
    │                                                            body: {code: XXX}               │
    │                                                            ◀───────────────────────────────│
    │                                                            │   verify code (GETDEL atomic)│
    │                                                            │   issue fresh access+refresh │
    │                                                            ────────────────────────────────▶│
    │                                                                                            │
    │                                                                            store tokens   │
    │                                                                            redirect /     │
```

## Безопасность (прод-чеклист)

- ✅ Handoff-код одноразовый (Redis GETDEL — atomic).
- ✅ TTL 30s — короткое окно для перехвата.
- ✅ Code привязан к {user_id, universe_id} — нельзя использовать
  на другой вселенной.
- ✅ Идентичности подписываются JWKS через identity-service.
- ✅ HTTPS-only в проде (dev — accept http://localhost).
- ✅ Audit log: каждое issue + exchange логируется.
- ✅ Rate-limit на `/auth/universe-token` (per user, per IP).
- ✅ Error-responses без раскрытия внутренней структуры
  ("invalid_handoff" без деталей).

## План реализации (8 шагов)

### Шаг 1. Audit identity-service — что есть

Проверить состояние:
- `POST /auth/universe-token` — существует ли, какой контракт.
- `POST /auth/token/exchange` — то же.
- Audit log таблица (`audit_handoff` или похожая).
- Rate-limit middleware на этих эндпоинтах.

### Шаг 2. Portal-backend: `POST /api/universes/{id}/session`

Новый endpoint:
- Bearer-auth обязателен (existing identity-JWKS middleware).
- Validates: universe в registry; status=active.
- Прокси в identity: `POST /auth/universe-token` с
  переданным Bearer-токеном и body `{universe_id}`.
- Returns:
  ```json
  {
    "redirect_url": "http://localhost:5176/auth/handoff?code=XXX",
    "expires_in": 30
  }
  ```
  где base URL берётся из `universes.yaml` (dev_url в dev, или
  построенный prod-URL).

### Шаг 3. Portal-frontend: переключить кнопку «Играть»

`HomePage.tsx`:
- Заменить `<a href={universeURL}>` на `<button onClick={handleJoin}>`.
- `handleJoin(universe)`: POST к `/api/universes/{id}/session` →
  получить redirect_url → `window.location.assign(redirect_url)`.
- Disabled при `user === null` + tooltip «Войдите чтобы играть».
- Обработка ошибок: 401 → редирект на /login; 503 → toast.

### Шаг 4. Origin-фронт: auth-gate + удаление LoginScreen

- Router: если `accessToken === null` и `pathname !== '/auth/handoff'`
  → `window.location.assign(PORTAL_URL)` (на портал, не на свой /login).
- LoginPlaceholder.tsx — **удалить файл** + ссылку из router.
- HandoffPage обработать ошибки exchange (TTL expired, invalid code) →
  redirect на портал с error-параметром в URL.
- PORTAL_URL — env-переменная `VITE_PORTAL_URL` (dev:
  http://localhost:5174, prod: https://oxsar-nova.ru).

### Шаг 5. Nova-фронт: то же

- Router auth-gate (как в origin).
- LoginScreen удалить.
- HandoffPage уже есть — проверить.

### Шаг 6. Refresh-токен flow

Проверить — реализован ли refresh-flow в origin/nova auth-store +
api/client. Если 401 на API-запрос → попытаться `POST
/auth/refresh` с refresh-токеном → retry оригинальный запрос. Если
refresh failed → редирект на портал.

### Шаг 7. Тесты

- identity-service: `/auth/universe-token` + `/auth/token/exchange` —
  TTL, single-use, invalid code, fresh tokens.
- portal-backend: `/api/universes/{id}/session` — proxy, validates,
  401, 404.
- HandoffPage: error handling.

### Шаг 8. Документация

- ADR-0010: «Universe Session Handoff: single-use code flow для
  cross-domain SSO».
- Обновить `docs/ops/admin-access.md` если нужно (handoff не
  применяется к admin-консоли — там собственный BFF cookie).
- Обновить `docs/plans/72.2` с финальным статусом ✅ + ссылками на
  коммиты.

## Edge cases

- Юзер не залогинен на портале → 401, кнопка disabled.
- Universe не существует / not active → 404 / 403.
- Identity-service недоступен → 503 + toast.
- Code expired (>30s) → origin показывает «session expired»,
  редирект на portal.
- Двойной клик «Играть» → idempotency-key в request-headers.
- Backward-compat: старый bookmark `https://uni03.oxsar-nova.ru`
  без кода — редиректит на portal (auth-gate сработает).

## Что НЕ делается в этом плане

- Multi-tab handoff (юзер открыл портал в двух вкладках, кликнул
  «Играть» в одной — другая видит изменения). Пока не критично.
- Fingerprinting / device-binding handoff-кода. Если IP/UA-binding
  понадобится — отдельная задача.
- SSO между admin.oxsar-nova.ru и game-доменами. Admin использует
  собственный BFF cookie-flow; пересечения не предусмотрено.
- Migration legacy bookmarks с прямыми game-URL. Если у юзера
  bookmark на `uni03.oxsar-nova.ru` без кода — он попадёт на
  portal, что приемлемо.

## Что сделано

### identity-service
- `POST /auth/universe-token` уже существовал (handler.go:265-294, TTL
  30s, Redis). Добавлен **rate-limit 25/мин/IP** (раньше отсутствовал —
  audit пометил как critical).
- `POST /auth/token/exchange` уже существовал (handler.go:296-325,
  Redis GETDEL atomic, single-use). Добавлен **rate-limit 25/мин/IP**
  для защиты от brute-force перебора кодов.

### portal-backend
- Новый client `IdentityClient` (`internal/portalsvc/identity_client.go`)
  с методом `IssueHandoffToken`. Прокидывает Bearer юзера в identity
  `POST /auth/universe-token`. Sentinel ошибки `ErrIdentityUnavailable`,
  `ErrUnauthorized`.
- Новый handler `Handler.CreateUniverseSession`
  (`internal/portalsvc/handler.go`):
  - `POST /api/universes/{id}/session`.
  - Auth: existing `portalsvc.Middleware` (JWT через JWKS).
  - Validates: universe в registry, status=active.
  - Returns: `{redirect_url, universe_id, universe_name, expires_in}`.
  - HTTP-коды: 401 (не залогинен / битый токен), 404 (нет вселенной),
    400 (universe не active), 503 (identity недоступен).
- Подключение в `cmd/server/main.go` (под protected-router) +
  `IDENTITY_SERVICE_URL` env.
- Тесты `handler_session_test.go` — 5 кейсов (401/404/400/503/200).

### portal-frontend
- `api/portal.ts` — добавлен `universes.createSession(id)`.
- `pages/HomePage.tsx` — кнопка «Играть» теперь делает
  `POST /api/universes/{id}/session` → `window.location.assign(redirect_url)`.
  Disabled при не залогиненном юзере; обработка ошибок.

### origin-фронт
- Создан `features/auth/HandoffPage.tsx` — копия nova-логики, но в
  origin-store (`oxsar-origin-auth`).
- Создан `features/auth/AuthGate.tsx` — проверяет `accessToken`,
  при null редиректит на `VITE_PORTAL_URL`.
- `router.tsx`: `/auth/handoff` обрабатывается ДО AuthGate; всё
  остальное — внутри AuthGate.
- **Удалён** `features/login/LoginPlaceholder.tsx` (вместе с папкой).

### nova-фронт
- `App.tsx`: убран импорт `LoginScreen`, при `!token` происходит
  `window.location.replace(PORTAL_URL)`.
- **Удалён** `features/auth/LoginScreen.tsx`.
- `vite.config.ts` warmup обновлён (LoginScreen → HandoffPage).
- Исправлен баг контракта: HandoffPage слал `handoff_token`, identity
  ждёт `code` — переименовано.

### Refresh-flow
- Реализован в обоих `api/client.ts` (nova и origin):
  - На 401 → `tryRefresh()` → retry оригинального запроса.
  - Race-protection через глобальный `refreshInFlight` promise.
  - Если refresh failed → `logout()` → AuthGate видит null → редирект.

### Compose
- `IDENTITY_SERVICE_URL: http://identity-service:9000` для portal-backend.
- `VITE_PORTAL_URL: http://localhost:5174` для всех трёх game-фронтов.

### Документация
- ADR-0012: universe session handoff (rationale, альтернативы,
  безопасность, refresh-flow).

## End-to-end проверка

После реализации, под `admin1`/`DevPass123`:

```
$ curl -X POST .../api/universes/uni03/session -H "Authorization: Bearer $TOKEN"
→ 200 + redirect_url + handoff-code

$ curl -X POST .../auth/token/exchange -d '{"code":"<X>"}'
→ 200 + access+refresh с правильными rolesами

$ curl -X POST .../auth/token/exchange -d '{"code":"<X>"}'  # повтор
→ 401 (single-use ✓)

$ curl -X POST .../api/universes/uni03/session  # без Authorization
→ 401 ✓

$ curl -X POST .../api/universes/uni99/session -H "Authorization: Bearer $TOKEN"
→ 404 ✓
```
