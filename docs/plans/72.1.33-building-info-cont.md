## 1. Контекст

72.1.23 закрыла icon (catalog/icon в BuildingInfoScreen). Хвосты:
- **demolish** — legacy `BuildingInfo::DEMOLISH_NOW` ставит здание
  на снос: cost `(1/factor) × cost_at_current_level`, duration =
  build_dur × 0.5, event Kind=2 (KindDemolishConstruction).
  Обработчик `HandleDemolishConstruction` уже реализован (план 65
  Ф.1, есть golden-тесты).
- **pack-artefacts** — legacy `PackConstruction`/`PackResearch`
  упаковывают здание/исследование в артефакт (требует
  packing-артефакта на планете, проверка requirements).

## 2. Что сделано

### Backend (`internal/building`)

**`service.go::EnqueueDemolish(ctx, userID, planetID, unitID)`**:
1. lookup unit — есть ли spec и `spec.Demolish > 0`.
2. ownership-check (`p.UserID != userID` → ErrPlanetOwnership).
3. queue busy (одна задача на планету одновременно).
4. `curLevel > 0` (иначе ErrLevelZero).
5. cost = `(1 / spec.Demolish) × CostForLevel(spec.CostBase, factor, curLevel)`.
6. resource check + UPDATE planets - cost + res_log reason='demolish'.
7. duration = `BuildDuration / 2` (стандарт OGame).
8. INSERT в `construction_queue` (unit_type='building', target_level=curLevel-1).
9. INSERT в `events` (Kind=2 KindDemolishConstruction) с payload
   `{queue_id, unit_id, target_level}`.

**`ErrLevelZero`** — новая sentinel-ошибка.

**`handler.go::Demolish`**: POST
`/api/planets/{id}/buildings/{unitId}/demolish` → 201 + QueueItem.
Маппинг ошибок: ErrQueueBusy → 409, ErrNotEnoughRes/ErrLevelZero/
ErrUnknownUnit → 400, ErrPlanetOwnership → 403.

### Frontend (`features/info/BuildingInfoScreen`)

- `useResolvedPlanet()` для planetId.
- `fetchBuildingsOverview(planetId)` для определения `curLevel = levels[entry.id]`.
- Кнопка «Снести сейчас» появляется при `curLevel > 0`:
  - Показывает «Текущий уровень: N → N-1».
  - Confirm-prompt перед мутацией.
  - При успехе invalidate `buildingsOverview` + `planet`.
  - Ошибка в красном через `false`-класс.

### `api/buildings.ts`

- `demolishBuilding(planetId, unitId)` → POST .../demolish.

### i18n

- `buildinginfo` секция (новая): `demolish`, `demolishNow`,
  `demolishConfirm` (с `{{building}}` + `{{level}}`), `currentLevel`.

## 3. Часть 2 — pack-artefacts (закрыта)

### Catalog (configs/artefacts.yml)

4 новых артефакта (legacy ID 321-324):
- **packing_building (323)** — marker, активируется через
  PackBuilding endpoint.
- **packing_research (324)** — marker, через PackResearch.
- **packed_building (321)** — marker с payload, активируется через
  обычный Activate (special-case в сервисе).
- **packed_research (322)** — marker с payload, через Activate.

Effect.Type = `pack_building` / `pack_research` / `packed_building`
/ `packed_research` — special-case, computeChanges возвращает nil,
factor-flow не применяется.

### Backend (internal/artefact)

**`pack.go::PackBuilding(planetID, buildingUnitID)`**:
1. Ownership check планеты.
2. SELECT buildings.level (curLevel > 0).
3. SELECT artefacts_user WHERE state=held AND unit_id=323 AND
   planet_id LIMIT 1.
4. UPDATE buildings.level -= 1 (если стал 0 → DEC used_fields).
5. UPDATE artefacts_user.state='consumed' (packing).
6. INSERT packed-building артефакт с payload
   `{construction_id: unitID, level: 1}`, state='held'.

**`pack.go::PackResearch`** — аналог для research-таблицы (без
planet_id привязки, по user_id).

**`activate_packed.go::ActivatePackedBuilding(artefactID, planetID)`**:
1. SELECT артефакт + payload (parse `{construction_id, level}`).
2. Проверка ownership planetID.
3. UPSERT buildings: если строки нет — INSERT level=pl.Level
   + INC used_fields; иначе UPDATE level += pl.Level.
4. UPDATE artefacts_user.state='consumed'.

**`ActivatePackedResearch`** — аналог для research.

**`service.go::Activate`** расширен: pre-check unit_id артефакта.
Для packed_building/packed_research → делегирует в Activate*Packed*.
Для packing_building/packing_research → ошибка «use /pack endpoint».
Остальные типы → обычный flow.

### HTTP

- `POST /api/planets/{id}/buildings/{unitId}/pack` → PackBuilding.
- `POST /api/planets/{id}/research/{unitId}/pack` → PackResearch.
- `POST /api/artefacts/{id}/activate` (расширен на packed-types).

### Frontend

- `api/buildings.ts`: packBuilding/packResearch функции.
- `BuildingInfoScreen`: кнопка «📦 Упаковать здание» в demolish-блоке
  при curLevel > 0. Mutation invalidate
  buildingsOverview/planet/artefacts.
- `ResearchScreen`: 📦-кнопка для каждого исследования с lvl > 0.
  Mutation invalidate research/artefacts.
- i18n ru/en: `buildinginfo.packBuilding`, `packResearch`,
  `packConfirm`.
