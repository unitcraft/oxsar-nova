# План 72.1.31 — Battlestats BA-cont: INSERT has_aliens/moon_created/is_moon

**Дата**: 2026-05-01
**Статус**: ✅ Завершён

**Подплан** [плана 72.1](72.1-post-remaster-stabilization.md), задача
из §20.12 / [audit 72.1.2](72.1.2-screen-audit.md) #2 («/battlestats»),
правило 1:1. Закрывает хвост 72.1.10.

## 1. Контекст

72.1.10 ввела фильтры [`new_moon`/`show_aliens`/`moon_battle`] в
`/api/users/me/battles` (handler `battlereport.List`), используют
WHERE по колонкам `battle_reports.has_aliens`/`moon_created`/
`is_moon`. Миграция [0085](../../projects/game-nova/migrations/0085_battle_reports_filters.sql)
добавила эти колонки с `DEFAULT false`, индексы `WHERE col=true`.

Но **INSERT-запросы не были обновлены** — `attack.go`, `acs_attack.go`
не заполняли флаги. Все записи имели false → фильтры `new_moon=true`
возвращали пустоту, `show_aliens=true` — пустоту.

72.1.10 пометила это «**НЕ закрыто (правило 1:1)** — отложено до
72.1.3 BA-fixes». 72.1.3 закрыта параллельной сессией (audit + 6/15
закрытий BA-* находок), теперь хвост можно закрыть.

## 2. Что сделано

### Backend

#### `fleet/attack.go`

- `tryCreateMoon` сигнатура `(bool, error)` — возвращает `created=true`
  только если в этой битве была реально создана новая луна. Раньше
  возвращал только `error` (creator/non-creator неотличимы).
- `finalizeAttack` принимает 3 новых параметра: `isMoon, hasAliens,
  moonCreated bool`. INSERT в `battle_reports` теперь записывает
  все 3 флага.
- `hasAliens = holdingDef != nil` (alien holding на планете во
  время атаки). Без боя (early return при пустой планете без
  alien) → `false, false`.

#### `fleet/acs_attack.go`

- Аналогично заполняем `moonCreated` через `tryCreateMoon`. ACS
  никогда не имеет alien holding (`LoadHoldingDefender` вызывается
  только в single-attack), поэтому `has_aliens=false` всегда.
- INSERT добавил +3 параметра.

#### `simulator/handler.go`

- Симуляции имеют `planet_id=NULL`, поле `is_simulation=true`.
  INSERT теперь явно задаёт `false, false, false` для 3 флагов
  (раньше полагался на DEFAULT — теперь explicit, для консистентности
  при возможном будущем lifting DEFAULT).

#### `battlestats/handler.go` (admin-вариант `/api/battlestats`)

- Добавлены query-параметры `has_aliens`, `moon_created`, `is_moon`
  (trinary: "1"/"true" → WHERE col=true; "0"/"false" → WHERE col=false;
  "" → нет фильтра).
- SELECT возвращает 3 поля в DTO `battleRow` (для будущих admin-страниц
  или CSV-экспорта).
- `/api/users/me/battles` (handler `battlereport.List`) **уже имел**
  фильтры по этим колонкам — фикс INSERT просто заставил их работать.

### i18n cleanup (попутный)

Тест `i18n_consistency_test.go` ловил yaml mapping-key collision:
- `market.creditAmountLabel` дублировался в обоих ru.yml и en.yml
  (1236+1263 vs 1239+1266).
- `statistics.bsMoonBattle` дублировался (2947+2954).

Оставлены первые, удалены вторые.
