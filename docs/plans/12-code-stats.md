# План 12 — статистика по коду проекта

Сбор метрик по кодовой базе oxsar-nova. Цель — материал для статьи о создании
игры ([docs/project-creation.txt](../project-creation.txt),
[docs/ui-dev-log.md](../ui-dev-log.md)) и понимание техдолга/прогресса.

## Что считать

### Размер и структура

- LOC по языкам (Go / TS / SQL / YAML) и по доменам
  (`backend/internal/*`, `frontend/src/features/*`) — где сосредоточена масса.
- Кол-во пакетов/фич, средний размер файла, топ самых больших файлов
  (кандидаты на распил).
- Соотношение код / тесты по доменам — где тестов мало
  (порог 85% для `battle`, `economy`, `event`; 70% для остального).

### Прогресс портирования

- Матрица готовности из [docs/status.md](../status.md) в цифрах:
  done / in-progress / todo.
- Завершённые планы в `docs/plans/` vs оставшиеся.
- Сравнение размера с легаси: LOC `d:\Sources\oxsar2` (PHP/Yii)
  vs oxsar-nova (Go+TS) — «стоимость» переписывания.

### Качество

- Технический долг: кол-во записей в
  [docs/simplifications.md](../simplifications.md), открытые vs закрытые.
- TODO/FIXME в коде по доменам.
- Покрытие тестами (`go test -cover`, vitest coverage) по модулям.
- Golden-файлы боя в `testdata/battle/` — сколько сценариев зафиксировано.

### История разработки

- Коммиты по неделям/месяцам, распределение по типам (feat/fix/refactor/docs)
  из conventional commits.
- Топ изменяемых файлов (`git log --name-only` → hotspots,
  кандидаты на рефакторинг).
- Темп: коммитов в день, строк добавлено/удалено по месяцам.

## Формат результата

Одноразовый отчёт в `docs/code-stats.md` (или `.tsv` для сырых данных)
+ при желании скрипт `backend/cmd/tools/codestats` для повторного прогона.

## Самое ценное для статьи

График коммитов по времени + LOC по доменам + таймлайн «план → PR → коммит».
Визуально показывает, как проект рос.
