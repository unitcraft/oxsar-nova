# План 72.1.34 — Simulator cont: 3-channel tech + lost_points + part-by-part остатки

**Дата**: 2026-05-01
**Статус**: ✅ Завершён (части A+B+C+D)

**Подплан** [плана 72.1](72.1-post-remaster-stabilization.md), задача
из §20.12 / [audit 72.1.2](72.1.2-screen-audit.md) #4 (`/simulator`),
правило 1:1.

## 1. Контекст

Аудит-сравнение `Simulator.class.php` (768 LOC) vs origin
`SimulatorScreen.tsx` (702 LOC) выявил 5 функциональных разрывов:

1. **3-channel tech** (laser/ion/plasma) — backend поддерживает
   (`battle/types.go:66-68`), но фронт-форма не собирает (поля
   уходили как 0).
2. **Lost points / lost units в SimStats** — `SideResult.LostPoints`
   считается для каждой стороны, но `MultiRun` не агрегировал в
   SimStats. UI Report-блока их не показывал.
3. **target_buildingid + building_level** — legacy 171-196: селект
   «Уничтожить здание» (для гравитона/межпланетной защиты) +
   moonchance ввод вручную. У origin отсутствует целиком.
4. **Боевые артефакты** (`ARTEFACT_EFFECT_TYPE_BATTLE`) — legacy
   124-163: 6 артефактов с эффектами на side (бонус gun, shield,
   shell, ballistics, masking). Origin-форма не собирает.
5. **Рециклеры / трансплантеры** — legacy 208-211: расчёт
   нужного количества recycler-кораблей и transplanter-юнитов
   для поднятия debris после боя.

## 2. Часть A — закрыта (commit 135fd5e1f5)

### Backend

- `battle/types.go::SimStats` +поля:
  - `AttackerLostPoints` / `DefenderLostPoints` (среднее
    `Σ qty_lost × cost / 1000 × 2` по num_sim).
  - `AttackerLostUnits` / `DefenderLostUnits` (среднее
    `Σ qty_lost`).
- `battle/simstats.go::MultiRun` суммирует `s.LostPoints` и
  `s.LostUnits` из каждого SideResult и делит на num_sim.

### Frontend

- `SimulatorScreen::TechSet` +поля `laser/ion/plasma` с дефолтом 0.
- Форма «Установки» — 3 новые колонки (Laser/Ion/Plasma) для
  атакующего и защитника между Armoring и Ballistics.
- `onSubmit` пробрасывает `aTech.laser/ion/plasma` и
  `dTech.laser/ion/plasma` в `SimSide.tech`.
- Report-блок: 2 новые строки «Очки -N» (lost_points) и «Юнитов
  потеряно» (lost_units) — рендер только если backend вернул эти
  поля (опциональные).

### `api/simulator.ts`

- `SimStats` +`attacker_lost_points`/`defender_lost_points`/
  `attacker_lost_units`/`defender_lost_units` (опциональные).
- `SimSideResult` +`lost_points`/`lost_units` (опциональные).

## 3. Часть B — target_buildingid (закрыта)

### Backend
- `battle/types.go::Input` +`TargetBuildingID`, `+TargetBuildingLevel`.
- `battle/types.go::Report` +`BuildingDestroyChance` (0-100%),
  `+TargetDestroyed` (≥50%).
- `simulator/handler.go` после MultiRun: считает
  `chance = min(100, ds_count × 100 / level)` где `ds_count` —
  выжившие deathstar (unit_id=42) у атакующих при `Winner='attackers'`.

### Frontend
- `api/simulator.ts::SimInput` +2 опциональных поля.
- `SimReport` +2 опциональных поля.
- `SimulatorScreen`:
  - `useState` `targetBuildingID` (default '0' = no target),
    `targetBuildingLevel` ('1').
  - Селект из `catalogByGroup('building')` рядом с numSim.
  - В `onSubmit` пробрасывает в `SimInput` если оба > 0.
  - Report-блок: новая строка «Шанс разрушения {chance}% [✓]»
    (✓ если target_destroyed).

## 4. Часть C — боевые артефакты (закрыта)

### Backend
- `battle/types.go::Side` +`BattleArtefactIDs []int`.
- `simulator/handler.go::applyBattleArtefacts`:
  - Резолвит `config.ArtefactSpec` по ID из catalog.
  - Аккумулирует bonus через `artefact.ComputeBattleModifier`.
  - Multiplier применяется к `u.Attack/Shield/Shell` каждого юнита.

### Frontend
- `api/simulator.ts::SimSide` +`battle_artefact_ids?: number[]`.
- `SimulatorScreen`:
  - `BATTLE_ARTEFACT_IDS = [316, 317, 318, 359, 360, 361]` (legacy IDs).
  - `expandArtefactStacks(stacks)`: stack count → массив повторов
    (3 stack `attack_power` = `[318, 318, 318]`).
  - `useState` `aBattleArts`, `dBattleArts` (Record<id, count>).
  - Таблица «Боевые артефакты» с stack-input 0-3 на каждую сторону.

## 5. Часть D — рециклеры/трансплантёры (закрыта)

Derived calc на фронте, бэк не меняется.
- Recycler: `ceil((debris_metal + debris_silicon) / 20_000)`.
- Ship transplantator: `ceil((debris_metal + debris_silicon) / 2_000_000)`.
- 2 строки в Report-блоке (только если debris > 0).
