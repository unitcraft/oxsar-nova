## 1. Контекст

72.1.24 закрыла hydrogen-cost 10H при просмотре чужой системы. Хвосты
по audit:
- **activity** — маркер последней активности игрока в системе
  (legacy `Galaxy.class.php:230-240`).
- **rocket** — иконка 🚀 «ракетная атака» если impulse_engine достаточен
  (`getRocketRange = impulse_engine × 8 - 1`).
- **monitor-link** — `canMonitorActivity` из `star_surveillance` +
  `hyperspace_tech` (legacy `getMonitorActivityRange`).
- **e_points** — отдельная колонка очков боевого опыта (legacy
  Galaxy.tpl показывает рядом с rank).
- **alliance_rank** — порядковый номер игрока внутри альянса по очкам
  (legacy `UserList::setFetchRank(true)`).

## 2. Что сделано

### Backend (`internal/galaxy`)

#### `repository.go`

- **`CellView` расширен**: `owner_e_points`, `owner_alliance_rank`.
- **`SystemView` расширен**: `can_monitor_activity` (видим активность
  чужих игроков), `in_missile_range` (можем послать ракету в эту систему).
- **`ReadSystem`** теперь принимает `viewerPlanetID, numSystems`.
  Без `viewerPlanetID` (backwards-compat) оба флага false.
- **Helpers**:
  - `systemDiffWrap(a, b, numSystems)` — legacy `NS::getSystemsDiff`
    с wrap-around (`min(|a-b|, NUM_SYSTEMS - max + min)`).
  - `roundHalfAwayFromZero(v)` — PHP-стиль round (не Go banker's).
- **SQL расширен**: SELECT теперь возвращает `u.e_points` +
  computed `alliance_rank = COUNT(WHERE alliance_id=u.alliance_id
  AND points>u.points)+1`.
- **Viewer-meta SQL**: один запрос к `planets` + `buildings (id=55
  star_surveillance)` + `research (id=19 hyper_tech, id=21
  impulse_engine)` для подсчёта range'ей.

#### `handler.go`

- `from_planet_id` теперь используется не только для hydrogen-cost
  (72.1.24), но и для viewer-метаданных. Прокидывается в
  `ReadSystem(..., viewerPlanetID, h.numSystems)`.

## 4. Ограничения

- Activity-маркер вычисляется на фронте на момент рендера (через
  `Date.now()`). При длительном открытом экране метка может «застрять»
  на старом значении — это совпадает с legacy, который рендерит метку
  один раз (без auto-refresh).
- `viewer_planet_id` обязателен для viewer-meta. Без него флаги
  останутся false, и UI не покажет иконки. Это backwards-compat —
  старые клиенты без `currentPlanetId` в store не сломаются.
