# План 72.1.11 — Messages: bulk-delete (DeleteAll legacy)

**Дата**: 2026-05-01
**Статус**: ✅ Завершён

Пояснение по объёму работы: backend `DELETE /api/messages` уже был
реализован раньше (`internal/message/handler.go::DeleteAll`,
service.go DeleteAll с `folder=0` = удалить все, `folder>0` = по
папке). UI этой функциональности не использовал, плюс per-row
delete-confirm ошибочно показывал «Delete all messages?» вместо
«Delete this message?».

Сделано:
- UI-кнопка «Удалить все» в шапке /msg/inbox (показывается только
  при list.length>0).
- mutation deleteAllMut → invalidate ['messages'].
- Per-row confirm: t('message','deleteOneConfirm') вместо
  ошибочного 'deleteAll'.
- i18n: deleteAllBtn, deleteAllConfirm, deleteOneConfirm (ru/en).

Не сделано (TBD, отдельный план если потребуется):
- Bulk delete для папки 'sent' — backend семантику не поддерживает
  (DeleteAll фильтрует по `to_user_id = uid`; для sent нужен
  `from_user_id = uid`).
- Folder navigation legacy (5 папок: inbox/sent/spy/battle/transfer).
  В origin folder=int хранится в DB, но в UI только 'inbox|sent'.

**Подплан** [плана 72.1](72.1-post-remaster-stabilization.md), задача
из §20.12 / [audit 72.1.2](72.1.2-screen-audit.md).

## 1. Контекст

Legacy [`MSG.class.php`](../../projects/game-legacy-php/src/game/page/MSG.class.php)
имеет `setGetAction("id", "DeleteAll", "deleteAllMessages")` —
массовое удаление всех сообщений в папке. У нас в origin есть
`deleteMessage(id)` для удаления одного, но нет bulk-удаления.

Confirm-текст в [MessagesScreen.tsx:122](../../projects/game-nova/frontends/origin/src/features/messages/MessagesScreen.tsx#L122)
ссылается на `t('message','deleteAll')`, но фактически удаляется
только одно сообщение — это семантический баг (текст не
соответствует действию).

## 2. Что нужно

### Backend

- Новый endpoint `DELETE /api/messages?folder=<folder>` — удалить
  ВСЕ сообщения юзера в указанной папке.
- Idempotent: повторный вызов с пустой папкой возвращает 204
  (no rows affected).
- folder validation: `inbox|sent|spy|battle|transfer` (или те, что
  поддерживает `MessageFolder`).
- Семантика: для `sent` — удалить только из своих отправленных;
  для остальных — `WHERE to_user_id = uid AND folder = X`.

### Frontend

- `api/messages.ts::deleteAllMessages(folder)` — DELETE на
  `/api/messages?folder=<folder>`.
- В `MessagesScreen.tsx` — кнопка «Удалить всё» в шапке списка,
  при клике confirm + mutation. После успеха — invalidate
  `QK.messages(folder)`.
- Confirm-текст переписать чтобы не путать с per-row delete.

### i18n

- `message.deleteAllConfirm` — «Удалить все сообщения в папке?»
  (отдельно от `deleteAll`).
- `message.deleteAllBtn` — «Удалить всё».
