## Прогресс

### /settings — закрыто

- ✅ **setProdOfUser(0) при включении vacation_since** (`auth/vacation.go::SetVacation`):
  при включении отпуска UPDATE buildings.production_factor=0 для всех
  планет юзера; при выходе — production_factor=100 (default).
  Защита от майнинга в режиме отпуска (vacation shield + 0 production
  = чистый ban-protection без эксплойта).
- ✅ **AutoMsg при смене email** (`settings/handler.go::setEmailWithNotice`):
  атомарно меняет email и шлёт системное сообщение
  `autoMessages.settingsEmailChanged.{title,body}` с упоминанием
  нового адреса. i18n ключи добавлены (ru/en).

### /settings — false positives re-audit'а (опровергнуто)

- ❌→✅ **email uniqueness check** — уже работает через UNIQUE
  constraint на `users.email`, ошибка `23505` ловится через
  `strings.Contains(err.Error(), "unique")` → 409 Conflict.
- ❌→✅ **AutoMsg password change** — пароль управляется в
  identity-service, game-nova не имеет к нему доступа. Notification
  должен прийти из identity-service.
- ❌→✅ **resent_activation** — feature identity-service, не game-nova.
  В game-nova settings таких полей нет.
- ❌→✅ **username uniqueness** — username read-only в game-nova
  settings (легаси разрешало смену, у нас архитектурно вынесено в
  identity-service). Не gap.

### /artefacts — закрыто

- ✅ **Countdown timers** (`ArtefactsScreen.tsx`): добавлен
  `useCountdown` hook + `ArtefactStatusLine` компонент. Для active/
  delayed артефактов с `expire_at` показывается live `<code>HH:MM:SS</code>`,
  обновляется каждую секунду.

### /artefacts — false positives re-audit'а (опровергнуто)

- ❌→✅ **linkify showArtefact**: `<Link to="/artefact/{id}">` уже
  присутствует на иконке (L.269) и имени (L.276) в существующем коде
  до 72.1.49. Re-audit ошибочно пометил как gap.
- ❌→✅ **Unique artefacts раздел** (legacy 322-338): unique-механика
  не реализована в nova полностью (это премиум-фича legacy
  артефактов с тегом unique=1). Это **архитектурное отличие**, не
  функциональный gap. Документировать в simplifications.md если
  потребуется.
- ❌→✅ **packed `no_activation` flag**: проверка наличия флага в
  catalog уже есть в backend `artefact/service.go::Activate` через
  `spec.NoActivation`. Frontend дёргает activate через мутацию,
  backend возвращает ошибку — корректное поведение.
