## 1. Контекст

Legacy [`Preferences.class.php`](../../projects/game-legacy-php/src/game/page/Preferences.class.php)
имеет 4 actions:

| Action legacy | Что делает | В origin |
|---|---|---|
| `saveuserdata` (`updateUserData`) | большая форма настроек | ✅ `PUT /api/settings` (частично) |
| `update_deletion` (`updateDeletion`) | поставить/снять метку удаления; legacy `delete = time() + 604800` (7 дней) | ✅ через email-код (`POST /api/me/deletion/code` + `DELETE /api/me`) |
| `resent_activation` (`resendActivationMail`) | повторно выслать email активации | ❌ нет |
| `disable_umode` (`disableUmode`) | выключить отпуск, prod=100% | ⚠ backend есть (`DELETE /api/me/vacation` через `auth/vacation.go`), UI **отсутствует** |

Также в backend есть `POST /api/me/vacation` (включить отпуск) — UI
тоже **отсутствует**.

## 3. Декомпозиция

### A. Vacation toggle (P1)

**Что**: добавить в `/settings` блок с состоянием отпуска и
кнопками включить/выключить.

**API на бэке уже готов**:
- `POST /api/me/vacation` — `SetVacation` ([auth/vacation.go:57](../../projects/game-nova/backend/internal/auth/vacation.go#L57)).
  - 409 `ErrVacationAlreadyActive` — уже в отпуске.
  - 409 `ErrVacationIntervalNotMet` — < 20 дней с прошлого окончания.
  - 409 `ErrVacationBlocked` — есть pending events (build/fleet/research).
- `DELETE /api/me/vacation` — `UnsetVacation` ([auth/vacation.go:91](../../projects/game-nova/backend/internal/auth/vacation.go#L91)).
  - 409 `ErrVacationNotActive` — не в отпуске.
  - 409 `ErrVacationTooEarly` — < 48 часов с включения (spec §18.8).

**Состояние юзера**: уже доступно в `GET /api/me` через
`vacation_since`, `vacation_unlock_at`, `vacation_last_end`
(см. [`api/me.ts:35-37`](../../projects/game-nova/frontends/origin/src/api/me.ts#L35)).

**Что делать на фронте**:

1. В `api/settings.ts` добавить две функции:
   ```ts
   export function enableVacation(): Promise<void>;
   export function disableVacation(): Promise<void>;
   ```
   Обе ходят на `/api/me/vacation` (POST/DELETE) с idempotency-key.

2. В `SettingsScreen.tsx` добавить секцию (по структуре legacy
   `Preferences.class.php` — отдельный блок «Режим отпуска»):
   - Если `vacation_since == null`:
     - кнопка «Включить отпуск» → `enableVacation()`.
     - дисплей причин блокировки если 409 («у вас pending events»,
       «20 дней не прошло»).
   - Если `vacation_since != null`:
     - текст «Вы в отпуске с <дата>».
     - кнопка «Выключить отпуск» (disabled до
       `vacation_unlock_at`):
       - если ещё рано (`now < vacation_unlock_at`) — показать
         таймер.
       - иначе — кнопка активна → `disableVacation()`.

3. i18n-ключи (новые в `prefs` группе):
   `vacationOn`, `vacationOff`, `vacationSince`,
   `vacationCannotEndYet`, `vacationCannotStart`,
   `vacationBlockedByEvents`, `vacationIntervalNotMet`.

**Эстимат**: 2-3 ч (UI + 2 mutation + i18n).

### B. Deletion flow — оценка соответствия legacy (P3)

**Что**: проверить, что нынешний flow «request code → confirm code»
семантически = legacy `update_deletion` (`time() + 604800`).

В legacy: при подтверждении `delete` ставится `time + 7 days` —
аккаунт **не удаляется немедленно**, а помечается. Удаление
происходит cron'ом через 7 дней.

В origin: `DELETE /api/me` с кодом — нужно проверить, удаляется ли
немедленно или ставится grace-period.

**Действия**:
1. Прочитать `internal/auth` deletion-handler.
2. Если немедленно — записать в `docs/simplifications.md` либо
   привести к семантике legacy (grace 7 дней + revert-кнопка).
3. Если grace — добавить UI «отменить удаление» (legacy
   `update_deletion` с `delete=0`).

**Эстимат**: 1-2 ч (sверка + при необходимости UI кнопки revert).

### C. Resend activation (P3)

**Что**: legacy `resent_activation` повторно высылает email с
ссылкой активации непод. юзеру.

**В origin**: вход через portal (handoff с identity-сервиса).
Активация email — это identity-feature, не game-сервер.

**Решение**: эта механика **должна жить в портале** (как
Support/UA в §20.12 задаче 1). Зафиксировать в плане как
«не порт» — такой же логикой как Support/UA.

**Действия**:
1. Записать в `docs/simplifications.md`:
   `[P72.1.5.RESEND_ACTIVATION_PORTAL]` — resend activation
   делегируется порталу (identity-сервис), не игровому серверу.
2. В `Preferences.class.php` legacy — оставить как есть (legacy
   служит pixel-perfect референсом для других экранов; сам endpoint
   resent_activation не используется на live oxsar-nova).
