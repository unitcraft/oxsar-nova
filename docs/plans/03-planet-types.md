# План: тип планеты, размер и визуализация

Добавлено: 2026-04-22.

## Цель

Каждая планета имеет **тип** (биом), определяемый при создании из позиции в системе
по аналогии с OGame. Тип влияет на визуализацию: картинка учитывает и тип, и размер
планеты (маленькая ледяная планета выглядит маленькой и ледяной).

---

## Шаг 1 — тип планеты в БД и при создании

### Миграция

Добавить колонку `planet_type text NOT NULL DEFAULT ''` в таблицу `planets`.

### Логика выбора типа (по OGame)

| Позиция | Возможные типы |
|---------|----------------|
| 1–3     | trockenplanet, wuestenplanet |
| 3–7     | dschjungelplanet, normaltempplanet |
| 6–10    | normaltempplanet, wasserplanet |
| 9–13    | wasserplanet |
| 12–15   | eisplanet, gasplanet |

Конкретный тип выбирается детерминированно из `coordsSeed(galaxy, system, position)` —
так же как сейчас выбирается диаметр. Результат стабилен: одни координаты всегда дают
один тип.

### Где проставлять

- `backend/internal/fleet/colonize.go` — при колонизации
- `backend/internal/fleet/expedition.go` — при экспедиции
- `backend/internal/planet/starter.go` — при создании домашней планеты
- `backend/internal/fleet/attack.go` — при создании луны (тип = `moon`)

### Температура — привязать к позиции

Сейчас температура случайная (`-40 + rng(80)`). Нужно привязать к слоту как в OGame:

```
tempMax = 120 - position * 10 + rng(-10..+10)  // слот 1 ≈ +110°, слот 15 ≈ -30°
tempMin = tempMax - 40
```

## Шаг 3 — картинки планет

### Требования

- Картинка должна отражать **тип** (биом) и **размер** планеты.
- Размер кодируется в имени файла или через CSS `width/height` при отображении.
- Маленькая ледяная планета (diameter < 8000) → маленькая картинка ледяной планеты.
- Большая газовая планета (diameter > 14000) → большая картинка газового гиганта.

### Варианты реализации

**Вариант A (CSS-масштабирование, проще):**
Оставить текущие картинки по типу, но рендерить `<img>` с размером пропорциональным
диаметру: `size = 32 + (diameter / 15000) * 32` → от 32px до 64px.
Не нужно новых файлов.

**Вариант B (отдельные файлы по размеру, точнее):**
Три градации размера (small/medium/large) × 7 типов = 21 набор картинок.
Требует генерации/подбора картинок.

**Рекомендация: Вариант A** — достаточно для геймплея, реализуется без художника.

### Функция `planetImageOf`

Добавить аргумент `planetType: string`:

```ts
export function planetImageOf(position: number, planetId: string, planetType?: string, diameter?: number): string
```

- Если `planetType` передан — использовать его напрямую вместо вычисления из позиции.
- Размер `<img>` вычислять в компоненте из `diameter`.

---

## Шаг 4 — фронтенд

- `planetImageOf` принимает `planet_type` и `diameter`.
- `PlanetOverviewCard` и карусель передают оба поля.
- Размер картинки масштабируется: `width = height = 32 + Math.round((diameter / 15000) * 32)`.
- В карусели — фиксированный контейнер 48×48, картинка внутри масштабируется через `object-fit: contain`.
