## 1. Контекст

Legacy [`MonitorPlanet.class.php`](../../projects/game-legacy-php/src/game/page/MonitorPlanet.class.php)
— отдельный экран `?go=MonitorPlanet&id=<planet>`:
- Выбирает чужую планету.
- Проверяет range: scanner.galaxy = target.galaxy, моя главная планета
  должна иметь UNIT_STAR_SURVEILLANCE >=1, |scanner.system - target.system|
  ≤ NS::getMonitorActivityRange().
- Проверяет hydrogen >= STAR_SURVEILLANCE_CONSUMPTION (5000H).
- Списывает 5000H.
- Возвращает события флота связанные с этой **планетой** (src или dst).
- Если target.user != scanner.user и target.spyware >= scanner.spyware/2:
  отправляет AutoMsg `MSG_SURVEILLANCE_DETECTED` цели в folder=11
  (MSG_FOLDER_SURVEILLANCE_DETECTED).

Origin [`MonitorPlanetScreen`](../../projects/game-nova/frontends/origin/src/features/monitor-planet/MonitorPlanetScreen.tsx)
— 50 строк, без параметров, показывает СВОИ флоты через `fetchFleet`.
Это совсем не legacy: нет ?id=, нет range/hydrogen/AutoMsg.

В backend уже есть `TransportService.Phalanx` (план 20 Ф.4) — по сути
тот же скан, но system-level. План 72.1.20 — обернуть его в новый
`monitor/` пакет, отфильтровать по конкретной планете и добавить
surveillance-AutoMsg.

## 2. Что нужно

### Backend (`internal/monitor/handler.go` — новый)

- `Monitor GET /api/monitor-planet?id=<planet_id>`
- Шаги:
  1. Найти target планету (из planets).
  2. Найти scanner — луна текущего юзера в той же галактике с
     `buildings.unit_id=55 (STAR_SURVEILLANCE) AND level>=1`.
  3. Делегировать `transport.Phalanx(scanner, target.galaxy,
     target.system)` — он списывает 5000H, проверяет range.
  4. Отфильтровать события на `target.galaxy/system/position`
     (legacy: `e.planetid = $target OR e.destination = $target`).
  5. Если target.user != me и target.spyware >= my.spyware/2:
     `automsg.SendDirect(target, folder=11, title, body)`.
- DTO: `MonitorResult{ target_planet, scanner, events[], detected }`.

### Frontend

- `api/monitor.ts` (новый): `fetchMonitorPlanet(planetId): Promise<MonitorResult>`.
- `MonitorPlanetScreen` переписать:
  - URL: `/monitor-planet/:planetId` или `?id=`.
  - Резолв planetId из URL.
  - Кнопка «Сканировать» (single-shot — каждое сканирование
    списывает 5000H!), не auto-refetch.
  - Таблица event'ов как в legacy (timer + сообщение).

### i18n

- `autoMessages.surveillanceDetected.title/body` (с {{planet}}/{{galaxy}}/
  {{system}}/{{position}}).
- `monitorPlanet.scanBtn`, `monitorPlanet.scanCost` для UI.
