# План 55: Синхронизация документации после identity-rename

**Дата**: 2026-04-27
**Статус**: Активный
**Зависимости**: план 51 ([rename auth → identity](51-rename-auth-to-identity.md))
закрыт коммитом `a68e6e2fe1`. Этот план закрывает «хвост» —
документацию, которая ссылалась на старые пути auth-сервиса.
**Связанные документы**: [docs/legal/](../legal/) (проверено — там
ссылок на старые пути нет), [docs/ops/license-audit.md](../ops/license-audit.md),
[docs/ops/legal-compliance-audit.md](../ops/legal-compliance-audit.md).

---

## Цель

После переименования `projects/auth/` → `projects/identity/` (план 51)
часть технической документации продолжает ссылаться на старые пути и
имена. Синхронизировать.

**Что важно сохранить нетронутым:**
- **Исторические записи** в `docs/project-creation.txt` (дневник
  работы) — там события записаны в момент их совершения, ссылки на
  `auth-service` отражают реальность того момента, переписывание
  истории искажает дневник.
- **simplifications.md** — там описание упрощений, принятых в
  определённой версии. Тоже исторический документ; если упрощение
  ещё актуально и реализация переехала в `identity` — добавить
  пометку, но не переписывать.
- **Memory-записи** — обновляются в `~/.claude/projects/.../memory/`,
  не в репо.

**Что обновить:**
- Активные процедурные документы (`docs/ops/*.md`): они описывают
  «как сейчас работает», должны быть актуальными.
- Активные планы (`docs/plans/*.md` со статусом «Активный»): должны
  ссылаться на правильные пути.
- Промпты (`docs/prompts/*.md`): они выполняются в будущих
  сессиях, должны давать корректные инструкции.
- Аудит-документы (`docs/ops/license-audit.md`,
  `docs/ops/legal-compliance-audit.md`): там таблицы с путями
  подпроектов, которые надо переотметить.

## Что меняем

### 1. `docs/ops/ugc-moderation.md`

Замена `auth-service` → `identity-service` (функционально это тот же
сервис).

### 2. `docs/ops/license-audit.md`

Замена `projects/auth/backend` → `projects/identity/backend` в
таблице со статистикой по модулям.

### 3. `docs/ops/legal-compliance-audit.md`

В сводной таблице покрытия и в тексте — замена `auth` → `identity`.

### 4. `docs/prompts/legal-compliance-audit.md`

Поиск автообнаружением через манифест-файлы — там пути не захардкожены,
но в комментариях и примерах есть упоминания «auth-сервис». Заменить
на «identity-сервис» где уместно.

### 5. `docs/plans/44-personal-data-152fz.md`

Активный плановый документ. В разделах с реализацией (Ф.1–Ф.5) есть
указание `projects/auth/migrations/`, `projects/auth/backend/...`.
Заменить на `identity/`. Сохранить пометку «изначально реализовано
в auth, переименовано в identity по плану 51».

### 6. `docs/plans/47-offer-tos.md`

То же — упоминания auth-сервиса в реализации.

### 7. `docs/plans/50-game-origin-legal-fix.md`

Мой собственный план для game-origin. Упоминает `/api/reports`
auth-сервиса (Ф.5). Переименовать в `identity-service /api/reports`.

### 8. `CLAUDE.md`

Ссылка `(plan 36, переименован в плане 51)` уже добавлена соседним
агентом — корректно. Дополнительной правки не требуется.

## Чего НЕ делаем

- Не правим `docs/project-creation.txt` — это исторический дневник.
- Не правим `docs/simplifications.md` — это история принятых
  упрощений.
- Не правим `docs/ui/dev-log.md` — это исторический dev-log.
- Не правим memory `feedback_*.md` / `reference_*.md` /
  `completed_plans.md` в `~/.claude/projects/.../memory/` — там при
  необходимости агент сам обновит при следующих обращениях; форсить
  правки не нужно.
- Не правим записи о коммитах вроде «коммит 0a012d8be4 fix(plan-36
  Ф.12.5)» — тогда сервис назывался auth-service, это верно.
