## 1. Контекст

Legacy [`MSG.class.php`](../../projects/game-legacy-php/src/game/page/MSG.class.php)
реализует **многопапочную** систему сообщений:

- Index `?go=MSG` показывает **список папок** с счётчиками (всего/новых/
  размер); каждая папка — ссылка `?go=MSG&id=ReadFolder&folder=N`.
- Папки хранятся в таблице `folder (folder_id, userid, label, is_standard,
  display_order)`. is_standard=1 — общие для всех (системные); userid≠0 —
  пользовательские.
- Constants ([config/consts.php:514+](../../projects/game-legacy-php/config/consts.php)):
  - `MSG_FOLDER_FLEET=1` — отчёты флота
  - `MSG_FOLDER_RECYCLER=2` — мусорщики
  - `MSG_FOLDER_SPY=3` — шпионаж
  - `MSG_FOLDER_REPORT=4` — боевые отчёты
  - `MSG_FOLDER_TRANSPORT=5` — транспорт
  - `MSG_FOLDER_ALLIANCE=6` — альянс
  - `MSG_FOLDER_ARTEFACT=7` — артефакты
  - `MSG_FOLDER_CREDIT=8` — кредитные транзакции
  - `MSG_FOLDER_EXPEDITION=9` — экспедиции
  - `MSG_FOLDER_SEND=10` — отправленные мной
  - …и т.д. (точное число — TBD при реализации)
- AutoMsg.class.php switch case переводит `MSG_*` → нужный `folder_id`.

Origin (план 72 Ф.5 Spring 4):
- [`MessagesScreen.tsx`](../../projects/game-nova/frontends/origin/src/features/messages/MessagesScreen.tsx)
  показывает только два таба inbox/sent.
- Backend `GET /api/messages` принимает `?folder=inbox|sent`,
  фильтрует по `to_user_id=me` или `from_user_id=me`.
- AutoMsg.SendDirect пишет в произвольный `folder int`, но единого
  списка папок и UI для них нет.

## 2. Что нужно

### Backend

#### Миграция `0087_message_folders.sql`
```sql
-- Таблица папок (legacy `folder`).
CREATE TABLE IF NOT EXISTS message_folders (
    folder_id   smallint PRIMARY KEY,
    label       text NOT NULL,            -- i18n-ключ (например 'msgFolder.fleet')
    is_standard boolean NOT NULL DEFAULT true,
    display_order smallint NOT NULL DEFAULT 0
);

-- Seed системных папок (legacy const).
INSERT INTO message_folders (folder_id, label, display_order) VALUES
    (1,  'msgFolder.fleet',      10),
    (2,  'msgFolder.system',     20),  -- existing (auto-msg system) — оставляем семантику
    (3,  'msgFolder.spy',        30),
    (4,  'msgFolder.report',     40),
    (5,  'msgFolder.transport',  50),
    (6,  'msgFolder.alliance',   60),
    (7,  'msgFolder.artefact',   70),
    (8,  'msgFolder.credit',     80),
    (9,  'msgFolder.expedition', 90),
    (10, 'msgFolder.sent',      100)
ON CONFLICT (folder_id) DO NOTHING;
```

(Если у нас уже используется `folder=2` для system/welcome — оставим
семантику; в legacy folder=2 = recycler; уточнить при реализации,
возможно сдвинуть существующие данные.)

#### Новые/изменённые endpoints

- **`GET /api/messages/folders`** (новый):
  ```json
  {"folders":[
    {"folder_id":1, "label":"msgFolder.fleet", "messages":42, "unread":3, "storage":2048},
    ...
  ]}
  ```
  Список **системных** папок + счётчики для текущего юзера.

- **`GET /api/messages?folder=<id>`** — изменить семантику параметра
  с `'inbox'|'sent'` на численный `folder_id`. Backwards-compat:
  если параметр строка → 400.

- **`POST /api/messages`** — компоновщик отправки. При успехе
  сохранять копию в папку `MSG_FOLDER_SEND=10` (folder=10 для
  отправителя; legacy так делает).

- **`DELETE /api/messages?folder=<id>`** — DeleteAll по папке (уже
  есть, но обобщить семантику).

### Frontend

- Новый corner-route `/msg` (без folder) — список папок с счётчиками.
- Существующий `/msg/inbox` → редирект на `/msg/folder/2` (или новый
  адрес `/msg/folder/<id>`).
- Существующий `/msg/sent` → редирект на `/msg/folder/10`.
- Новый компонент `MessageFoldersScreen` — таблица folders с
  unread/total counts, ссылками на папки.
- `MessagesScreen` принимает folderId из URL, рендерит содержимое
  выбранной папки.

### i18n

- Группа `msgFolder`: `fleet`, `system`, `spy`, `report`, `transport`,
  `alliance`, `artefact`, `credit`, `expedition`, `sent` (10 ключей).

### AutoMsg маппинг

Существующие AutoMsg-вызовы используют hardcoded folder int — все надо
свериться с legacy AutoMsg.class.php switch и поправить:
- welcome/starter → folder=2 (system) — корректно.
- creditProfessionChanged → folder=8 — корректно (только что добавлено).
- inactivityReminder → folder=2 (system) — корректно.
- battle reports → должны идти в folder=4 (report).
- spy reports → folder=3.
- transport → folder=5.
- alliance events → folder=6.
- artefact notice → folder=7.
- expedition → folder=9.

Прогон по `internal/automsg/service.go` + всем вызовам `SendDirect`/
`Send` чтобы folder соответствовал legacy.
