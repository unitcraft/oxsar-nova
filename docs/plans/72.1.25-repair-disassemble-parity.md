**Дата**: 2026-05-01

## 1. Контекст

Legacy [`Repair.class.php`](../../projects/game-legacy-php/src/game/page/Repair.class.php)
— один класс с unit_type-разделением (repair/disassemble). Origin
вынес `/disassemble` отдельно. По чек-листу §20.12:

**Backend EnqueueRepair / EnqueueDisassemble**:
- ❌ нет umode-блока (legacy `if(umode) throw`).
- ❌ нет observer-блока.
- ❌ нет PLANET_UNDER_ATTACK блока (legacy
  `Logger::dieMessage('PLANET_UNDER_ATTACK')`).

**Frontend RepairScreen / DisassembleScreen**:
- ❌ не показывает required/earn ресурсы в таблице (legacy:
  `metal_required / metal_earned`).
- ❌ нет abort-кнопки для активной задачи (backend `Cancel` есть).

**Закрываем всё** (без упрощений):
- VIP startImm (мгновенный старт за credit) — новый backend handler
  + UI кнопка.
- dock_capacity (per-unit fields через unit_fields × count ≤
  free_storage).

## 2. Что нужно

### Backend `repair/service.go`

Добавить umode/observer/under-attack проверки в EnqueueDisassemble и
EnqueueRepair:
- `users.umode = true` → `ErrInVacation`.
- `users.is_observer = true` → `ErrIsObserver`.
- planet под атакой (есть `fleet_events` с `mode='attack'` и
  `fire_at > now()`) → `ErrPlanetUnderAttack`.

### Frontend

**DisassembleScreen** — расширить таблицу:
- Колонки «Получите» (return - required = earn) на M/Si/H.
- Кнопка abort у задач в очереди (вызов cancel-mutation).

**RepairScreen** — то же самое:
- required-ресурсы видны (в legacy их видно даже если не хватает —
  с подсветкой).
- abort-кнопка.

### i18n
- `repair.colReturns` — «Получите».
- `repair.abortBtn` — «Отменить».
- `repair.errInVacation`, `repair.errObserver`, `repair.errUnderAttack`.
