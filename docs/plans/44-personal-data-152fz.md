# План 44: 152-ФЗ — обработка персональных данных пользователей

**Дата**: 2026-04-26
**Статус**: Активный
**Зависимости**: ничего технически блокирующего; должен быть выполнен **до**
открытия публичной регистрации (любого канала, где сторонний пользователь
может оставить email/имя/IP).
**Связанные документы**: [LICENSE](../../LICENSE),
[39-license-ru-jurisdiction.md](39-license-ru-jurisdiction.md),
[41-origin-rights.md](41-origin-rights.md).

---

## Цель

Привести проект в соответствие с Федеральным законом № 152-ФЗ
«О персональных данных» (РФ). Без этого нельзя легально открывать
публичную регистрацию: штрафы по ст. 13.11 КоАП — до 18 млн ₽
за повторное нарушение для организаций, до 700 тыс. ₽ для ИП и
самозанятых физических лиц.

Что закрываем:
1. Регистрация автора как **оператора персональных данных** в
   Роскомнадзоре (РКН) — бесплатно, онлайн.
2. Публичная **Политика конфиденциальности** (`docs/legal/privacy-policy.md`
   + страница на портале).
3. Чекбокс **согласия на обработку ПДн** в форме регистрации (frontend
   game-nova и portal); запись согласия в БД с timestamp и IP.
4. **Технические меры** хранения ПДн в РФ (серверы Selectel, в memory
   уже зафиксировано как требование).
5. **Право на удаление** (ст. 14 152-ФЗ): API-эндпоинт + UI-кнопка
   "удалить аккаунт".

## Что меняем

### 1. `docs/legal/privacy-policy.md` (новый)

Структура (минимальный обязательный набор по 152-ФЗ):

- **Оператор**: автор-правообладатель (идентификация в публичной
  политике конфиденциальности по требованиям 152-ФЗ; ИНН не
  публикуется, регистрируется только в РКН).
- **Цели обработки**: предоставление игрового сервиса, аутентификация,
  поддержка пользователей, направление сервисных уведомлений.
- **Категории субъектов**: пользователи игры oxsar-nova и портала
  oxsar-nova.ru.
- **Категории ПДн**: email, никнейм, IP-адрес, данные авторизации
  (хешированный пароль, токены), внутриигровые идентификаторы.
- **Правовые основания**: согласие субъекта (ст. 6 п. 1 пп. 1 152-ФЗ)
  + договор-оферта с пользователем (ст. 6 п. 1 пп. 5).
- **Способ обработки**: автоматизированный, без передачи третьим лицам
  (кроме платёжных шлюзов — YooKassa — для проведения платежей).
- **Сроки хранения**: до удаления аккаунта пользователем или 3 года
  с момента последней активности.
- **Права субъекта**: доступ, изменение, удаление, отзыв согласия;
  email для запросов.
- **Меры защиты**: серверы в РФ (Selectel), парольное хеширование,
  HTTPS, изоляция БД от публичной сети.
- **Контакты оператора**: email из COMMERCIAL-LICENSE.md.

Страница на портале — `projects/portal/frontend/src/pages/Privacy.tsx`
(или аналог) с тем же текстом, плюс ссылка в футере и в форме
регистрации.

### 2. Чекбокс согласия в регистрации

В формах регистрации (game-nova и portal):

- Чекбокс **обязательный** (без галочки кнопка "Зарегистрироваться"
  заблокирована).
- Текст: "Я ознакомлен с [Политикой конфиденциальности](/privacy) и даю
  согласие на обработку персональных данных в соответствии с
  Федеральным законом № 152-ФЗ".
- При submit — backend сохраняет в таблицу `user_consents`:
  `user_id`, `consent_type='pdn_processing'`, `consent_text_version='1.0'`,
  `accepted_at`, `accepted_ip`, `accepted_user_agent`.

### 3. Миграция `user_consents`

Новая таблица в auth-сервисе (или там, где живёт регистрация):

```sql
CREATE TABLE user_consents (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL REFERENCES users(id),
  consent_type VARCHAR(64) NOT NULL,
  consent_text_version VARCHAR(16) NOT NULL,
  accepted_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  accepted_ip INET,
  accepted_user_agent TEXT
);
CREATE INDEX ON user_consents(user_id, consent_type);
```

Это страхует на случай спора: всегда можно показать, кто, когда и с
какого IP принял какую версию документа.

### 4. Право на удаление аккаунта

Новый API-эндпоинт `DELETE /auth/users/me` (или `POST /auth/users/me/delete`
с подтверждением):

- Удаляет/анонимизирует ПДн пользователя: email → `deleted_<id>@oxsar`,
  никнейм → `deleted_user_<id>`, пароль → невалидный.
- Игровые объекты (планеты, флоты, рейтинги) — оставляем (их история
  важна для других игроков), но владелец становится "удалённым".
- В UI — страница "Настройки аккаунта" → кнопка "Удалить аккаунт"
  с подтверждением.
- API возвращает 204 после удаления.

Полная физическая деинициализация всех связанных данных не
требуется по 152-ФЗ — достаточно деперсонализации (ст. 3 п. 9).

### 5. Уведомление в РКН

Внерепный шаг (не коммитится):

- Зайти на портал РКН
  (https://www.gosuslugi.ru/600127/2 или https://pd.rkn.gov.ru/operators-registry/notification/form/)
  с подтверждённой учётной записью Госуслуг.
- Заполнить форму уведомления оператора ПДн (тип учётной записи
  выбирается оператором по применимому налоговому режиму).
- Получить регистрационный номер оператора.
- Внести регистрационный номер в `docs/legal/privacy-policy.md` в поле
  "Оператор" (формат: `регистрационный номер РКН: XX-XXXXXX-XXX`).

### 6. Cookie-баннер (опционально, минимальный)

Поскольку oxsar-nova использует cookies (сессионная авторизация) —
короткий баннер "Этот сайт использует cookies для авторизации; продолжая
использовать сайт, вы соглашаетесь с этим" + ссылка на Privacy. Без
сложной cookie-management-системы — для аудитории РФ это избыточно
(GDPR не применим).

## Что НЕ делаем

- Не подключаем сложные cookie-management-системы (OneTrust и т.п.) —
  избыточно для РФ-аудитории.
- Не делаем DPO (Data Protection Officer) — для оператора с одной
  игрой не обязательно.
- Не делаем DPIA (Data Protection Impact Assessment) — у нас нет
  спецкатегорий ПДн (биометрия, здоровье и т.п.).
- Не интегрируем GDPR-механизмы (DSAR, "право на портативность" и т.п.) —
  GDPR не применим к проекту с РФ-аудиторией; добавлять при выходе на ЕС.

## Тестирование

- Регистрация нового пользователя без галочки — кнопка submit заблокирована.
- Регистрация с галочкой — запись в `user_consents` появляется с
  корректным IP/UA/timestamp.
- Удаление аккаунта — email/никнейм деперсонализируются, авторизоваться
  старыми credentials нельзя.
- Privacy Policy открывается по прямой ссылке `/privacy`.
