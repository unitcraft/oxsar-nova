#### Cascade-payback (если общее производство H < общее потребление H)

Применяется к каждой virt-строке:
1. Если `consH > prodH-currentConsH` → over_prod = excess
2. Уменьшить consH этой строки на over_prod, добавить consSilicon =
   `over_prod * MARKET_BASE_CURS_SILICON / MARKET_BASE_CURS_HYDROGEN` (=0.5)
3. Если consSilicon тоже не хватает → cascade в metal через
   `MARKET_BASE_CURS_METAL/MARKET_BASE_CURS_SILICON` (=1.5)

Эта логика — 1:1 с `Planet.class.php::getProduction()` строки 481-540.
Сложность переноса значительная — тщательно тестировать на golden-fixtures.

## 4. Решения / trade-off

- **Cascade-payback**: legacy реализует через хитрую цепочку if'ов по
  3 ресурсам. Перенесём 1:1, но если в тестах будет divergence на
  edge-cases — зафиксируем как simplification (хотя по правилу 1:1
  это запрещено, поэтому будем доводить до точного равенства).
- **Halting fleets**: используем существующую таблицу `events` с
  `kind=KindHolding` (=17). Если `data` не содержит `ships` структуру
  в JSON — это уже отдельный баг event-системы, fix в этом плане не
  делаем (отметить в audit).
