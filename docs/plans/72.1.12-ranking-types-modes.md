## 1. Контекст

Legacy [`Ranking.class.php`](../../projects/game-legacy-php/src/game/page/Ranking.class.php)
поддерживает:
- **mode**: player / alliance / player_old_vacation / player_observer
- **type**: dm_points / max_points / points / b_points / r_points /
  u_points / b_count / r_count / u_count / e_points / battles
- **avg**: режим «среднее за день»
- **pos**: пагинация по 25 строк

Origin [`RankingScreen.tsx`](../../projects/game-nova/frontends/origin/src/features/ranking/RankingScreen.tsx)
показывает только один список топ-100 по `points`. Сильно урезан
(P72.S3.F simplification).

Backend `score.Service` уже умеет:
- `Top(scoreType)` для total/b/r/u/a/e — col-mapping в `columnFor`.
- `TopAlliances` — рейтинг альянсов.
- `VacationPlayers` — игроки в отпуске.
- `PlayerRank/PlayerScore` — позиция по типу.

В БД (план 72.1 ч.17, миграция 0082) уже есть `dm_points`, `max_points`,
индекс `users_dm_points_idx`.

## 2. Что нужно

### Backend
- Расширить `score.columnFor` чтобы понимать `dm`, `max`.
  - `dm` → `dm_points`, `max` → `max_points`.
- `Top(scoreType)` уже использует `columnFor`, изменений не требует.
- `PlayerRank/PlayerScore` уже используют `columnFor` — тоже ок.

### Frontend
- В `api/highscore.ts` — добавить параметр `type` к
  `fetchHighscore`/`fetchHighscoreMe`.
- В `RankingScreen`:
  - Табы `player` / `alliance` / `vacation` (3 mode'а; observer и
    old_vacation — низкий приоритет, пропускаем).
  - Селект type — показывает b/r/u/a/e/total/dm/max в зависимости от
    выбранного таба (для alliance — только total).
  - URL state через `useSearchParams` — `?mode=...&type=...` чтобы
    закладки/назад работали.

### i18n
- Ключи `score.tab*` и `score.type*` (group=score). Часть уже есть
  (`scoreTypeTotal`); добавить scoreTypeBuildings/Research/Units/
  Achievements/Energy/Dm/Max + tabPlayers/tabAlliances/tabVacation.

### Не сделано (вне scope)
- **avg-режим** — отдельная сложность, нужен JOIN на regtime; YAGNI.
- **b_count/r_count/u_count** — нет агрегатных колонок, нужен GROUP BY.
- **player_observer / player_old_vacation** — нишевые режимы.
- **Пагинация по 25** — топ-100 без пагинации, для MVP достаточно.
- **battles** — нужна колонка с числом боёв.
