## 1. Контекст

§20.12 задаёт 6-пунктный чек-лист для каждого из 40 экранов.
Display-only экраны (0 actions) самые простые — достаточно сверить:
1. Все ли поля показаны.
2. Действия отсутствуют (по определению).
3. Валидации не нужны (read-only).
4. Формулы не применяются.
5. Побочные эффекты отсутствуют.
6. Особые ветки кода (если есть).

### `/records` ([Records.class.php](../../projects/game-legacy-php/src/game/page/Records.class.php))
- Legacy: топ-1 по каждой постройке (max(level)), исследованию (max(level)),
  юнитам (max(sum_quantity)) с привязкой к игроку. 5 секций
  (construction/moon/shipyard/defense/research).
- Origin: backend [`internal/records/handler.go`](../../projects/game-nova/backend/internal/records/handler.go)
  отдаёт `MAX(b.level)` для buildings с `GROUP BY user`,
  `MAX(level)` для research, `SUM(s.count)` для ships/defense.
  Дополнительно — `topScore()` (записи по очкам) которой в legacy нет
  (расширение).
- UI [`RecordsScreen`](../../projects/game-nova/frontends/origin/src/features/records/RecordsScreen.tsx)
  показывает 5 секций + личный результат игрока (`my_value` —
  расширение, в legacy нет).
- ✅ паритет + расширения (score topик, my_value).
