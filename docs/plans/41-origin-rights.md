## Цель

Зафиксировать в публичных документах репозитория:

1. oxsar-nova — самостоятельное произведение на стеке Go/TypeScript;
   автор кода — Е. Головин; AI-ассистент Claude Code использовался как
   технический инструмент под контролем автора (соответствие ст. 1257 ГК РФ).
2. Связь с legacy-проектом oxsar2 — на уровне идей, геймплейных механик,
   формул и числовых констант (не охраняются авторским правом — ст. 1259 п.5
   ГК РФ); исходный код, графические ассеты и тексты — самостоятельные.
3. Согласованную формулировку авторства AI во всех юридических документах
   ("инструмент", не "соавтор") и в trailer'ах коммитов на будущее.

## Что меняем

### 1. `docs/origin-rights.md` (новый файл)

Структура:

- **Автор и владелец прав на oxsar-nova.** Е. Головин, физлицо. AI-инструменты
  (Claude Code) использовались как технические ассистенты; творческий отбор,
  проверка, интеграция и компоновка кода выполнены автором.
- **Отношение к oxsar2.** oxsar-nova — самостоятельное произведение,
  написанное с нуля на другом стеке (Go + TypeScript). Использует общедоступные
  идеи и геймплейные механики; не является переводом, портом или производным
  произведением в смысле ст. 1259 п.2 ГК РФ.
- **Что не заимствуется напрямую.** Дословный исходный код, графические ассеты,
  тексты wiki/описаний/UI-сообщений в их точной форме — самостоятельные.
- **Авторство AI как инструмента.** Формулировка соответствия ст. 1257 ГК РФ.
- **Применимое право и юрисдикция.** РФ, синхронно с COMMERCIAL-LICENSE.md и
  CLA.md (по плану 39).
- **Атрибуция в истории коммитов.** До коммита `<SHA>` (дата) использовался
  trailer `Co-Authored-By: Claude <noreply@anthropic.com>` как техническая
  метка процесса использования AI-ассистента; это не юридическое заявление
  о соавторстве. Начиная с указанного коммита trailer заменён на
  `Generated-with: Claude Code` для точного отражения статуса AI как
  инструмента.
- **История версий документа.** Дата, изменение.

### 2. `README.md` — раздел "## Лицензия"

Заменить строку про "совместно с Claude Code" на:

> Автор кода — Evgeniy Golovin. При разработке использовался AI-ассистент
> Claude Code как технический инструмент; творческий отбор, проверка,

Других правок нет. Раздел "Лицензия" уже получает строку про юрисдикцию по
плану 39 — порядок коммитов важен (сначала 39, потом 41).

### 3. `.claude/settings.json` (проектный) — атрибуция коммитов

Добавить блок:

```json
"attribution": {
  "commit": "Generated-with: Claude Code",
  "pr": ""
}
```

Эффект: будущие коммиты получают trailer `Generated-with: Claude Code` вместо
`Co-Authored-By`. Произвольная метка, не интерпретируемая Git и GitHub как
соавторство. PR-описания не получают AI-атрибуцию.

Правка только в проектном `.claude/settings.json`, не в глобальном.

### 4. `memory/feedback_commits.md` — обновить правило

Заменить старую формулировку про `Co-Authored-By: Claude` на актуальную:
коммиты идут с trailer'ом `Generated-with: Claude Code` через
`attribution.commit` в проектном settings.json.

### Ф.1. Создать `docs/origin-rights.md`

README.md, LICENSE, CLA.md, COMMERCIAL-LICENSE.md.

### Ф.2. Аудит дословных заимствований

Проверить и при необходимости переписать совпадения oxsar-nova ↔ oxsar2 в:

- **Ф.2.1.** Wiki ([projects/game-nova/frontend](../../projects/game-nova/frontend),
  [docs/wiki/](../wiki/)) — сравнить с шаблонами `.tpl` исходного проекта;
  поиск ключевых фраз через grep.

  `description`, `name_ru`, `tooltip`.

  Короткие общеупотребительные слова ("Атаковать", "Построить") не охраняемы;
  искать длинные фразы.
- **Ф.2.4.** Комментарии в коде Go/TS — проверить grep'ом по характерным
  фразам.

Найденные совпадения переписать своими словами с сохранением смысла. Объём
правок зафиксировать одной строкой в "История версий" `docs/origin-rights.md`.

Заменить формулировку (см. выше). Добавить ссылку на `docs/origin-rights.md`.

### Ф.4. Атрибуция коммитов на будущее

1. Добавить блок `attribution` в проектный `.claude/settings.json` (см. выше).
2. Обновить memory `feedback_commits.md` — новая формулировка правила.
3. После коммита плана 41 проверить через `git log -1 --format=%B`, что в нём
   уже `Generated-with: Claude Code`. Если settings ещё не подхвачены —
   разделительным считается следующий коммит, его SHA указывается в
   `docs/origin-rights.md`.

- `git status` — в индексе только файлы Ф.1+Ф.3+Ф.4: `README.md`,
  `docs/origin-rights.md` (новый), `.claude/settings.json`, и файлы из Ф.2
  (если найдены заимствования). Добавлять файлы поимённо, не `git add -A`.
- Обновить `docs/project-creation.txt`: новая итерация — "план 41:
  origin-rights, переформулировка авторства, attribution коммитов".

- После `git commit` взять SHA коммита и вписать его в
  `docs/origin-rights.md` в раздел "Атрибуция в истории коммитов" как
  разделительный — отдельным коротким коммитом
