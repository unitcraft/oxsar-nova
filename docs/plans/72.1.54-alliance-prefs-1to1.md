# План 72.1.54 — Alliance prefs 1:1 (закрытие P72.S2.ALLIANCE_PREFS)

> Подплан 72.1.2 / закрытие simplification из 72.1.52 §C
> (`[P72.S2.ALLIANCE_PREFS]`). По распоряжению пользователя
> «убрать упрощения, сделать 1:1» полностью реализуем legacy
> `Alliance.class.php::updateAllyPrefs` (L.979-1015) — 6 полей
> которые ранее были задокументированы как simplification.

## 1. Контекст

Legacy `updateAllyPrefs` принимает 10 полей. До этого плана nova
реализовывала 4: `open, description_external, description_internal,
description_apply` (плюс `tag, name` через отдельный
`updateAllianceTagName`). Не реализовано было 6: `logo, homepage,
foundername, showmember, showhomepage, memberlistsort`.

Раньше (72.1.52) они были задокументированы как
`[P72.S2.ALLIANCE_PREFS]` с обоснованием:
- logo/homepage = UGC URL → требует image-proxy (plan-46-tier).
- foundername = nova: rank='owner' уже есть title.
- showmember/showhomepage/memberlistsort = privacy/preference → nova
  показывает всё всем по дефолту.

Пользователь сказал «не упрощать». Реализуем.

## 2. Решения по архитектуре

### 2.1 logo/homepage — UGC URL без image-proxy

Legacy 1:1: regex-валидация URL без скачивания. Это слабее plan-46
moderation (нет SSRF-защиты на стороне сервера), но **legacy так и
работает** — валидация `isValidImageURL`/`isValidURL` есть только
на формате URL, контент не проверяется.

Идти legacy 1:1: regex `^https?://...\.(png|jpg|jpeg|gif|webp|svg)$`
для logo, `^https?://...$` для homepage. Длина ≤128 (legacy
`Str::length($logo) > 128`).

**Известные риски** (документированы здесь, не в simplifications.md
потому что это legacy 1:1):
- Owner может вписать URL на phishing-страницу как homepage.
- Owner может вписать ссылку на трекер как logo (browser tracking).
- Open redirect через homepage.

Mitigation: target="_blank" rel="noopener noreferrer" на homepage-link
во фронте; img без referrer и без crossorigin (browser сам не шлёт
cookies на сторонние img URL'ы).

### 2.2 foundername / showmember / showhomepage / memberlistsort

Безопасные простые поля:
- foundername — text ≤64.
- show_member / show_homepage — bool с default true (legacy дефолт
  тоже true в большинстве установок).
- memberlist_sort — smallint 0..4 (legacy enum, FE интерпретирует).

## 4. Что **не делается**

- Не вводим image-proxy для logo (legacy 1:1; известные UGC-риски
  документированы в §2.1).
- Не вводим server-side filtering memberlist по show_member (это
  отдельный backend change в `members` endpoint; для outsiders
  AllianceMembersScreen вообще не доступен — он member-only). Если
  понадобится — сделаем когда будет outsider-view memberlist.
- Не реализуем серверную сортировку memberlist по memberlist_sort
  (это preference для FE, FE может сортировать клиентским кодом
  на основе значения; пока FE использует фиксированную сортировку
  rank → join_at).
