## 2. Часть 1 — /Main retreat

**Что**: Legacy [`Main.class.php:38-50`](../../projects/game-legacy-php/src/game/page/Main.class.php#L38)
имеет `setPostAction("retreat", "retreatFleet")` + кнопку «↩» прямо в
строке fleet event. Origin вынес recall на /fleet-operations
(FleetOperationsScreen.tsx), в Main кнопки не было.

**API готов**: [`recallFleet()` в `api/fleet.ts:28`](../../projects/game-nova/frontends/origin/src/api/fleet.ts#L28)
дёргает `POST /api/fleet/{id}/recall`. Backend
[`transport.go:613`](../../projects/game-nova/backend/internal/fleet/transport.go#L613)
требует `state='outbound'` — после прибытия recall не имеет смысла.

**Реализация**:

- [`MainScreen.tsx`](../../projects/game-nova/frontends/origin/src/features/main/MainScreen.tsx):
  - import: `useState`, `useMutation`, `useQueryClient`, `recallFleet`,
    `ApiError`.
  - state `recallErr: string | null`.
  - mutation `recall = useMutation({ mutationFn: recallFleet, onSuccess
    invalidate fleet, onError → recallErr })`.
  - В строке fleet (только если `f.state === 'outbound'`) добавить
    `<button>↩</button>` с `window.confirm` и `disabled={recall.isPending}`.
  - Под блоком events — строка ошибки `recallErr` если есть.

- i18n: ключи `fleet.recall`/`fleet.recallErr` уже существуют в
  [`configs/i18n/ru.yml:1016,1035`](../../projects/game-nova/configs/i18n/ru.yml#L1016).

**Тесты**: vitest для main/ нет (компонент тяжелый, требует QueryClient/
i18n provider). Регрессия покрыта tsc + общим run vitest (133/133).
Дополнительный unit-тест для retreat-кнопки — необязательный, MainScreen
не тестируется компонентно в origin вообще.

**Симплификации**: маркер `[P72.S4.MAIN_RETREAT]` вводить **не надо** —
gap закрыт, не отложен. Обновляется только snapshot 72.1.2.

## 3. Часть 2 — /battlestats sort + date-range (план 72.1.10 wave 2)

**Реализовано** (коммит идёт после ч.1):

- Backend [`battlereport/handler.go`](../../projects/game-nova/backend/internal/battlereport/handler.go):
  - Выделены pure-функции `clampDateRange` (legacy
    [`Battlestats.class.php:65-88`](../../projects/game-legacy-php/src/game/page/Battlestats.class.php#L65),
    окно `[now-18d .. now-3d]` через `battlesHiddenDays=3` +
    `battlesWindowDays=15`) и `sortClause` (legacy whitelist
    [`Battlestats.class.php:94-102`](../../projects/game-legacy-php/src/game/page/Battlestats.class.php#L94)).
  - Добавлены legacy sort-поля `outcome` (winner) и `moon`
    (is_moon) — теперь backend whitelist'ит 6 полей.
  - Серверная защита: даже если клиент пришлёт `date_min` за
    пределами `[now-18d .. now-3d]`, мы его сужаем; пустой
    диапазон → дефолт всё окно.
  - SQL-инъекция через `sort_field`/`sort_order` невозможна
    (whitelist + DESC fallback), покрыто unit-тестом.

- Backend тест [`battlereport/handler_test.go`](../../projects/game-nova/backend/internal/battlereport/handler_test.go):
  - 5 кейсов `clampDateRange`: empty/too-wide/within-window/
    invalid-string/inverted (max < min → swap).
  - 11 кейсов `sortClause`: все 6 полей × 2 направления + 3
    edge case (unknown / SQL-injection / unknown order).

- OpenAPI [`api/openapi.yaml`](../../projects/game-nova/api/openapi.yaml#L482):
  расширен `GET /api/users/me/battles` с 11 query параметрами
  (date_min/max, user/alliance_filter, 5 boolean-фильтров,
  sort_field enum [date,rounds,debris,loot,outcome,moon],
  sort_order enum). Description ссылается на legacy + clamp-окно.

- Frontend [`api/battles.ts:47`](../../projects/game-nova/frontends/origin/src/api/battles.ts#L47):
  тип `sort_field` расширен до `'date' | 'rounds' | 'debris'
  | 'loot' | 'outcome' | 'moon'`.
- Frontend [`features/battlestats/BattleStatsScreen.tsx:50`](../../projects/game-nova/frontends/origin/src/features/battlestats/BattleStatsScreen.tsx#L50):
  state-тип расширен; в `<select>` добавлены 2 новых
  `<option>`.

- i18n: ключи `statistics.bsSortOutcome`/`bsSortMoon` (ru
  заполнены, en пустые в стиле остальных bsSort* — ru-only).

**Отложено в wave 3**:
- Поле `planet_name` legacy (`Battlestats.class.php:96` →
  `p.planetname`) требует JOIN с planets и колонки
  `target_planet_id` в `battle_reports` (которой нет — нужна
  миграция + backfill в attack/acs_attack/expedition INSERT'ах).
- Integration-тест handler'а с реальной pgx pool (testcontainer)
  — в проекте нет паттерна, не вводим в этом плане.

**1:1 паритет**: ✅ для всех wave 2 пунктов. /battlestats
закрывается ✅ (backlog wave 3 = только planet_name sort, не
блокер).
