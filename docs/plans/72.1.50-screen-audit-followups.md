## 5. Часть 4 — Idempotency rollout (план 77 Ф.3.2)

План 77 Ф.2 создал middleware [`pkg/idempotency/middleware.go`](../../projects/game-nova/backend/pkg/idempotency/middleware.go);
Ф.3 закрепил его за конкретными route'ами. План 72.1.49 закрыл
shipyard + repair (3 route'а). Часть 4 этого плана — **rollout
оставшихся мутирующих route'ов**, для которых фронт уже шлёт
`Idempotency-Key` (см. [`api/*.ts`](../../projects/game-nova/frontends/origin/src/api/)).

Middleware (`pkg/idempotency/middleware.go:96-100`) идемпотентен
по дизайну: без header'а просто пропускает запрос → можно безопасно
оборачивать любой роут даже если часть клиентов ключ не шлёт.

### A. Builds + economy (закрыто)

- Backend [`cmd/server/main.go`](../../projects/game-nova/backend/cmd/server/main.go):
  обёрнуты `idemMW.Wrap` для:
    - `POST /planets/{id}/buildings`, `DELETE buildings/queue/{taskId}`,
      `POST .../{taskId}/vip`, `POST .../{unitId}/demolish`,
      `POST .../{unitId}/pack`, `POST research/{unitId}/pack`
    - `POST research`, `DELETE research/{queueId}`,
      `POST research/{queueId}/vip`
    - `DELETE shipyard/{queueId}`, `POST shipyard/{queueId}/vip`
    - `DELETE repair/queue/{queueId}`, `POST repair/queue/{queueId}/vip`
    - `POST artefacts/{id}/activate`, `POST .../deactivate`,
      `POST .../sell`
    - `POST artefact-market/offers/{id}/buy`,
      `DELETE artefact-market/offers/{id}`
    - `POST officers/{key}/activate`, `POST professions/me`

  Итого: 18 новых обёрток + 3 уже было от плана 72.1.49.

- OpenAPI [`api/openapi.yaml`](../../projects/game-nova/api/openapi.yaml):
    - добавлен общий компонент `responses.IdempotencyConflict`
      (text/plain, как middleware пишет на mismatched body).
    - shipyard/repair (плана 72.1.49) переведены с inline `409`
      на `$ref: IdempotencyConflict`.
    - 13 endpoint'ов получили `IdempotencyKey` параметр + 409 ref:
      buildings(POST/DELETE), research, shipyard, repair (4),
      artefacts (3), artefact-market (2), officers, professions.

  **Документационный gap (вне scope A)**: 8 route'ов
  (`buildings/queue/{taskId}/vip`, `buildings/{unitId}/demolish`,
  `buildings/{unitId}/pack`, `research/{unitId}/pack`,
  `research/{queueId}`, `research/{queueId}/vip`,
  `shipyard/{queueId}`, `shipyard/{queueId}/vip`,
  `repair/queue/{queueId}`, `repair/queue/{queueId}/vip`)
  **не описаны в openapi вовсе**. Обёртка middleware на бэке
  применена; добавление этих 10 route'ов в спеку — самостоятельный
  план «openapi coverage gap», не блокер для функционального
  паритета.

- Тесты: backend `go test ./...` зелёный (плана 72.1.49 регрессионный
  тест `idempotency_integration_test.go` остаётся 3/3, дополнительные
  тесты для каждого нового route'а — нерациональны: middleware
  покрыт unit-тестами в `pkg/idempotency/middleware_test.go`).
