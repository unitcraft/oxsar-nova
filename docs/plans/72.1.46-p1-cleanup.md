# План 72.1.46 — P1 cleanup после 72.1.45

**Дата**: 2026-05-01
**Статус**: ✅ Закрыт

**Подплан** [плана 72.1](72.1-post-remaster-stabilization.md) §20.12.
Закрывает реальные P1-пробелы, обнаруженные при аудите 72.1.45.

## Контекст

Аудит после закрытия 72.1.45 первоначально выявил 4 P1-нарушения.
По уточнении вселенной (origin = не-премиум) **2 пункта оказались
false positive** и сняты с плана. Реально устранено 2 пункта.

### Реальные P1 (закрыты)

1. **Sellers per-broker fee не списывается** — `exchange.BuyLot`
   отдавал продавцу полную цену; `fee_percent` из `exchange_settings`
   использовался только в BrokerStats display. Покупатель платит N,
   продавец получал N (без комиссии) → нарушение `Exchange.class.php`
   `EXCH_NEW_PROFIT_TYPE` (always-on в проде).

2. **`GrantWithBattle` нигде не вызывался + 4 INSERT-места минуют
   `Service.Grant`** — таблица `artefact_history` оставалась пустой
   независимо от источника артефакта (expedition/alien/admin/pack).

### False positives (не нарушения)

3. ~~`AlwaysAllowPermit` — merchant-permit gating отключён~~ —
   В legacy `Artefact::getMerchantMark()` зависит от премиум-вселенных
   ([P68.B] simplifications.md). В origin/nova премиум-инфраструктуры
   нет → AlwaysAllowPermit это правильное поведение для не-премиум
   окружения. Снять из «нарушений §20.12».

4. ~~`ExchangeOpts` без broker-permit проверки~~ — Если permit не
   требуется (см. #3), любой user может настраивать свои broker-настройки.
   Это и есть legacy-поведение в не-премиум вселенной. Проверка
   `auth.UserID` достаточна — каждый user редактирует свои settings.
   Снять.

## Definition of Done

- [x] Sellers per-broker fee:
   - `exchange.BuyLot` читает `exchange_settings.fee_percent` для
     `lot.SellerUserID` через `getBrokerSettingsTx` (read-only, не
     создаёт записи).
   - Buyer платит `lot.PriceOxsarit` целиком.
   - Seller получает `price × (100-fee)/100`. Остаток (fee) —
     системная комиссия (в origin/nova нет broker-юзеров отдельно).
   - При `tx==nil` (unit-тесты с fakeRepo) возвращается дефолт без
     SQL-запроса.
- [x] `GrantWithSource` API:
   - `Service.GrantWithSource(ctx, userID, unitID, planetID, source,
     battleReportID)` пишет артефакт с `payload.acquisition_source`.
   - `GrantWithBattle` стал обёрткой над `GrantWithSource(source='battle')`.
- [x] DB trigger `trg_artefacts_user_history` (миграция 0093):
   - Автоматически пишет в `artefact_history` при INSERT в
     `artefacts_user`. Работает для всех 4+ мест:
     `service.go::Grant`, `expedition.go`, `alien.go`,
     `admin/user_profile.go`, `pack.go`.
   - Читает `payload->>'acquisition_source'` (валидируется по enum
     battle/expedition/quest/admin/market) → `source`.
   - Читает `payload->>'battle_report_id'` → `battle_report_id` (uuid).
   - Default source='admin' если нет в payload.
- [x] Wire callers:
   - `expedition.go::expArtefact` → payload `{"acquisition_source":"expedition"}`.
   - `alien/alien.go::grantRandomArtefact` → `{"acquisition_source":"quest"}`.
   - `admin/user_profile.go::Grant` → дефолт source='admin' (без payload).
   - `pack.go` → дефолт source='admin' (переупаковка — операция игрока).
- [x] Тесты:
   - `TestBuyLot_HappyPath` ассерты обновлены: seller получает
     950 (было 1000), система удерживает 50 = 5%.
   - `TestProperty_OxsaritInvariant` инвариант смягчён с «total
     константна» на «total **не растёт**» (oxsarit не появляется
     из ниоткуда; уменьшение ОК — это fee).
- [x] go test (все пакеты ok) + go vet (clean) + tsc (clean).

## Прогресс

- ✅ P1#1: sellers per-broker fee deduction (5% удерживается).
- ✅ P1#2: artefact_history naturally populated (DB trigger 0093 + 4 callers).
- ❌→ P1#3 снят: AlwaysAllowPermit корректен для не-премиум вселенной.
- ❌→ P1#4 снят: проверка broker-permit неактуальна (см. #3).
