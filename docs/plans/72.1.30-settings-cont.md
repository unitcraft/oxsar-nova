## 1. Контекст

Legacy [`Preferences.class.php::updateDeletion`](../../projects/game-legacy-php/src/game/page/Preferences.class.php)
ставит `users.delete = time() + 604800` (7 дней) — flag «удалить
через неделю». Юзер может **отменить** удаление в течение grace-period
через тот же `update_deletion` с `delete=0`. Cron физически удаляет
по истечении.

Origin при ConfirmDeletion немедленно делает soft-delete без grace
(simplification P72.1.5.B).

## 2. Что нужно

### Backend

#### Миграция `0089_users_delete_at.sql`
```sql
ALTER TABLE users
    ADD COLUMN IF NOT EXISTS delete_at timestamptz;

-- Partial-индекс для cron-воркера (поиск expired).
CREATE INDEX IF NOT EXISTS users_delete_at_idx
    ON users (delete_at)
    WHERE delete_at IS NOT NULL AND deleted_at IS NULL;
```

#### `internal/settings/delete.go`

- `ConfirmDeletion` → НЕ выполняет soft-delete сразу, ставит
  `delete_at = now() + 7 days`.
- Возвращает `{ delete_at, ttl_seconds }`.
- Новый endpoint `POST /api/me/deletion/cancel` →
  `users.delete_at = NULL`. Только когда `delete_at IS NOT NULL`,
  иначе 400.
- `MeInfo` отдаёт `delete_at` для UI индикатора.

#### `internal/settings/delete_event.go` (новый)

Cron-обработчик: каждый час сканирует `users WHERE delete_at <
now() AND deleted_at IS NULL`, выполняет soft-delete через
старую логику performDeletion (UPDATE deleted_at, username, email,
alliance_id; cancel market lots; etc).

Альтернатива: event KindAccountDelete с fire_at = delete_at,
вставляется в ConfirmDeletion. Это проще — переиспользуем
event-loop. Использую этот путь.

### Frontend

- `api/me.ts::MeInfo` +поле `delete_at?: string`.
- `api/settings.ts::cancelDeletion()` — POST на новый endpoint.
- `SettingsScreen`:
  - Если `me.delete_at != null && delete_at > now()` →
    показать предупреждение + кнопку «Отменить удаление».
  - При клике на удалить аккаунт show confirmation с текстом
    «Аккаунт будет удалён через 7 дней. Вы можете отменить».
- При успешной ConfirmDeletion вместо logout — показать предупреждение
  с countdown'ом.

### i18n
- `prefs.deletionScheduledTitle`, `deletionScheduledBody` (с
  countdown), `cancelDeletionBtn`, `deletionCancelled`.
