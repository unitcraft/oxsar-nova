## Цель

Привести проект в соответствие с двумя законами:

1. **ФЗ № 436** "О защите детей от информации, причиняющей вред их
   здоровью и развитию" — обязательная возрастная маркировка ("12+",
   "16+", "18+") для информационной продукции, включая онлайн-игры.
   Без маркировки — штраф до 200 тыс. ₽ для ИП/самозанятых, до
   1 млн ₽ для организаций (ст. 6.17 КоАП).

2. **ФЗ № 149** "Об информации, информационных технологиях и о защите
   информации" — обязанность владельцев информационных ресурсов
   удалять противоправный пользовательский контент (UGC: ники,
   названия альянсов, чат, описания планет) по требованию РКН/ФАС/суда
   и в инициативном порядке.

В oxsar-nova есть UGC-поверхности: **никнеймы**, **названия альянсов**,
**чат**, **описания планет/комментарии в боях**. Без модерации
нарушитель регистрирует ник вроде "Героин-2024" → жалоба → блокировка
домена в РКН без предварительного предупреждения.

## Что меняем

### 1. Возрастная маркировка

Решение: **12+** (стратегические игры с боевыми действиями без
натуралистичных сцен жестокости попадают в "12+" по методическим
рекомендациям РКН).

Где разместить:
- **Главная страница портала** (`projects/portal/frontend/src/pages/Home.tsx`)
  — иконка "12+" в шапке/футере.
- **Лендинг игры** в game-nova — иконка "12+" в footer/меню.
- **Страница с описанием игры** в портале — текстовое упоминание
  возрастной категории.
- **Регистрационная форма** — короткая фраза "Игра предназначена для
  лиц старше 12 лет".
- **Метаданные сайта** (Open Graph, schema.org) — поле возрастной
  категории.

Создать `projects/portal/frontend/src/components/AgeRating.tsx` (или
аналог) — общий компонент с иконкой "12+", чтобы менять в одном месте.

### 2. UGC-модерация: автоматическая

**Чёрные списки** для никнеймов и названий альянсов:

- Список запрещённых корней (наркотики, экстремизм, ненормативная
  лексика, имена узнаваемых медиа-персон, государственные символы).
- Источник: открытый список Минцифры
  (https://reestr.rublacklist.net/ — для запрещённых ресурсов;
  + готовые open-source bad-words лексиконы для русского).
- При попытке регистрации/смены ника/создания альянса — отказ с
  сообщением "Имя содержит запрещённое слово".

Реализация:
- `projects/auth/backend/internal/moderation/blacklist.go` — функция
  `IsForbidden(name string) (bool, reason string)`.
- Список — в `configs/moderation/blacklist.yaml`, читается при старте.
- Применяется в auth-service (регистрация), game-nova
  (rename/alliance create).

### 3. UGC-модерация: пользовательские жалобы

В UI:
- Кнопка **"Пожаловаться"** рядом с никнеймом другого игрока, в шапке
  чата, в шапке альянса. Открывает модалку с типом нарушения
  (выбор из списка) и комментарием.
- API endpoint `POST /reports` принимает жалобу, сохраняет в таблицу
  `user_reports`.

Backend:
- Таблица `user_reports` (миграция в auth или game-nova):
  ```
  reporter_user_id, target_user_id, target_type (user/alliance/chat_msg/...),
  target_id, reason, comment, created_at, status (new/resolved/rejected)
  ```
- Админ-панель (план 14, расширяется): просмотр жалоб, действия
  (warn / mute / rename ник / ban / no action).
- Все действия логируются в audit-log (план 14 уже реализован Ф.1).

### 4. UGC-модерация: чат (минимально)

- **Bad-words фильтр на лету**: при отправке сообщения в чат — проверка
  через тот же `blacklist`. Если срабатывает — сообщение режется
  (блокируется или заменяется на `***`).
- **Rate-limit** для предотвращения спама (~10 сообщений в минуту).
- **Лог чата** хранится не менее 6 месяцев на случай претензий
  (требование 149-ФЗ — предоставлять данные по запросу).

### 5. Регламент модерации

`docs/ops/ugc-moderation.md` (новый):
- Кто модерирует жалобы (на старте — лично автор, потом — community manager).
- SLA реакции (24 часа на изначальный ответ, 7 дней на разрешение).
- Эскалация в РКН при нарушениях со стороны злоумышленных
  пользователей (доксинг, угрозы, экстремизм).
- Процедура удаления контента по предписанию РКН/суда.

Это внутренний регламент, но публично можно ссылаться из Terms of
Service (план 47).

## Что НЕ делаем

- Не вводим обязательную верификацию возраста (паспорт/Госуслуги) — для
  "12+" не требуется, достаточно декларации в форме регистрации.
- Не подключаем сторонние сервисы модерации (Yandex SpeechKit,
  Two Hat) — для старта избыточно; добавляем позже при росте
  community.
- Не делаем 18+ контент-модель — упрощает классификацию, исключает
  необходимость возрастной верификации.
- Не делаем сложные ML-фильтры контента — простой blacklist + жалобы +
  ручная модерация достаточно для запуска.

## Тестирование

- Регистрация с ником "героин" — отказ с понятным сообщением.
- Создание альянса с матерным названием — отказ.
- Кнопка "Пожаловаться" работает, жалоба появляется в админке.
- Сообщение в чате с запрещённым словом — режется/блокируется.
- Rate-limit срабатывает на 11-м сообщении за минуту.
- Иконка "12+" видна на главной портала, на странице игры, в шапке
  game-nova.

## Итог

Один frontend-компонент (AgeRating) + один blacklist-конфиг + одна
миграция + один API + один UI-флоу + регламент. 1–2 коммита
(возможно по фазам). Закрывает риски по 436-ФЗ (штрафы за отсутствие
возрастной маркировки) и базовую часть 149-ФЗ (UGC-модерация);
снижает риск блокировки домена за противоправный контент.
