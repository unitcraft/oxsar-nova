## 2. Реальный gap

Frontend [shipyard.ts:35](../../projects/game-nova/frontends/origin/src/api/shipyard.ts#L35)
**уже шлёт** `Idempotency-Key` header (через `idempotencyKey: newIdempotencyKey()`).
Аналогично [repair.ts:31,42,53,72](../../projects/game-nova/frontends/origin/src/api/repair.ts#L31).

**Но**:
1. [`api/openapi.yaml:616-644`](../../projects/game-nova/api/openapi.yaml#L616) не описывает параметр
   `Idempotency-Key` для `POST /api/planets/{id}/shipyard` — спека
   расходится с реальным контрактом, OpenAPI-консьюмер не знает о
   header'е.
2. Backend `shipyardH.Enqueue` [регистрируется без обёртки](../../projects/game-nova/backend/cmd/server/main.go#L524)
   — `idemMW.Wrap` применяется только к `/alien-missions/{mission_id}/buyout`
   и `/planets/{id}/teleport` ([main.go:587,592](../../projects/game-nova/backend/cmd/server/main.go#L587)).
   Сервер игнорирует ключ → дедупа нет, double-submit (двойной клик
   «Построить») создаёт **две задачи в очереди**.

То же справедливо для `POST /api/planets/{id}/repair/disassemble` и
`POST /api/planets/{id}/repair/repair`.

Legacy реализовал anti-double-submit через 5-секундный
[`acquireLock`](../../projects/game-legacy-php/src/game/page/Shipyard.class.php#L382).
В nova паттерн другой (Idempotency-Key + Redis), но для shipyard/repair
он **не подключён**.

## 3. Цель

Закрыть один реальный gap минимальными правками:

- OpenAPI: добавить `Idempotency-Key` параметр к `POST /shipyard`
  + `POST /repair/disassemble` + `POST /repair/repair`.
- Backend: обернуть эти три route'а в `idemMW.Wrap` (план 77 Ф.2,
  middleware уже создан в [main.go:381](../../projects/game-nova/backend/cmd/server/main.go#L381)).
- Тест: добавить регрессионный сценарий «двойной POST с тем же
  ключом не создаёт второй заказ».
- План 72.1.2: обновить snapshot, удалив ⚠ для shipyard/defense.

**За пределами scope**: остальные мутирующие endpoint'ы (research,
fleet, exchange, market, professions, officers, friends, messages,
notepad, chat, alliance) — это объёмная задача плана 77 Ф.2,
оформляется отдельно. Здесь — только то, что 72.1.2 явно объявил
gap'ом.

## 4. Декомпозиция

### A. Backend (Go)

1. [`cmd/server/main.go:524`](../../projects/game-nova/backend/cmd/server/main.go#L524)
   `pr.Post("/planets/{id}/shipyard", shipyardH.Enqueue)` →
   `pr.With(idemMW.Wrap).Post(...)`.
2. [`cmd/server/main.go:539-540`](../../projects/game-nova/backend/cmd/server/main.go#L539)
   `pr.Post("/planets/{id}/repair/disassemble", ...)` и
   `pr.Post("/planets/{id}/repair/repair", ...)` → обернуть тем же.

### B. OpenAPI

Добавить `- $ref: "#/components/parameters/IdempotencyKey"` в
`parameters` блок этих трёх POST'ов (компонент уже определён
[openapi.yaml:3604](../../projects/game-nova/api/openapi.yaml#L3604)).
Добавить `409 Conflict` response (Idempotency-Key reuse with
different body — middleware возвращает именно это).

### C. Тест

`shipyard/handler_test.go` (или новый `idempotency_test.go`):
табличный e2e-тест — дважды POST с одним ключом → второй вернёт
кешированный ответ, в БД одна запись `shipyard_queue`.

Если в shipyard ещё нет интеграционного теста с реальным
middleware — провести через чистый `idempotency.NewMiddleware(rdb, 0)`
с in-memory redisCmd-заглушкой (паттерн из
[`pkg/idempotency/middleware_test.go`](../../projects/game-nova/backend/pkg/idempotency/middleware_test.go)).
