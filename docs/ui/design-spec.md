## 0. Индексная страница — выбор вселенной

До авторизации и сразу после неё (если игрок ещё не выбрал вселенную)
показывается **страница выбора вселенной**. Это первое, что видит
пользователь при открытии сайта.

### 0.1 Концепция

Каждая вселенная — независимый игровой сервер со своими настройками
скорости, правилами и сообществом. Страница должна дать игроку достаточно
информации для выбора и не перегрузить лишним.

### 0.2 Макет

**Шапка страницы** (до авторизации):
```
┌────────────────────────────────────────────────────────────────┐
│  ✦ OXSAR-NOVA                           [Войти]  [Регистрация] │
└────────────────────────────────────────────────────────────────┘
```

**После авторизации** в шапке: имя игрока + кнопка "Выйти".

**Героя блок** (полная ширина, высота 30vh, космический фон):
```
┌────────────────────────────────────────────────────────────────┐
│                                                                │
│          ✦  OXSAR-NOVA  ✦                                      │
│     Космическая стратегия реального времени                    │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

**Список вселенных** — карточки в grid:

```
┌─────────────────────────────┐  ┌─────────────────────────────┐
│  🌌 Вселенная «Орион»       │  │  🌌 Вселенная «Андромеда»   │

│  Дебри:      30%            │  │  Дебри:      30%            │

│  Начало:     2025-11-01     │  │  Начало:     2024-03-15     │

│                             │  │                             │

│                             │  │  Планета:    [1:45:8]      │

└─────────────────────────────┘  └─────────────────────────────┘
```

### 0.3 Карточка вселенной — детали

**Обязательные поля:**
- Название вселенной (уникальное, с иконкой/цветом)
- Скорость строительства (×N)
- Скорость экономики / добычи ресурсов (×N)
- Скорость флота (×N)
- % обломков от уничтоженных кораблей
- АКС (совместные атаки): Да / Нет
- Дата старта сервера
- Текущее число активных игроков (обновляется при загрузке страницы)

**Статус игрока в этой вселенной** (только для авторизованных):
- "Не зарегистрирован" — кнопки "Подробнее" + "Войти" (создаст аккаунт)
- "Активен" — координаты основной планеты, кнопка "Играть"
- "В отпуске" — иконка 🌙, кнопка "Играть"
- "Забанен" — иконка 🚫, без кнопки входа

**Кнопки:**
- **Подробнее** — разворачивает карточку / открывает модальное окно с
  полным описанием правил вселенной, топ-5 игроков, новостями сервера.
- **Войти / Играть** — переход в игру (если не авторизован — сначала
  форма входа/регистрации, потом вход).

### 0.4 Сортировка и фильтрация

Над сеткой карточек:
```
┌─────────────────────────────────────────────────────────┐
│  Показать: [Все ▼]   Сортировка: [По игрокам ▼]  🔍    │
└─────────────────────────────────────────────────────────┘
```

Фильтры: Все / Мои вселенные / Новые (< 30 дней) / Быстрые (×3+).  
Сортировка: По игрокам / По дате / По скорости.

### 0.6 Адаптивность

- Десктоп (lg+): 3 карточки в ряд.
- Планшет (md): 2 карточки в ряд.
- Телефон (xs/sm): 1 карточка в ряд, полная ширина.
- Модальное окно на телефоне: fullscreen bottom sheet.
- Hero-блок на xs: высота 20vh, уменьшенный шрифт.

### 0.7 Неавторизованный пользователь

- Страница полностью доступна без входа.
- Статус игрока в карточках скрыт.
- Кнопки: "Войти" и "Зарегистрироваться" (открывают модальное окно
  с формой прямо на этой странице — без перехода).
- После успешной авторизации страница обновляется и показывает
  статус игрока во всех вселенных.

## 1. Философия и стиль

### 1.1 Концепция

Oxsar-nova — космическая стратегия реального времени. Игрок управляет
планетами, строит флот, атакует соперников и торгует ресурсами. UI должен
передавать ощущение живой галактики: ресурсы текут, флоты летят, таймеры
тикают — всё без перезагрузки страницы.

**Ключевое требование**: всё, что меняется со временем, должно меняться
на экране в реальном времени средствами фронтенда (JS-тикеры, анимации
CSS, интерполяция значений). API вызывается только для получения начального
состояния и отправки команд.

### 1.2 Визуальный стиль

- **Тема**: тёмный космос. Фон — глубокий чёрно-синий (#050d18), с тонкой
  анимированной звёздной текстурой (CSS `background`, не canvas — чтобы не
  нагружать GPU).
- **Акцент**: холодный голубой (#38bdf8) для интерактивных элементов,
  янтарный (#f59e0b) для предупреждений, красный (#ef4444) для опасности,
  изумрудный (#10b981) для успеха/профицита.
- **Типографика**: системный sans-serif для текста, `JetBrains Mono` / `Fira Code`
  для чисел и координат — моноширинный шрифт делает числа стабильными при
  изменении (не прыгают по ширине).
- **Панели**: `backdrop-filter: blur(12px)` + полупрозрачный фон
  (`rgba(10, 20, 40, 0.85)`) — эффект стекла (glassmorphism). Тонкая рамка
  `1px solid rgba(56, 189, 248, 0.15)`.
- **Иконки**: пиксельные спрайты юнитов сохраняются как часть стиля игры
  (ностальгия), но отображаются на современных карточках с hover-эффектами.
- **Анимации**: плавные, не дольше 300ms. Transition-group для списков.
  Никаких тяжёлых 3D-трансформаций.
- **Скругления**: `border-radius: 8px` для карточек, `12px` для модалей,
  `4px` для кнопок.

### 1.3 Сетка и отзывчивость

Дизайн полностью адаптивен: **весь функционал игры доступен и удобен на
любом устройстве** — от вертикального смартфона (320px) до широкого
монитора (2560px). Горизонтальный скролл и "рекомендуем десктоп"
недопустимы.

**Контрольные точки (breakpoints):**

| Имя | Диапазон | Компоновка |
|-----|----------|------------|
| `xs` | 320–479px | Телефон вертикальный |
| `sm` | 480–767px | Телефон горизонтальный / планшет мини |
| `md` | 768–1023px | Планшет |
| `lg` | 1024–1279px | Ноутбук |
| `xl` | 1280–1535px | Десктоп |
| `2xl` | 1536px+ | Широкий монитор |

**Базовые принципы:**
- `fluid grid`: колонки через CSS Grid с `minmax()` и `auto-fill` — сетка
  сама перестраивается, не нужно писать breakpoint для каждой карточки.
- `container queries` там, где компонент должен адаптироваться к своему
  контейнеру (карточка здания в сайдбаре vs на полном экране).
- Все tap-цели: минимум **44×44px** (WCAG 2.5.5) — кнопки действий
  в таблицах не сжимаются меньше.
- Никаких `position: fixed` элементов на xs/sm кроме шапки и bottom-nav.

### 1.4 Тематичность и качество интерфейса

#### Общий принцип

Каждый элемент UI должен органично вписываться в тематику космической
стратегии. Интерфейс — это часть игрового мира, а не форма на сайте
государственных услуг. Игрок должен чувствовать, что управляет
межзвёздной империей, а не заполняет таблицы.

#### Нативные браузерные диалоги — запрещены

Следующие методы **не используются нигде в приложении**:

| Запрещено | Причина | Замена |
|-----------|---------|--------|
| `window.confirm()` | Системный вид, выбивает из атмосферы | Кастомный модальный диалог |
| `window.alert()` | Блокирует UI, нет стилизации | Toast-уведомление или inline-сообщение |
| `window.prompt()` | Нет стилизации, неудобно на мобильных | Inline input в карточке или модал |
| `<select>` нативный | Не стилизуется на iOS/Android | Кастомный dropdown/combobox |
| `<input type="date">` нативный | Разный вид в разных браузерах | Кастомный date picker или text input с маской |
| `browser confirm перед unload` | `beforeunload` диалог — нативный | Кастомное предупреждение внутри страницы |

#### Кастомный диалог подтверждения

Используется везде, где требуется подтверждение опасного действия
(удаление сообщения, роспуск альянса, отзыв флота и т.п.).

Визуально — тёмная модальная панель в стиле игры:

```
        ┌──────────────────────────────────────┐
        │  ⚠️  Подтверждение                   │
        │                                      │
        │  Удалить сообщение?                  │
        │  Это действие необратимо.            │
        │                                      │
        │  [ Отмена ]        [ Удалить ]       │
        └──────────────────────────────────────┘
```

- Фон: `backdrop-filter: blur` + затемнение overlay.
- Кнопка подтверждения: красная (опасное действие) или голубая (нейтральное).
- Кнопка отмены: всегда слева, нейтральный цвет.
- Закрывается по Escape и клику на overlay.
- Анимация: fade-in + scale 0.95→1.0 за 200ms.
- На телефоне: bottom sheet вместо центрального модала.

#### Тематические тексты и иконки

- Тексты кнопок и подсказок — в терминологии игры:
  "Отправить флот" (не "Отправить"), "Распустить альянс" (не "Удалить"),
  "Покинуть вселенную" (не "Выйти из аккаунта").
- Иконки — космическая тематика: ракеты, планеты, звёзды, щиты, мечи.
  Не использовать generic иконки вроде "карандаш для редактирования" там,
  где есть тематическая альтернатива.
- Пустые состояния — с атмосферным текстом:
  "Ваш флот ещё не отправился в открытый космос" вместо "Нет данных".
- Ошибки — не "Error 403", а "Доступ запрещён. Только командиры альянса
  могут выполнить это действие."

#### Все интерактивные компоненты — кастомные

| Компонент | Реализация |
|-----------|------------|
| Выпадающий список | Кастомный с анимацией, поиском если > 8 пунктов |
| Checkbox | SVG-стилизованный, анимация checkmark |
| Radio | Кастомный с pulse-анимацией при выборе |
| Range (слайдер) | Стилизованный трек + thumb с tooltip текущего значения |
| Input number | С кнопками +/− по краям, запрет нечисловых символов |
| Tabs / переключатели | Анимированный indicator (sliding underline или pill) |
| Tooltip | CSS + JS, 200ms delay, тематический фон |
| Progress bar | Анимированные полоски, glow-эффект |
| Badge / счётчик | Pulse-анимация при появлении нового значения |
| Модальные окна | Кастомные, с backdrop, фокус-trap, ESC-закрытие |
| Drag & drop (если будет) | Кастомный, с ghost-элементом в стиле игры |

#### Единство визуального языка

- Все карточки, кнопки, панели — из одной дизайн-системы. Нет "серых
  браузерных" элементов рядом с красиво оформленными.
- Состояния: каждый интерактивный элемент имеет `default`, `hover`,
  `active`, `disabled`, `focus-visible` — все стилизованы.
- Курсор: `cursor: pointer` на кликабельных элементах, `cursor: not-allowed`
  на disabled (не просто opacity).
- Переходы: все изменения состояний через CSS `transition`. Мгновенных
  скачков нет.

## 2. Глобальная шапка (Header)

Закреплена вверху (`position: sticky; top: 0; z-index: 100`) на всех
экранах.

**Десктоп (lg+), высота 56px:**
```
┌─────────────────────────────────────────────────────────────────────┐
│ [🪐 Планета ▼]   🟠 1 234 567  💎 892 341  💧 45 123  ⚡ +340   [🔔][👤]│
└─────────────────────────────────────────────────────────────────────┘
```

**Телефон (xs/sm), высота 48px:**
```
┌──────────────────────────────────────────┐
│ [☰]  [🪐 Планета ▼]         [🔔] [👤]  │
└──────────────────────────────────────────┘
```
Ресурсы убраны из шапки → в отдельную **resource-bar** (липкую полоску
под шапкой, высота 32px, горизонтальный скролл):
```
┌──────────────────────────────────────────────────────────┐
│  🟠 1 234 567  ·  💎 892 341  ·  💧 45 123  ·  ⚡ +340  │  ← скроллится
└──────────────────────────────────────────────────────────┘
```

### 2.1 Переключатель планеты

- Текущая планета: название + координаты `[1:45:8]`.
- Клик → выпадающий список всех планет игрока с иконкой планеты/луны,
  координатами, текущим запасом ресурсов (снапшот).
- Активная планета выделена голубым.

### 2.2 Строка ресурсов

Четыре показателя всегда видны в шапке:

| Ресурс | Иконка | Поведение |
|--------|--------|-----------|
| Металл | 🟠 или custom svg | Тикер: +N в секунду |
| Кремний | 💎 | Тикер: +N в секунду |
| Водород | 💧 | Тикер: +N в секунду |
| Энергия | ⚡ | Статично: `+N` (профицит) или `-N` (дефицит, красный) |

**Тикер** (реализация):
- При загрузке страницы/смене планеты: API возвращает `{ metal, silicon,
  hydrogen, rate_per_second_metal, ... }`.
- `setInterval` каждые 200ms добавляет `rate * 0.2` к локальному состоянию.
- Каждые 60s — фоновый refetch для коррекции дрейфа.
- **Cap**: если текущее > 95% вместимости хранилища — число становится
  оранжевым, tooltip "Хранилище почти полно". При 100% — красным,
  производство не накапливается.

### 2.3 Уведомления

- 🔔 Колокол с бейджем (количество непрочитанных сообщений).
- При входящей атаке: колокол мигает красным, title-tab браузера меняется
  на "⚠️ Атака! — oxsar-nova", browser Notification API (с разрешения).
- Клик → выпадающая панель последних 5 событий (атаки, завершение строек,
  прибытие флотов).

## 3. Навигация (адаптивная)

### 3.1 Десктоп (lg+) — левый сайдбар

Ширина 200px, sticky, высота 100vh минус шапка.

```
┌──────────────┐
│ 🏠 Обзор     │  ← активный: полоска 3px + светлый фон
│ 🏗 Постройки │
│ 🔬 Исследов. │
│ 🚀 Верфь     │
│ 🔧 Ремонт    │
├──────────────┤
│ 🌌 Галактика │
│ 🛸 Флот      │
│ 🚀 Ракеты    │
├──────────────┤
│ 💱 Рынок     │
│ 💎 Артефакты │
│ 🏪 Арт-рынок │
├──────────────┤
│ 🏆 Рейтинг   │
│ 🥇 Достижен. │
│ 🎓 Туториал  │
│ 🤝 Альянс    │
│ 💬 Чат       │
└──────────────┘
```

При ширине 1024–1279px (lg) — сайдбар сворачивается до 56px (только иконки,
tooltip с названием по hover).

### 3.2 Планшет (md, 768–1023px) — drawer

Сайдбар скрыт по умолчанию. Кнопка ☰ в шапке открывает drawer
(slide-in слева, overlay с backdrop blur). Закрывается по клику вне
или по свайпу влево.

### 3.3 Телефон (xs/sm, < 768px) — bottom navigation bar

Сайдбар и drawer убраны. Вместо них — закреплённая нижняя панель
(высота 56px, `position: fixed; bottom: 0`):

```
┌─────────────────────────────────────────────────────┐
│  🏠       🌌       🛸       💬       ···            │
│ Обзор  Галакт.  Флот    Чат    Ещё▲               │
└─────────────────────────────────────────────────────┘
```

- 4 самых частых пункта + кнопка "Ещё" → sheet снизу со всем меню.
- Активный пункт: цветная иконка + подчёркивание сверху.
- Bottom sheet ("Ещё") — slide-up панель на 80% высоты экрана, скроллируемая,
  с разделителями по группам.
- Контент страницы имеет `padding-bottom: 56px` чтобы не перекрывался.

### 3.4 Общие правила навигации

- Активный пункт — всегда виден (scroll to active item если нужно).
- Счётчик непрочитанных у "Чат" и "Сообщения" — красный бейдж 16px.
- Переходы между экранами: fade 150ms.

## 4. Экран: Обзор (Overview)

Главный дашборд. Должен давать полный статус планеты с первого взгляда.

### 4.1 Структура

```
┌─ Текущая планета ──────────────────────────────────────────────────┐
```

### 4.2 Прогресс-бары активных процессов

- Ширина бара вычисляется из `(elapsed / total_duration) * 100%`.
- Анимация: CSS `@keyframes` сдвигает паттерн (полоски) внутри бара
  для ощущения движения — как индикатор загрузки, но с реальным прогрессом.
- Таймер обратного отсчёта тикает каждую секунду (`setInterval`).
- По завершении: бар исчезает с анимацией fade + slide, появляется toast
  "✅ Рудник металла ур.7 построен!".

### 4.3 Входящая атака

- Красный блок с мигающей иконкой ⚠️, координатами атакующего, обратным
  отсчётом до прибытия.
- Browser Notification при первом обнаружении.
- Кнопка "Подробнее" → переход в сообщения / шпионаж.

## 16. Состояния загрузки и ошибок

- **Skeleton screens**: вместо спиннеров — анимированные серые блоки
  (`shimmer` эффект) в форме карточек. Контент не прыгает при загрузке.
- **Ошибка API**: inline красный баннер под тем элементом, который вызвал
  ошибку. Не модальный диалог.
- **Пустые состояния**: иллюстрация (SVG космос) + текст.
  "Нет флотов в пути" + кнопка "Отправить флот".
- **Оптимистичные обновления**: действие отражается мгновенно (чат,
  отправка флота), откатывается при ошибке с toast.

## 18. Адаптивность по экранам и кроссбраузерность

### 18.1 Общие правила адаптации контента

**Карточки** (постройки, корабли, офицеры, артефакты):
- Десктоп: grid `repeat(auto-fill, minmax(280px, 1fr))` — 3–4 в ряд.
- Планшет: 2 в ряд.
- Телефон: 1 в ряд, карточка на всю ширину.
- Внутри карточки характеристики при узкой ширине переходят
  из двух колонок в одну (`flex-wrap: wrap`).

**Таблицы** (галактика, флоты, рейтинг, сообщения):
- Десктоп: стандартная таблица `<table>`.
- Планшет/телефон: каждая строка превращается в карточку
  (CSS `display: block` на `td`, `data-label` атрибут с именем колонки
  через `::before`). Пример:
  ```
  ┌──────────────────────────────────┐
  │ Позиция:   3                     │
  │ Планета:   🌍 Nexus  [🔭][⚔️]  │
  │ Владелец:  StarLord  🟡 4ч      │
  │ Обломки:   M: 120k  Si: 3k      │
  └──────────────────────────────────┘
  ```
- Кнопки действий в строке — минимум 44×44px, располагаются в ряд
  в нижней части карточки.

**Формы** (отправка флота, рынок, верфь):
- Все input имеют `width: 100%` в мобильном режиме.
- Wizard флота на телефоне: шаги занимают полный экран, навигация
  "Назад / Далее" закреплена снизу (sticky).
- Слайдеры скорости — достаточно широкие для пальца (высота 36px).

**Чат**:
- Занимает весь экран на xs/sm (без сайдбара).
- Emoji picker — 5×4 grid, кнопки 44px, чтобы попасть пальцем.
- Input прилипает к bottom (над bottom nav bar).

**Боевые отчёты и сообщения**:
- Двухколонный inbox (список + детали) → на телефоне: сначала список,
  тап на сообщение → полноэкранный вид деталей, кнопка "← Назад".

**Галактика**:
- Навигация G/S/P — крупные input (48px высота) с кнопками +/−.
- Стрелки ← → ↑ ↓ — размером 44px, расположены в одном ряду.
- Таблица → карточки (описано выше).

**Overview / дашборд**:
- Прогресс-бары активных процессов — всегда полная ширина.
- Карточки планет → вертикальный список на телефоне.

### 18.2 Touch-специфика

- Все интерактивные элементы: `min-height: 44px; min-width: 44px`.
- Hover-эффекты работают только на устройствах с указателем
  (`@media (hover: hover) and (pointer: fine)`). На touch — нет.
- Swipe-to-dismiss для toast-уведомлений (touch event).
- Drawer и bottom sheet закрываются свайпом.
- Pinch-to-zoom не заблокирован (`<meta name="viewport" content="width=device-width, initial-scale=1">` без `user-scalable=no`).
- Поля input при фокусе не вызывают скачок страницы: контент не прячется
  за открывшуюся клавиатуру (`visualViewport` API для позиционирования
  sticky элементов).

### 18.3 Производительность на мобильных

- Анимации только через `transform` и `opacity` — не трогают layout.
- `will-change: transform` только на элементах с активной анимацией,
  снимается после окончания.
- Звёздный фон (CSS animation) отключается на xs (`@media (prefers-reduced-motion: reduce)` и `@media (max-width: 479px)`).
- Тикер ресурсов: `requestAnimationFrame` вместо `setInterval` для плавности.
- Lazy-load изображений юнитов (`loading="lazy"`).

### 18.4 Кроссбраузерность

**Целевые браузеры** (поддержка обязательна):

| Браузер | Минимальная версия |
|---------|-------------------|
| Chrome / Edge (Chromium) | последние 2 версии |
| Firefox | последние 2 версии |
| Safari (macOS + iOS) | 16+ |
| Samsung Internet | 21+ |
| Chrome Android | последние 2 версии |

**Потенциальные проблемы и решения:**

| Фича | Проблема | Решение |
|------|----------|---------|
| `backdrop-filter: blur()` | Не работает в Firefox без флага до v126 | Fallback: `background: rgba(10,20,40,0.95)` через `@supports not (backdrop-filter: blur(1px))` |
| `container queries` | Safari 15 не поддерживает | Полифил `container-query-polyfill` или CSS Grid вместо CQ там где Safari важен |
| CSS `:has()` | Firefox < 121 | Не используем для критичного поведения; применяем только как enhancement |
| `color-mix()` | Safari 15 | CSS custom properties как fallback |
| `scrollbar-gutter` | Safari | `overflow-y: scroll` как fallback чтобы контент не прыгал |
| `dvh / svh / lvh` | Safari 15, Chrome 107- | Fallback `100vh` везде где нет scroll-проблем |
| WebSocket | Все целевые браузеры | — (нативная поддержка) |
| `Notification API` | iOS Safari < 16.4 | Feature detect + graceful degradation (без уведомлений) |
| `Intl.NumberFormat` | Все целевые | — (нативная поддержка) |

**Тестирование:**
- BrowserStack / LambdaTest для реальных устройств.
- Обязательно: iPhone SE (375px), iPhone 14 (390px), iPad (768px),
  MacBook (1280px), Full HD (1920px).
- Автоматические тесты: Playwright с viewport-профилями для каждого
  breakpoint.

### 18.5 Типография на разных экранах

Масштабирование через CSS `clamp()`:

```css
--ox-text-base:  clamp(13px, 1.5vw, 15px);
--ox-text-sm:    clamp(11px, 1.2vw, 13px);
--ox-text-lg:    clamp(15px, 1.8vw, 18px);
--ox-text-xl:    clamp(18px, 2.2vw, 24px);
--ox-text-num:   clamp(12px, 1.4vw, 14px);  /* числа в шапке */
```

Числа ресурсов в шапке на xs — сокращение: `1 234 567` → `1.23M`.

### 18.6 Ориентация экрана (landscape на телефоне)

При `orientation: landscape` на xs/sm:
- Шапка высотой 40px (уменьшена).
- Bottom nav скрывается, появляется компактный left rail (44px) с иконками.
- Контент занимает оставшееся пространство без скролла по вертикали
  там, где это возможно (чат, галактика).
