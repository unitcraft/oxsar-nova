# Активные параллельные Claude Code сессии

**Лайв-документ.** Каждый агент при старте задачи дописывает строку,
при finish — удаляет.

## Зачем

В проекте oxsar-nova одновременно работает 3-5 Claude Code сессий
на одном репозитории. Без явной координации:
- Один агент видит правки другого как «неизвестные изменения», тратит
  время на анализ.
- Конфликты в общих файлах (`cmd/server/main.go`, `pkg/metrics/metrics.go`,
  `openapi.yaml`, `migrations/00NN_*`) случаются регулярно.
- Прецеденты захвата чужих файлов: 4 раза за последние 2 дня
  (см. memory `feedback_parallel_session_check.md`).

Этот документ даёт **общую карту**: кто что делает, какие файлы
трогает, до какого момента.

## Правила

### При старте сессии

Каждый Claude Code агент **сразу после прочтения промпта** дописывает
строку в раздел «Активные сессии» этой страницы:

```markdown
| Slot | План / Задача | Файлы (паттерны) | Старт | Ожидаемый коммит |
|---|---|---|---|---|
| 1 | План 65 Ф.6 KindTeleportPlanet | internal/event/, internal/billing/client/, openapi.yaml | 2026-04-29 10:30 | feat(event-loop): KindTeleportPlanet |
```

Slot — натуральное число от `1` без ведущих нулей (`1`, `2`, …).
Каждая новая сессия берёт следующий свободный номер. Не переиспользуй
номера завершённых сессий — нумерация сквозная, чтобы при координации
было однозначно («слот 3 конфликтует с 5»).

### При finish сессии

Агент **удаляет свою строку** перед последним коммитом. Если строка
осталась — это сигнал что сессия зависла или забыла удалить.

### Перед `git add`

Агент сверяет свой список файлов в commit с **другими активными
строками** этой таблицы. Если видит пересечение — останавливается и
спрашивает пользователя.

## Примеры пересечений (история, для обучения)

### Прецедент 1: миграции 0078 (2026-04-28)

- Я (главная сессия) добавил `0078_notepad_check_length.sql` для
  R15 пост-фикса плана 69.
- Агент-67 параллельно создал `0078_alliance_leadership_codes.sql`
  для transfer-leadership.
- Конфликт обнаружен только при попытке коммита агента-67.
- Решение: переименование в `0079_alliance_leadership_codes.sql`.

**Урок:** при создании миграции — заблокировать номер сразу записью
в active-sessions.md под колонкой «Файлы»: например,
`migrations/0079_*`. Тогда другой агент возьмёт 0080.

### Прецедент 2: захват чужих файлов в коммит (4 раза за 2 дня)

При `git commit -m "..."` без `-- path1 path2` я брал чужие
staged-файлы, потому что не делал `git diff --cached --name-only`
непосредственно перед `git commit`.

**Решение в memory** (`feedback_parallel_session_check.md`):
ВСЕГДА `git commit -m "..." -- path1 path2 path3` с двойным тире.

Active-sessions помогает увидеть **до старта**, чьи файлы могут
оказаться в `git status` — и не путать их со своими.

## Как читать таблицу

- **Slot пуст** → можно стартовать любую задачу безопасно.
- **Slot заполнен** → сверь свои файлы с колонкой «Файлы (паттерны)».
  Если пересечение — выбери другие файлы или дождись освобождения слота.
- **Slot заполнен >24h** → возможно зомби-сессия (агент завис, забыл
  удалить строку). Проверь у пользователя.
