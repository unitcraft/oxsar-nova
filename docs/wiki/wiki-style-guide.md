# Wiki Style Guide

Правила оформления страниц вики oxsar-nova. Применяй при любой работе
над `docs/wiki/` и `configs/wiki-descriptions.yml`.

---

## Язык и терминология

- **Только русский.** Никаких английских слов без необходимости (не `SI`,
  `H`, `geometric`, `rapidfire`, `mission=10`).
- Ресурсы: **металл**, **кремний**, **водород**, **энергия**.
- Параметры юнита: **Атака**, **Корпус**, **Щит**
  (не Attack/Shell/Shield/HP).
- Числа форматировать с пробелами: `500 000`, не `500000`.
- Время: `10 мин`, `2 ч 30 мин` (не `600 секунд`, не `0.041 ч`).
- Коэффициент роста стоимости: «каждый следующий уровень дороже в N раза»
  (не «геометрическая прогрессия»).

## Перекрёстные ссылки

### Ссылки на юниты — `[[unit:N]]`
Рендерится как иконка + имя юнита, кликабельная ссылка на страницу юнита.
Использовать всегда вместо написания имени вручную:

```markdown
Стройте [[unit:1]] до уровня 5.
```

ID юнитов: металл=1, кремний=2, водород=3, солнечная=4, термояд=5,
фабрика роботов=6, верфь=8, нанофабрика=7, склады=9/10/11,
лаборатория=12, астрофизика=13, ...
(полный список в `frontend/src/api/catalog.ts`)

### Ссылки на разделы вики — `[[wiki:category/slug|подпись]]`
Рендерится как синяя жирная ссылка, навигация внутри SPA (без перезагрузки):

```markdown
Подробнее — [[wiki:combat/index|на странице боя]].
```

Категории: `getting-started`, `resources`, `buildings`, `ships`,
`defense`, `research`, `missions`, `expedition`, `combat`.

## Структура страницы юнита (генерируется wiki-gen)

Порядок разделов строго фиксирован:

1. `## Описание` — prose из `configs/wiki-descriptions.yml` (`long`)
2. `## Требования` — список `[[unit:N]] — уровень X` из `configs/requirements.yml`
3. `## Стоимость и время` — первый уровень + коэффициент + базовое время
4. `## Параметры` — только для кораблей и обороны (Атака/Корпус/Щит/…)
5. `## Скорострельность по целям` — только если есть rapidfire
6. `## Уязвим к скорострельности` — только если есть
7. `## Стоимость по уровням` — таблица до 10-го уровня

Frontmatter обязательно содержит: `title`, `category`, `entity_key`, `unit_id`.

---

## Структура рукописной страницы (index.md категории)

Frontmatter: `title`, `category`, `order` (число для сортировки в сайдбаре).

**Не дублировать `# Заголовок`** в теле: WikiScreen уже рендерит `title`
из frontmatter как `<h1>`. Тело начинается сразу с текста или `##`.

Обязательный раздел в конце:
```markdown
## Что читать дальше

- [[wiki:category/slug|Название]] — одна фраза зачем.
```

---

## Описания юнитов (configs/wiki-descriptions.yml)

- `short`: одна строка, без точки в конце, для подсказок.
- `long`: несколько абзацев markdown, `[[unit:N]]` для перекрёстных ссылок.
- Для зданий-ускорителей (robotic_factory, nano_factory, defense_factory):
  обязательно указывать **формулу** ускорения и **практический пример**
  («при уровне 10 — в 11 раз быстрее»).
- Не ссылаться на «OGame» явно в тексте — аналогии допустимы имплицитно.

## Рендерер md.ts — поддерживаемый синтаксис

| Элемент | Синтаксис |
|---|---|
| Заголовки | `# H1`, `## H2`, `### H3` |
| Жирный | `**текст**` |
| Курсив | `*текст*` |
| Код | `` `код` `` |
| Ссылка | `[текст](url)` |
| Список | `- пункт` или `1. пункт` |
| Таблица | `\| A \| B \|` + `\|---\|---\|` |
| Цитата | `> текст` |
| Разделитель | `---` |
| Юнит | `[[unit:N]]` |
| Вики-ссылка | `[[wiki:cat/slug\|подпись]]` |

Списки поддерживают перенос строк: продолжение пункта — отступ ≥ 2 пробела.

## Генерация страниц (wiki-gen)

```bash
cd backend/cmd/tools/wiki-gen && go run .
```

Регенерирует все страницы в `docs/wiki/ru/` из:
- `configs/buildings.yml`, `configs/ships.yml`, `configs/defense.yml`,
  `configs/research.yml`
- `configs/wiki-descriptions.yml` — описания
- `configs/requirements.yml` — требования

Рукописные страницы (`getting-started/`, `resources/`, `combat/`,
`missions/`, `expedition/`) wiki-gen **не трогает** — они редактируются вручную.

## Формулы и код в вики

- **Все переменные в формулах — по-русски.** Не `shieldDamageFactor`, а `фактор_урона_щита`.
  Исключение: общепринятые латинские однобуквенные обозначения (`A`, `H`, `S`, `N`).
- Блоки кода — fenced (` ``` `), без указания языка. Рендерер применяет
  подсветку автоматически: числа зелёные, операторы жёлтые, функции фиолетовые,
  скобки серо-голубые, комментарии `//` серые курсивом.
- Комментарии в формулах через `//` — кратко, по-русски.
- Не упоминать «порт из oxsar2-java» и внутренние имена Java-классов (`Units.processAttack`).
- Термины: не `Ablation`, не `damaged` — **частичные потери**, **повреждённый юнит**.
- Ссылку на исходный код давать в разделе `## Исходный код` в конце страницы —
  список функций в `backend/internal/battle/`, а не inline в тексте.

---

## Что НЕ делать

- Не писать «геометрическая прогрессия», «SI», «H», «Attack», «Shell», «rapidfire».
- Не дублировать `# Заголовок` если есть `title` в frontmatter.
- Не использовать относительные пути в ссылках (`../resources/`) —
  только `[[wiki:...]]` для внутренней навигации.
- Не вставлять скриншоты внешних игр (OGame и пр.).
- Не описывать WHAT делает код (это уже видно) — только WHY неочевидно.
