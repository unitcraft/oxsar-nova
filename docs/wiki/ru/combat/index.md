Бой пошаговый — до **6 раундов**. В каждом раунде стороны одновременно
стреляют по снимку, сделанному в начале раунда. Бой детерминирован при
фиксированном `seed` — повторный расчёт даёт идентичный результат.

## Основные параметры юнита

- **Атака** (`A`) — урон одного юнита за раунд.
- **Корпус** (`H`) — прочность. Накапливает урон после пробития щита;
  при `H ≤ 0` юнит уничтожается.
- **Щит** (`S`) — регенерирует в начале каждого раунда до 100 %.
- **Front** — приоритет цели (выше → атакуют первым). Большинство юнитов
  имеют front=10. У [[unit:102]] front=8 — бьёт раньше обычных.

## Влияние технологий

Каждый уровень исследования даёт **линейный** бонус +10 %:

```
эфф_атака  = A × (1 + тех_оружие × 0.10)
эфф_корпус = H × (1 + тех_броня × 0.10)
эфф_щит    = S × (1 + тех_щит × 0.10)
```

Для [[unit:15]] / [[unit:16]] / [[unit:17]] на уровне 10 — +100 % к атаке /
щиту / корпусу.

## Алгоритм одного выстрела

Один стек из `N` юнитов наносит `N × эфф_атака` урона за раунд,
распределяемого по целям пропорционально их `вес = 2^Front × количество`.

Когда выстрел доходит до цели, применяется **трёхступенчатый
алгоритм щита**:

### 1. Порог игнора щита

```
порог_игнора = базовый_щит / 100
```

Если `атака ≤ порог_игнора`, выстрел **полностью поглощается щитом**,
корпус не страдает.

Важное исключение: для **планетарных щитов** ([[unit:49]] и [[unit:50]])
`порог_игнора = 0` — любая атака пробивает их (так в legacy).

`порог_игнора` считается от **базового** щита, **без** tech-множителя.
Это значит: тех_щит повышает абсорбцию щита, но не делает его
абсолютным барьером.

### 2. Распределение выстрелов

В начале раунда щит полный → `фактор_урона_щита = 1.0` →
`фактор_разрушения = 0.01` (минимум clamp). Это значит, что в первом
ударе **99 % выстрелов идут в щит**, **1 % проходит к корпусу**. По мере
истощения щита `фактор_разрушения` растёт, и больше выстрелов проходят
к корпусу.

```
фактор_урона_щита  = щит_раунда / щит_начало_раунда
фактор_разрушения  = clamp(1 - фактор_урона_щита, 0.01, 1.0)

выстрелы_в_щит    = ceil(выстрелы × (1 - фактор_разрушения))
выстрелы_в_корпус = выстрелы - выстрелы_в_щит
```

### 3. Расчёт урона

**По щиту** (если `атака > порог_игнора`):

```
урон_щита = min(
  атака × выстрелы_в_щит,    // поток силы
  щит   × выстрелы_в_щит,    // cap: один выстрел не сильнее щита одного юнита
  щит_раунда                  // cap: щит не уходит в минус
)
щит_раунда -= урон_щита
```

**По корпусу**:

```
урон_корпуса = min(
  атака   × выстрелы_в_корпус,
  корпус  × выстрелы_в_корпус,
  корпус_раунда
)
корпус_раунда -= урон_корпуса
```

### 4. Регенерация

В начале каждого следующего раунда:

```
щит_раунда    = N × эфф_щит   // 100 % регенерация
корпус_раунда остаётся как есть // корпус не восстанавливается
```

## Скорострельность (rapidfire)

Некоторые юниты получают дополнительные выстрелы за раунд против
определённых целей:

```
выстрелы_итого = количество × rapidfire[стрелок][цель]
```

Например, [[unit:33]] имеет ×6 против [[unit:31]] и ×10 против
[[unit:43]] — за один раунд один крейсер делает 6 выстрелов по
лёгким истребителям или 10 по ракетным установкам.

[[unit:42]] имеет rapidfire почти против всего флота — её таблица:
×250 против [[unit:29]] / [[unit:30]] / [[unit:37]] / [[unit:38]] /
[[unit:39]], ×200 против [[unit:31]] / [[unit:43]] / [[unit:44]],
×100 против [[unit:32]] / [[unit:45]] / [[unit:46]], ×50 против
[[unit:47]] / [[unit:48]], ×33 против [[unit:33]], ×30 против
[[unit:34]], ×25 против [[unit:40]], ×5 против [[unit:41]].

[[unit:102]] имеет ×3 против [[unit:42]] — именно поэтому лансер
относительно эффективен против Звезды смерти.

Полные таблицы есть на странице каждого юнита в секциях
«Скорострельность по целям» и «Уязвим к скорострельности».

## Баллистика и маскировка

[[unit:103]] стрелка и [[unit:104]] цели работают друг против друга:

```
эффект_маскировки = max(0, маскировка_цели - баллистика_стрелка)
фактор            = 1 - 1 / (1 + эффект_маскировки × 0.2)
промахи           = floor(выстрелы × фактор)
выстрелы_итого    = выстрелы - промахи
```

Если `баллистика ≥ маскировка`, эффекта нет. При `маскировка - баллистика = 5`
теряется ~50 % выстрелов; при разнице 10 — ~67 %.

Некоторые юниты имеют ненулевую базовую маскировку:

- [[unit:325]]: маскировка=4, баллистика=3 (стелс-разведчик)
- [[unit:42]]: баллистика=4
- [[unit:48]]: баллистика=2, маскировка=2
- [[unit:47]]: баллистика=1, маскировка=2

## Частичные потери

Если урона хватает не на целое число юнитов, остаётся один **повреждённый**
с долей корпуса `процент_корпуса`:

```
уничтожено = floor(корпус_раунда / корпус)
остаток    = корпус_раунда - уничтожено × корпус

если остаток > 0:
  количество     = уничтожено + 1
  повреждён      = 1
  процент_корпуса = (остаток / корпус) × 100
```

Повреждённый юнит переносит пониженный корпус на следующий раунд. После боя
его можно починить в [[unit:100]].

## Победитель

После 6 раундов (или раньше, если одна сторона полностью уничтожена):

- **атакующий** — у атакующего остался хотя бы один юнит, у защитника — нет
- **защитник** — наоборот
- **ничья** — обе стороны живы или обе мертвы

## Обломки и поля обломков

**30 %** стоимости (металл + кремний) уничтоженных кораблей падает на
орбиту в виде поля обломков. **Уничтоженная оборона восстанавливается
на 70 %** после боя.

Собирают обломки [[unit:37]] (миссия — переработка). Чем больше
обломков на координате, тем дольше идёт сбор.

## Образование луны

Если в бою образовалось много обломков, с шансом до **20 %**
(пропорционально объёму, до 2 млн в металлическом эквиваленте) на
координате появляется луна. Луна — отдельная «планета», на ней можно
строить уникальные сооружения: [[unit:55]] и [[unit:56]].

## Исходный код

Боевой движок реализован в `backend/internal/battle/engine.go`:

- `applyShots` — алгоритм щита и распределение выстрелов
- `applyMasking` — формула баллистики и маскировки
- `runRound` — один раунд боя
- `RunBattle` — точка входа, 6 раундов
