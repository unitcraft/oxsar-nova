Бой пошаговый — до **6 раундов**. В каждом раунде стороны одновременно
стреляют по снимку, сделанному в начале раунда. Бой детерминирован при
фиксированном `seed` — повторный расчёт даёт идентичный результат.

## Основные параметры юнита

- **Атака** (`A`) — урон одного юнита за раунд.
- **Корпус** (`H`) — прочность. Накапливает урон после пробития щита;
  при `H ≤ 0` юнит уничтожается.
- **Щит** (`S`) — регенерирует в начале каждого раунда до 100 %.
- **Front** — приоритет цели (выше → атакуют первым). Большинство юнитов
  имеют front=10. У [[unit:102]] front=8 — бьёт раньше обычных.

## Влияние технологий

Каждый уровень исследования даёт **линейный** бонус +10 %:

```
эффективная_атака  = A × (1 + gun_tech × 0.10)
эффективный_корпус = H × (1 + shell_tech × 0.10)
эффективный_щит    = S × (1 + shield_tech × 0.10)
```

Для [[unit:15]] / [[unit:16]] / [[unit:17]] на уровне 10 — +100 % к атаке /
щиту / корпусу.

## Алгоритм одного выстрела

Один стек из `N` юнитов наносит `N × эффективная_атака` урона за раунд,
распределяемого по целям пропорционально их `weight = 2^Front × Quantity`.

Когда выстрел доходит до цели, применяется **трёхступенчатый
алгоритм щита** (порт из oxsar2-java `Units.processAttack`):

### 1. Порог игнора щита

```
ignoreAttack = базовый_щит / 100
```

Если `attack ≤ ignoreAttack`, выстрел **полностью поглощается щитом**,
корпус не страдает.

Важное исключение: для **планетарных щитов** ([[unit:49]] и [[unit:50]])
`ignoreAttack = 0` — любая атака пробивает их (так в legacy).

`ignoreAttack` считается от **базового** щита, **без** tech-множителя.
Это значит: shield_tech повышает абсорбцию щита, но не делает его
абсолютным барьером.

### 2. Распределение выстрелов

В начале раунда щит полный → `shieldDamageFactor = 1.0` →
`shieldDestroyFactor = 0.01` (минимум clamp). Это значит, что в первом
ударе **99 % выстрелов идут в щит**, **1 % проходит к корпусу**. По мере
истощения щита `shieldDestroyFactor` растёт, и больше выстрелов проходят
к корпусу.

```
shieldDamageFactor   = turnShield / startTurnShield
shieldDestroyFactor  = clamp(1 - shieldDamageFactor, 0.01, 1.0)

shieldShotsNumber    = ceil(shots × (1 - shieldDestroyFactor))
shellShotsNumber     = shots - shieldShotsNumber
```

### 3. Расчёт урона

**По щиту** (если `attack > ignoreAttack`):

```
shieldPower = min(
  attack × shieldShotsNumber,    // поток силы
  shield × shieldShotsNumber,    // cap: один выстрел не сильнее щита одного юнита
  оставшийся_turnShield          // cap: щит не уходит в минус
)
turnShield -= shieldPower
```

**По корпусу**:

```
shellPower = min(
  attack × shellShotsNumber,
  shell  × shellShotsNumber,
  оставшийся_turnShell
)
turnShell -= shellPower
```

### 4. Регенерация

В начале каждого следующего раунда:

```
turnShield = N × эффективный_щит   // 100 % регенерация
turnShell остаётся как есть          // корпус не восстанавливается
```

## Скорострельность (rapidfire)

Некоторые юниты получают дополнительные выстрелы за раунд против
определённых целей:

```
shots_итого = quantity × rapidfire[стрелок][цель]
```

Например, [[unit:33]] имеет ×6 против [[unit:31]] и ×10 против
[[unit:43]] — за один раунд один крейсер делает 6 выстрелов по
лёгким истребителям или 10 по ракетным установкам.

[[unit:42]] имеет rapidfire почти против всего флота — её таблица:
×250 против [[unit:29]] / [[unit:30]] / [[unit:37]] / [[unit:38]] /
[[unit:39]], ×200 против [[unit:31]] / [[unit:43]] / [[unit:44]],
×100 против [[unit:32]] / [[unit:45]] / [[unit:46]], ×50 против
[[unit:47]] / [[unit:48]], ×33 против [[unit:33]], ×30 против
[[unit:34]], ×25 против [[unit:40]], ×5 против [[unit:41]].

[[unit:102]] имеет ×3 против [[unit:42]] — именно поэтому лансер
относительно эффективен против Звезды смерти.

Полные таблицы есть на странице каждого юнита в секциях
«Скорострельность по целям» и «Уязвим к скорострельности».

## Баллистика и маскировка

[[unit:103]] стрелка и [[unit:104]] цели работают друг против друга:

```
maskingEffect = max(0, masking_цели - ballistics_стрелка)
factor        = 1 - 1 / (1 + maskingEffect × 0.2)
missed        = floor(shots × factor)
shots_итого   = shots - missed
```

Если `ballistics ≥ masking`, эффекта нет. При `masking - ballistics = 5`
теряется ~50 % выстрелов; при разнице 10 — ~67 %.

Некоторые юниты имеют ненулевую базовую маскировку:

- [[unit:325]]: masking=4, ballistics=3 (стелс-разведчик)
- [[unit:42]]: ballistics=4
- [[unit:48]]: ballistics=2, masking=2
- [[unit:47]]: ballistics=1, masking=2

## Ablation — частично повреждённые юниты

Если урона хватает не на целое число юнитов, остаётся один **damaged**
с долей корпуса `shellPercent`:

```
fullKills = floor(turnShell / shell)
remainder = turnShell - fullKills × shell

если remainder > 0:
  quantity = fullKills + 1
  damaged  = 1
  shellPercent = (remainder / shell) × 100
```

Damaged-юнит переносит пониженный корпус на следующий раунд. После боя
повреждённые юниты можно починить в [[unit:100]].

## Победитель

После 6 раундов (или раньше, если одна сторона полностью уничтожена):

- **attackers** — у атакующего остался хотя бы один юнит, у защитника — нет
- **defenders** — наоборот
- **draw** — обе стороны живы или обе мертвы

## Обломки и поля обломков

**30 %** стоимости (металл + кремний) уничтоженных кораблей падает на
орбиту в виде поля обломков. **Уничтоженная оборона восстанавливается
на 70 %** после боя.

Собирают обломки [[unit:37]] (миссия — переработка). Чем больше
обломков на координате, тем дольше идёт сбор.

## Образование луны

Если в бою образовалось много обломков, с шансом до **20 %**
(пропорционально объёму, до 2 млн в металлическом эквиваленте) на
координате появляется луна. Луна — отдельная «планета», на ней можно
строить уникальные сооружения: [[unit:55]] и [[unit:56]].
