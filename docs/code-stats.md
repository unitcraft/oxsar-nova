## TL;DR

- **3 дня разработки** (2026-04-21 … 2026-04-23), один автор, **359 коммитов**.
- **~50 000 LOC** своего кода (Go + TS/TSX + SQL + YAML), без сгенерированного.
- Легаси `oxsar2` (PHP+TPL+JS+CSS) весит ~77k LOC. Новая версия переносит
  ядро примерно в 1.5× меньшем объёме, при этом добавляет тесты и типизацию.
- Покрытие тестами неоднородное: `battle` и `economy` — полноценно,
  большинство доменов — заглушка или ноль.

## Материал для статьи о создании игры

Самые сильные факты:

1. **Три дня, один автор, 359 коммитов, ~50k LOC своего кода** — темп
   возможен только при плотной связке «план → PR → коммит» с AI-ассистентом.
2. **Доля `docs:` коммитов — 24%** — процесс записан в четырёх файлах
   (plans/, simplifications.md, project-creation.txt, ui-dev-log.md)
   и индексируется чаще, чем код меняется.
3. **Порт легаси при 65% объёма** — без потери функциональности, с типизацией
   и тестами, за счёт ухода от TPL+jQuery к React+TS и от PHP к Go.
4. **`battle` покрыт 1:1** кодом и тестами (759 vs 628 LOC) — осознанное
   выполнение порога 85% из CLAUDE.md.
5. **84 зафиксированных trade-off'а** в одном файле с планом возврата —
   пример, как можно не терять техдолг в ходе быстрой разработки.

### Что стоит добавить позже

- Таймлайн «план → PR → коммит» — ручная связка планов с sha.
- Coverage через `go test -cover` и `vitest --coverage` (сейчас не запускалось).
- Hotspot-анализ в связке с simplifications (какие файлы и простят, и чаще всего трогают).
